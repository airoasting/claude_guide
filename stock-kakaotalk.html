STEP 1 · DART API 키 발급

OpenDART에서 API 키 발급받기

DART 스킬이 공시 데이터를 조회하려면 인증 키가 먼저 필요합니다

DART 스킬은 금융감독원 전자공시시스템 DART의 Open API를 사용합니다. 무료로 발급받을 수 있고 하루 10,000건까지 호출할 수 있어, 개인 분석 용도로는 충분합니다.

🔗 opendart.fss.or.kr 바로가기 금융감독원
1
사이트 접속과 회원가입
opendart.fss.or.kr에 접속해 회원가입을 진행합니다. 이메일 인증까지 끝내야 다음 단계로 넘어갈 수 있습니다.
2
인증키 신청 페이지로 이동
로그인한 다음 상단 메뉴에서 인증키 신청/관리 > 인증키 신청으로 들어갑니다.
3
신청서 작성과 제출
이용 목적(예: "개인 투자 분석")을 입력하고 신청합니다. 대개 즉시 발급되며, 늦어도 다음 영업일에는 이메일로 도착합니다.
4
API 키 복사
인증키 관리 페이지에서 발급된 40자 키를 복사합니다. 이 키는 STEP 4에서 DART 스킬의 .env 파일에 등록합니다.
보안

API 키는 비밀번호와 같습니다. GitHub 공개 저장소에 절대 커밋하지 마세요. STEP 4에서 .env 파일에 저장하고 .gitignore에 추가합니다.

.env는 environment(환경)의 줄임말로, API 키처럼 노출되면 안 되는 값을 코드 바깥에 따로 담아두는 텍스트 파일입니다. 키를 코드 안에 그대로 적어두지 않고 이 파일로 빼두면, 실수로 GitHub에 함께 올릴 위험을 줄일 수 있습니다.

.gitignore는 Git이 추적하지 않을 파일 목록을 적어두는 곳입니다. 여기에 .env를 한 줄 적어두면 git push를 해도 .env는 자동으로 빠집니다.

STEP 2 · PlayMCP 가입 & 도구함 추가

PlayMCP에서 카카오톡 도구함 만들기

Claude Cowork에 연결하기 전에, PlayMCP에서 먼저 회원가입하고 쓸 도구를 도구함에 담아둬야 합니다

PlayMCP는 카카오톡, 네이버 같은 한국 서비스를 묶어 Claude로 보내주는 MCP 커넥터입니다. Claude에 연결하기 전에 PlayMCP 쪽에서 먼저 계정을 만들고, 쓸 도구를 도구함에 담아두는 준비 작업이 필요합니다.

🔗 PlayMCP 공식 사이트: playmcp.kakao.com PlayMCP
1
PlayMCP 공식 사이트 접속
playmcp.kakao.com에 접속해 가입 절차를 시작합니다.
2
카카오 계정으로 회원가입
본인이 메시지를 받을 카카오 계정으로 가입합니다. 이 계정은 STEP 3의 OAuth 인증에서도 똑같이 사용합니다.
3
도구함 열기
PlayMCP 대시보드에서 도구함 메뉴로 이동합니다. 도구함은 Claude에서 쓸 도구를 모아두는 공간입니다.
4
"카카오톡 나에게 보내기" 도구 추가
도구 카탈로그에서 카카오톡 · 나에게 보내기 도구를 찾아 도구함에 담습니다. 담은 뒤에 꼭 저장하세요.
5
도구함 구성 확인
도구함 목록에 카카오톡 도구가 들어갔는지 확인합니다. 여기까지 끝나면 PlayMCP 쪽 준비는 완료입니다. 다음은 Claude Cowork에 연결하는 단계입니다.
핵심

도구함에 담지 않은 도구는 Claude에서 쓸 수 없습니다. 카톡 외에 네이버 검색, 캘린더 같은 다른 도구도 함께 쓰고 싶다면 미리 담아두세요.

STEP 3 · Claude Cowork에 카톡 연결

Claude Cowork 커넥터에 PlayMCP 연결하기

PlayMCP 도구함을 Claude Cowork에 연동하면 채팅에서 바로 카톡을 보낼 수 있습니다

Claude Cowork의 커넥터 메뉴에서 PlayMCP를 추가합니다. OAuth로 STEP 2의 카카오 계정에 연결하면, 이후 Cowork 대화에서 카톡 전송을 자연어로 요청할 수 있습니다.

📖 PlayMCP 공식 가이드: Claude에 도구함 연결하기 Notion
✨ Claude Cowork Pro / MAX 사용자
  1. Claude Cowork에서 설정 > 커넥터로 이동합니다.
  2. 상단의 [커넥터 둘러보기]를 선택합니다.
  3. 디렉토리에서 PlayMCP를 검색하거나 목록에서 고릅니다.
  4. [연결]을 누르면 Cowork와 PlayMCP 연동 과정이 시작됩니다.
  5. STEP 2에서 PlayMCP에 가입할 때 쓴 카카오 계정으로 로그인한 뒤 OAuth 인증과 동의를 진행합니다.
  6. 연결이 끝나면 Cowork 대화에서 PlayMCP 도구함에 담은 도구를 바로 쓸 수 있습니다.
👥 Claude Cowork Team / Enterprise 사용자
  1. Cowork 관리자 설정 > 커넥터로 이동합니다. (Owner만 가능합니다.)
  2. 하단의 [커넥터 둘러보기]를 선택합니다.
  3. 디렉토리에서 PlayMCP를 검색하거나 목록에서 고릅니다.
  4. [팀에 추가]를 눌러 팀 구성원이 커넥터를 연결할 수 있게 합니다.
  5. 이후 각 구성원이 개인 설정 > 커넥터에서 PlayMCP를 찾아 [연결]을 누릅니다.
유의 사항

Claude 커넥터는 유료 플랜(Pro, MAX, Team, Enterprise)에서만 쓸 수 있습니다. PlayMCP 설정 > 연결된 서비스에서 언제든 Cowork의 액세스 권한을 해제할 수 있습니다.

연결 확인

Cowork에서 "카톡으로 '테스트' 라고 보내줘"라고 입력했을 때 본인 카톡에 메시지가 오면 연결이 끝난 것입니다.

STEP 4 · DART 스킬 등록

DART 스킬을 Claude Cowork에 등록하기

3

# 프로젝트 루트/.env 파일에 추가 DART_API_KEY=여기에_40자_API_키_붙여넣기

Cowork에서 인식 확인

STEP 5 · daily-stock-report 플러그인 패키징

분석과 카톡 전송을 daily-stock-report 플러그인으로 묶기

매번 두 번씩 명령하지 않고, 한 줄로 분석과 전송이 한꺼번에 일어나도록 패키징합니다

STEP 4에서는 분석 명령을, STEP 3에서는 카톡 전송 기능을 준비했습니다. 이제 둘을 하나의 플러그인으로 묶어, Cowork에서 한 줄로 호출할 수 있게 만듭니다. 이 플러그인은 두 가지 방식으로 동작합니다.

  • /daily-stock-report — 기본 종목 리스트 전체를 분석
  • /daily-stock-report 카카오 — 뒤에 종목명을 붙이면 그 종목만 분석

Claude Cowork에 다음과 같이 요청하세요.

~/.claude/plugins/daily-stock-report 폴더에 PLUGIN.md 파일을 만들어줘. 플러그인 이름: daily-stock-report 목적: 관심 종목을 DART로 분석하고 카카오톡으로 요약 전송 동작: - 인자 없이 호출 (/daily-stock-report): 기본 종목 리스트 전체 분석 - 인자로 종목명 (/daily-stock-report 카카오): 해당 종목 하나만 분석 - 분석이 끝나면 PlayMCP 카카오톡 나에게 보내기로 요약 전송 내용: - frontmatter에 name, description, trigger 정의 - 기본 종목 리스트: 카카오, 삼성전자, NAVER - 각 종목마다 /dart {기업명} 실행 → HTML 리포트 생성 - 요약 포맷: 매출/영업이익 YoY, 최고/최저 페르소나, 1줄 코멘트

Claude Cowork가 만들어주는 PLUGIN.md 예시 구조는 이렇습니다.

--- name: daily-stock-report description: 관심 종목을 DART로 분석하고 카카오톡으로 요약 전송. 인자 없이 호출하면 기본 종목 리스트 전체를, 종목명을 인자로 받으면 그 종목 하나만 처리. "아침 종목 리포트", "/daily-stock-report"로도 트리거. --- # Daily Stock Report ## 사용법 - 전체 종목 분석: `/daily-stock-report` - 단일 종목 분석: `/daily-stock-report {기업명}` (예: `/daily-stock-report 카카오`) ## 기본 종목 리스트 (인자가 없을 때 사용) - 카카오 - 삼성전자 - NAVER ## 실행 단계 1. 인자가 있으면 그 종목만, 없으면 기본 리스트 순서대로 `/dart {기업명}` 실행 → HTML 리포트 생성 2. 분석이 끝난 종목의 요약을 추출 3. PlayMCP 카카오톡 도구로 나에게 보내기로 전송 4. 전송 완료 후 output/ 경로를 한 줄로 안내 ## 카톡 메시지 포맷 [종목명] 매출 X (YoY +X%) / 영업이익 X (YoY +X%) 최고: {페르소나} (점수): 1줄 근거 최저: {페르소나} (점수): 1줄 근거 한 줄 코멘트

이제 Cowork 어디서든 /daily-stock-report 한 줄이면 기본 종목 전체를 한꺼번에 분석합니다. /daily-stock-report 카카오처럼 종목명을 붙이면 그 종목 하나만 빠르게 받아볼 수 있습니다.

종목 리스트 변경

PLUGIN.md의 ## 기본 종목 리스트 섹션만 고치면 분석 대상을 자유롭게 바꿀 수 있습니다. Cowork를 다시 켜지 않아도 곧바로 반영됩니다.

잠깐, 스킬과 플러그인이 뭐가 다른가요?

DART는 스킬, daily-stock-report는 플러그인. 둘은 비슷해 보이지만 역할이 다릅니다

스킬 (Skill)

  • 한 가지 작업을 잘 해내는 단위
  • 예: dart (종목 분석)
  • 경로: ~/.claude/skills/{이름}/SKILL.md
  • SKILL.md 한 파일에 동작과 트리거를 정의

플러그인 (Plugin)

  • 여러 스킬과 도구를 묶은 워크플로우 단위
  • 예: daily-stock-report (분석 + 카톡 전송)
  • 경로: ~/.claude/plugins/{이름}/PLUGIN.md
  • 스킬을 조합해 한 번에 실행할 작업으로 묶음

쉽게 말하면 스킬은 부품, 플러그인은 그 부품을 모아 만든 제품입니다. DART 스킬은 "분석"이라는 한 가지 일만 하고, daily-stock-report 플러그인은 그 스킬을 호출한 뒤 PlayMCP 카톡 도구로 결과를 보내는 두 단계를 한 줄로 묶어줍니다.

STEP 6 · 스케줄 자동화

월~금 아침 9시에 daily-stock-report 자동 실행

Claude Cowork의 schedule 기능에 등록해두면 매일 아침 카톡이 알아서 도착합니다

Claude Cowork에는 정기 작업을 등록하는 schedule 스킬이 있습니다. cron 표현식으로 원하는 시간을 지정합니다.

/schedule 이름: 아침 종목 리포트 설명: 매주 평일 오전 9시에 daily-stock-report 자동 실행 cron: 0 9 * * 1-5 실행 내용: /daily-stock-report

cron 표현식 0 9 * * 1-5는 분(0), 시(9), 일(*), 월(*), 요일(1~5, 월~금)을 뜻합니다.

등록을 마치면 매주 월~금 오전 9시에 Cowork가 자동으로 이렇게 움직입니다.

⏰
09:00 자동 시작
등록된 cron 시각이 되면 schedule이 /daily-stock-report를 실행합니다.
📊
종목 분석
PLUGIN.md에 등록된 기본 종목 리스트를 순서대로 /dart로 분석하고 HTML 리포트를 만듭니다.
💬
카톡 전송
요약 메시지가 PlayMCP 카카오톡 도구로 나에게 보내기에 도착합니다. 9시 10분쯤 출근길에서 확인할 수 있습니다.
스케줄 관리

등록된 작업 목록은 /schedule list로 확인합니다. 임시 중지와 삭제는 /schedule 메뉴에서 고르면 됩니다.

🚀 확장 아이디어
  • 이상 징후 알림: 영업이익이 전년 대비 30% 넘게 줄면 즉시 카톡으로 알립니다
  • 실적 발표 시즌: 분기 실적 발표 주에는 하루 두 번(오전 9시, 오후 6시) 실행합니다
  • 동종 비교: 카카오, 네이버 같은 같은 업종 3사를 비교하는 리포트를 추가합니다
  • 팀 공유: PlayMCP 슬랙 도구를 도구함에 함께 담아 팀 채널에도 전송합니다
  • 주간 회고: 매주 금요일 저녁에 한 주간 리포트를 모아 평가 정확도를 점검합니다
트러블슈팅

막힐 때 자주 만나는 5가지 문제

실제로 따라하다가 멈추는 지점을 미리 모았습니다

Q1. /dart나 /daily-stock-report를 입력해도 자동완성에 안 나옵니다

원인: Claude Cowork가 스킬 디렉터리를 인식하지 못한 상태입니다.

해결:

  1. 다음 명령으로 두 파일이 모두 있는지 확인합니다.
    • Mac · Linux (터미널): ls ~/.claude/skills/dart/SKILL.md 와 ls ~/.claude/plugins/daily-stock-report/PLUGIN.md
    • Windows (명령 프롬프트): dir %USERPROFILE%\.claude\skills\dart\SKILL.md 와 dir %USERPROFILE%\.claude\plugins\daily-stock-report\PLUGIN.md
    • Windows (PowerShell): Get-ChildItem $HOME\.claude\skills\dart\SKILL.md 와 Get-ChildItem $HOME\.claude\plugins\daily-stock-report\PLUGIN.md
  2. Cowork 세션을 새로고침하거나 새 대화를 시작합니다.
  3. 그래도 안 보이면 /help로 스킬 목록을 확인합니다.
Q2. DART_API_KEY 환경변수가 없습니다 에러가 납니다

원인: .env 파일이 잘못된 위치에 있거나, 키 형식이 틀렸습니다.

해결:

  1. .env는 스킬을 실행하는 프로젝트 루트(현재 작업 디렉터리)에 두어야 합니다.
  2. 키 형식을 확인합니다. DART_API_KEY=... 사이에 공백이나 따옴표 없이 붙여 씁니다.
  3. echo $DART_API_KEY로 셸 환경에 키가 들어갔는지 확인합니다.
Q3. 카카오톡으로 메시지가 안 옵니다

원인 후보 4가지:

  • PlayMCP 도구함에 카톡 도구 누락: STEP 2에서 "카카오톡 나에게 보내기"를 도구함에 담았는지 확인합니다.
  • Cowork 커넥터 미연결: Cowork 설정 > 커넥터에서 PlayMCP가 "연결됨" 상태인지 확인합니다.
  • OAuth 권한 만료: PlayMCP에서 카카오 계정으로 다시 로그인해야 할 수 있습니다.
  • "나에게 보내기" 비활성화: 카카오톡 앱에서 본인 채널이 켜져 있어야 합니다.

1차 진단: Cowork에서 "카톡으로 테스트 보내줘"라고 입력해 봅니다. 메시지가 안 오면 PlayMCP 연결이나 도구함 문제, 오면 플러그인 쪽 문제입니다.

Q4. 일일 호출 한도 초과 메시지가 뜹니다

원인: DART API 일일 한도 10,000건을 넘었습니다. (한 종목 분석에 보통 30~50건을 호출합니다.)

해결:

  • 한도는 매일 한국 시간 자정에 초기화됩니다. 다음 날이면 다시 쓸 수 있습니다.
  • 단기 회피: 종목 수를 줄이거나 분석 빈도를 낮춥니다.
  • 장기 해결: OpenDART에 추가 한도를 신청합니다 (기업 계정에 한해 가능합니다).
Q5. 스케줄에 등록했는데 9시에 실행이 안 됩니다

원인 후보:

  • Cowork 세션 비활성: 로컬 schedule은 Cowork가 켜져 있어야 동작합니다. 노트북이 절전모드면 실행되지 않습니다.
  • cron 표현식 오류: /schedule list로 등록 상태와 다음 실행 시간을 확인합니다.
  • 시간대 불일치: 시스템 타임존이 한국 시간이 아니면 cron 시간이 어긋납니다.

대안: 24/7 실행이 필요하면 원격 스케줄링(서버나 호스팅 routine)을 고려하세요.
