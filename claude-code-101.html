클로드 코드란?

채팅 한 번으로 코드와 파일을 다루게 합니다

코딩을 몰라도 한국어로 부탁만 하면 됩니다

클로드 코드는 채팅으로 부탁하면 실제 파일을 만들고 고치고 정리해 주는 도구입니다. 비개발자도 평소처럼 "이렇게 해줘"라고 한국어로 말하면, Claude가 알아서 폴더를 둘러보고 파일을 만들거나 바꿉니다.

두 가지 사용 방식

채팅 앱 방식 (이 가이드)

  • Claude Desktop 앱 실행
  • Code 모드 선택
  • 한국어로 자연어 요청
  • 진행 상황을 채팅에서 확인

터미널 방식 (개발자용)

  • 터미널(검은 창) 실행
  • claude 명령어 입력
  • 슬래시 명령으로 세밀 제어
  • 자동화 워크플로우에 적합

Windows · Git 설치

Windows · Git 설치 Windows

내 노트북에 맞는 파일을 고르고, 설치 마법사는 Next만 누르면 됩니다

🔍 받기 전, 내 윈도우 사양부터 확인하세요

Git 설치 파일은 노트북에 따라 32-bit, 64-bit, ARM64 세 가지로 나뉩니다. 맞지 않는 파일을 받으면 설치는 되어도 제대로 돌지 않거나 느려집니다. 받기 전에 한 번만 확인하세요.

  • 키보드에서 Windows 키와 Pause/Break 키를 함께 누르세요. Pause 키가 없는 노트북이면 시작 → 설정 → 시스템 → 정보 순서로 들어가도 됩니다.
  • 열린 화면의 "장치 사양" 항목에서 "시스템 종류" 줄을 찾으세요.
  • 그 줄에 적힌 내용을 보고 받을 파일을 고르면 됩니다.

64-bit (대부분 여기)

  • "64비트 운영 체제, x64 기반 프로세서"
  • 최근 5~10년 안에 산 노트북은 거의 다 여기에 해당합니다
  • git-scm.com의 기본 다운로드 버튼을 누르면 이 파일이 받아집니다

32-bit 또는 ARM64

  • "32비트 운영 체제, x86 기반 프로세서"면 32-bit 설치 파일을 받으세요
  • "ARM 기반 프로세서"(예: Snapdragon 노트북)면 ARM64 설치 파일을 받으세요
  • 두 경우 모두 git-scm.com 페이지 아래의 "Other Git for Windows downloads"에서 직접 고르면 됩니다
1
공식 사이트 접속
git-scm.com/download/win에 접속하세요. 위에서 확인한 사양에 맞는 파일을 받으면 됩니다. 64-bit면 메인 버튼을 누르고, 32-bit나 ARM64면 페이지 아래 "Other Git for Windows downloads"에서 고르세요.
2
설치 파일 더블클릭
받은 Git 설치 파일을 더블클릭하세요. 옵션 화면이 여러 번 뜨는데, 모두 Next를 누르고 마지막에 Install을 누르면 됩니다. 기본값이 가장 무난합니다.
3
끝
"Finish" 화면이 뜨면 닫으세요. Git은 평소에 직접 쓸 일이 거의 없습니다. Claude가 알아서 호출해 줍니다.
Git을 따로 열어 봐야 하나요

아니요. 설치만 해두면 됩니다. 작업 중 파일이 잘못됐을 때 Claude한테 "방금 작업 되돌려줘"라고 부탁하면, Claude가 Git을 대신 써줍니다.

Mac · Git 설치

Mac · Git 설치 macOS

맥은 더 간단합니다. 자동으로 깔립니다

macOS는 Git이 시스템 안에 들어 있어서 별도 다운로드가 필요 없습니다. 처음 한 번만 깔라는 안내를 받아주면 그걸로 끝입니다.

1
터미널 열기
Cmd + Space로 Spotlight를 열고 터미널이라고 치면 검은 창이 뜹니다.
2
한 줄만 치기
아래 한 줄을 복사해 붙여 넣고 엔터를 누르세요. Git이 없으면 맥이 자동으로 설치 창을 띄워줍니다. "설치"를 누르면 됩니다.
Terminal (macOS)
xcode-select --install
"이미 설치되어 있습니다" 메시지가 뜨면

이미 다 됐다는 뜻입니다. 그대로 터미널을 닫으셔도 됩니다. 다음 단계로 가세요.

첫 시작

Claude Desktop 열고 첫 부탁 시키기

Windows와 Mac 모두 같은 흐름입니다

이제부터는 모든 작업이 Claude Desktop 채팅창 안에서 이뤄집니다. 명령어를 외울 필요 없습니다. 한국어로 부탁만 하면 됩니다.

1
작업 폴더 미리 만들어두기
바탕화면이나 문서 폴더에 빈 폴더를 하나 만드세요. 예를 들어 claude-test라는 이름이면 됩니다. 이 폴더가 Claude의 작업 공간이 됩니다.
2
Claude Desktop 실행
앱을 켭니다. 채팅 화면이 뜨면 됩니다.
3
Code 모드 켜기
채팅 입력창 왼쪽 하단의 + 버튼 또는 위쪽 모델 선택 드롭다운에서 "Claude Code"를 고르세요. 선택하면 입력창 위에 "Code" 표시가 뜹니다.
4
작업 폴더 알려주기
화면 안내에 따라 1단계에서 만든 폴더를 선택하세요. 폴더를 채팅창에 그냥 끌어 놓아도 됩니다.
5
자연어로 부탁하기
아래 예시를 그대로 복사해 넣어보세요. Claude가 폴더를 둘러보고 답을 줍니다.
첫 부탁 예시
이 폴더에 뭐가 들어 있는지 보고, 내가 여기서 어떤 일을 할 수 있을지 한 문장으로 말해줘.
✅ 답이 돌아오면 끝

Claude가 폴더 안을 둘러보고 답을 주면 설치와 첫 실행 모두 정상입니다. 이제 진짜 작업을 시작할 수 있습니다.

"파일에 접근해도 될까요"

Claude는 파일을 만들거나 바꾸기 전에 매번 물어봅니다. 처음에는 꼼꼼히 읽고 허용하세요. 익숙해진 뒤에 자주 쓰는 동작만 자동 허용으로 바꾸셔도 됩니다.

중요한 파일은 작업 폴더 밖에 두세요

Claude는 알려준 폴더 안에서만 작업합니다. 그래서 건드리면 안 되는 파일은 처음부터 작업 폴더에 넣지 않는 것이 가장 안전합니다. 회사 기밀 문서·계약서·미공개 재무 자료는 별도 폴더에 두세요.

잘 쓰는 법

잘 쓰는 5가지 습관

알아두면 실패 확률을 크게 줄입니다

1
실행 전, 무엇을 만들지부터 정리하기
바로 "만들어줘"라고 하면 안 됩니다. 방향이 잘못된 채로 작업이 시작되면 중간에 되돌리기 어렵습니다. 한국어로 부탁하기 전에, 짧게라도 무엇을 만들지부터 채팅에서 정리하세요.
자세히 보기 멀티 페르소나 검토 + 프롬프트 예시
▾
💡 여러 시각으로 한 번 더 보기

혼자 생각하면 놓치는 부분이 많습니다. Claude에게 기획자, 개발자, 실사용자 세 사람의 시각으로 같은 아이디어를 검토시키면, 빠진 곳이 잘 보입니다.

  • 기획자: 이 기능이 정말 필요한가, 더 간단한 방법은 없는가
  • 개발자: 구현할 때 막히는 부분은 없는가, 예상 복잡도는 어느 정도인가
  • 실사용자: 실제로 쓸 때 불편하지는 않은가, 빠진 것은 없는가

이 방식을 더 깊게 쓰고 싶으면 멀티 페르소나 가이드를 참고하세요.

계획 토론 프롬프트
나는 [만들고 싶은 것]을 만들려고 해. 세 사람의 시각으로 검토해줘: 1. 기획자: 목적과 범위가 적절한지, 더 쉬운 접근법은 없는지 2. 시니어 개발자: 기술적으로 무엇이 필요한지, 함정은 무엇인지 3. 실사용자: 쓰다 보면 뭐가 불편할 것 같은지 토론 후 합의된 최종 계획을 정리해줘.
2
작게 쪼개서 부탁하기
한 번에 너무 많이 요청하면 방향이 틀어져도 되돌리기 어렵습니다. "1단계만 먼저 만들어줘, 확인하고 이어서 진행할게"가 훨씬 안전합니다.
3
시작 전과 잘 된 시점에 백업하기
Claude는 파일을 직접 수정하거나 삭제합니다. 작업 시작 전에 "git commit으로 현재 상태부터 저장해줘"를 부탁하고, 중간 결과가 마음에 들면 그때마다 한 번 더 같은 부탁을 하세요. 다음 작업이 잘못돼도 여기까지는 안전하게 돌아올 수 있습니다.
4
구체적으로 요청하기
"예쁘게 만들어줘"보다 "배경은 흰색, 텍스트는 16px 이상, 버튼은 파란색"이 훨씬 정확합니다. 색, 크기, 동작을 한 줄씩 풀어 적어주세요.
5
중간에 멈추고 개입하기
예상과 다른 방향으로 가면 끝까지 두지 말고 바로 "잠깐, 이 부분은 다시 생각해보자"라고 끊으세요. 빠르게 끊을수록 되돌릴 양이 줄어듭니다.
🚨 잘못됐을 때, 되돌리는 순서
  1. 즉시 멈추고 상태 확인: "작업 중단. 지금까지 수정한 파일 목록을 알려줘"
  2. Git이 있으면 한 줄로 복구: "git으로 마지막 커밋 상태로 되돌려줘"
  3. Git이 없으면 백업에서 복구: 작업 전 백업 폴더에서 원본을 꺼내 덮어쓰세요
  4. 원인 반영 후 재시작: "이번엔 이렇게 하지 말아줘"라는 규칙을 다음 부탁에 미리 적어 다시 요청하세요
예방이 최선

작업 시작 전 "먼저 git commit으로 현재 상태 저장부터 해줘" 한 줄이면 돌이킬 수 없는 사고를 거의 막을 수 있습니다. Git을 몰라도 Claude가 알아서 처리합니다.

트러블슈팅

자주 막히는 곳

설치하거나 첫 실행하다가 멈추면 여기서 먼저 보세요

Claude Desktop이 안 열려요 공통

PC를 한 번 다시 켜보세요. 그래도 안 열리면 앱을 한 번 지우고 claude.com/download에서 새로 받아 설치해 보세요.

"확인되지 않은 개발자"라며 실행이 안 됩니다 macOS

맥의 보안 안내입니다. 시스템 설정 → 개인정보 보호 및 보안으로 들어가면 아래쪽에 "Claude를 그래도 열기" 버튼이 있습니다. 그걸 누르면 됩니다.

채팅창에서 "Claude Code" 메뉴가 안 보입니다 공통

앱이 옛 버전일 수 있습니다. 앱을 한 번 종료했다가 다시 열어보세요. 자동으로 최신 버전으로 올라옵니다. 그래도 없으면 claude.com/download에서 최신 버전을 다시 받으세요.

로그인 화면에서 막힙니다 공통

브라우저에서 한 번 claude.com으로 로그인이 되는지 먼저 확인하세요. 거기서 되면 앱을 껐다 켜시면 보통 풀립니다.

폴더 접근을 허용했는데 Claude가 못 본다고 합니다 macOS

시스템 설정 → 개인정보 보호 및 보안 → 파일 및 폴더에서 Claude 앱이 해당 폴더에 체크되어 있는지 확인하세요.

폴더 접근을 허용했는데 Claude가 못 본다고 합니다 Windows

OneDrive 동기화 폴더처럼 클라우드와 묶인 폴더는 피하고 평범한 로컬 폴더를 골라보세요.

회사 PC라서 설치가 막힙니다 공통

회사 보안 정책이 외부 앱 설치를 막는 경우입니다. 사내 IT 담당자에게 Claude Desktop과 Git 설치를 요청하세요. 두 도구 모두 공식적으로 검증된 프로그램입니다.

💡 그래도 안 풀리면

가장 빠른 길은 화면에 뜬 메시지를 그대로 캡처해 Claude 채팅에 붙여 넣고 "이게 뭐예요"라고 묻는 것입니다. 윈도우인지 맥인지, 어느 단계에서 막혔는지만 같이 알려주세요. 거의 한 번에 답이 나옵니다.
