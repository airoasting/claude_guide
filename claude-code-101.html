Claude Code란?

Claude Code, 터미널 없이도 됩니다

Claude Code는 원래 터미널(CLI)로 쓰는 도구입니다. 하지만 Claude Desktop 앱을 사용하면 검은 화면 없이 채팅 방식으로 같은 기능을 쓸 수 있습니다. 코딩을 전혀 몰라도 "파일 정리해줘", "간단한 앱 만들어줘"를 자연어로 요청하면 됩니다.

두 가지 사용 방식

노코드 방식 (이 가이드)

  • Claude Desktop 앱 실행
  • Code 모드 선택
  • 채팅으로 자연어 요청
  • 진행 상황 채팅에서 확인

터미널 방식 (CLI)

  • 터미널(검은 화면) 실행
  • claude 명령어 입력
  • 슬래시 명령어 활용
  • 더 세밀한 제어 가능

도입 체크

어디까지 맡길 수 있나: 규모 가이드

✅ 노코드로 충분

  • 파일 수백 개 일괄 정리·이름 변경
  • 엑셀/CSV 가공, 반복 보고서 생성
  • 1~3페이지 내부용 웹/대시보드
  • 단발성 데이터 크롤링·요약
  • 소요 시간 반나절 이내 작업

⚠️ CLI·개발자 협업 권장

  • 외부 서비스 배포가 필요한 제품
  • 결제·로그인 등 보안 민감 기능
  • DB 연동·대용량 데이터 파이프라인
  • 여러 사람이 협업하는 기존 코드베이스
  • 장기 유지보수가 전제되는 시스템
🔒 회사 도입 전, 보안 체크포인트
  • 읽기 범위: 지정한 작업 폴더 안의 파일만 읽습니다. 폴더 바깥은 접근하지 않습니다.
  • 전송 데이터: 요청한 파일·코드는 Anthropic 서버로 전송됩니다. 기밀 문서는 Team/Enterprise 플랜 사용을 권장합니다(기본 학습 미사용).
  • 격리 원칙: 고객 개인정보·계약서·미공개 재무 자료는 별도 폴더에 두고 작업 폴더에 포함하지 마세요.
  • 감사 추적: Claude의 모든 파일 변경은 채팅 로그에 남습니다. 주요 작업은 Git으로 버전 관리하세요.
계획 먼저

실행 전, 계획서부터 만드세요

시행착오를 줄이는 가장 중요한 습관

Claude Code에게 바로 "만들어줘"라고 하면 안 됩니다. 방향이 잘못된 채로 작업이 시작되면 중간에 되돌리기 어렵습니다. 실행 전에 채팅에서 계획서를 먼저 만드세요.

💡 멀티 페르소나 기법으로 계획서 만들기

멀티 페르소나란? Claude 채팅 한 번에 여러 전문가 시각을 동시에 얻는 방법입니다. 기획자, 개발자, 사용자 시점으로 각각 검토하면 혼자 생각할 때 놓치는 것들이 보입니다.

  • 기획자: "이 기능이 진짜 필요한가? 더 간단한 방법은?"
  • 개발자: "구현 시 기술적 제약은? 예상 복잡도는?"
  • 사용자: "실제로 쓸 때 불편하지 않나? 빠진 건 없나?"

토론 결과를 .md 파일(마크다운 문서)로 저장하면 Claude Code가 이 파일을 참고해서 훨씬 정확하게 작업합니다.

계획서 프롬프트 예시

Claude 채팅에서 먼저 이렇게 물어보세요

STEP 1 · 채팅에서 멀티 페르소나 토론 요청
나는 [만들고 싶은 것]을 만들려고 해. 다음 3명의 전문가가 각자 시각으로 검토해줘: 1. 기획자: 목적과 범위가 적절한지, 더 쉬운 접근법은 없는지 2. 시니어 개발자: 기술적으로 무엇이 필요한지, 함정은 무엇인지 3. 실사용자: 쓰다 보면 뭐가 불편할 것 같은지 토론 후 합의된 최종 계획을 정리해줘.
STEP 2 · 계획서를 .md 파일로 저장 요청
위 계획을 PLAN.md 파일로 저장해줘. 형식: 목적, 기능 목록, 제약사항, 실행 순서 포함
왜 .md 파일인가?

Claude Code는 프로젝트 폴더의 파일을 읽습니다. PLAN.md를 폴더에 두면 "PLAN.md를 참고해서 만들어줘"라고 할 수 있어서 매번 설명을 반복하지 않아도 됩니다.

탭하여 열기

PLAN.md 기본 템플릿

복사해서 바로 쓰는 구조 + 고도화 프롬프트

▾

처음엔 이 구조로 시작하세요. Claude에게 "이 형식으로 PLAN.md 만들어줘"라고 해도 됩니다

PLAN.md 구조
# 프로젝트명 ## 목적 [이걸 왜 만드는가, 한 문장으로] ## 핵심 기능 - 기능 1 - 기능 2 - 기능 3 ## 제약사항 - [시간, 예산, 기술 제약] - [절대 건드리면 안 되는 것] ## 실행 순서 1. 1단계 2. 2단계 3. 3단계 ## 하지 말 것 - 사용자 확인 없이 파일 삭제 금지 - [기타 주의사항] ## 중요 메모 - [작업하다 발견한 것, 결정 사항 계속 추가]
🔄 멀티 페르소나로 PLAN.md 고도화하기

초안이 생기면 그냥 쓰지 말고, 채팅에서 한 번 더 검토시키세요. 혼자 생각할 때 놓친 것들이 보입니다.

PLAN.md 고도화 프롬프트
아래는 내 PLAN.md야. 3명의 전문가 시각으로 검토해줘: 1. 기획자: 목적과 기능 범위가 적절한지, 빠진 건 없는지 2. 개발자: 기술적으로 실현 가능한지, 순서가 맞는지 3. 실사용자: 실제로 쓸 때 불편한 점은 없는지 검토 후 개선된 PLAN.md를 다시 작성해줘. --- [PLAN.md 내용 붙여넣기]
핵심 팁

PLAN.md는 한 번 만들고 끝이 아닙니다. Claude Code가 작업 중 "이 부분은 어떻게 할까요?"라고 물으면 그 내용을 PLAN.md에 추가하며 계속 다듬으세요. 규칙은 구체적으로 적을수록 좋습니다. "예쁘게"보다 "배경 흰색, 텍스트 16px 이상"이 훨씬 정확합니다.

실제 완성본 예시

주간 업무 보고 페이지용 PLAN.md

PLAN.md 실제 작성 예시
# 팀 주간 업무 보고 페이지 ## 목적 팀원이 매주 금요일 1분 안에 제출, 팀장이 월요일에 한 화면에서 조회. ## 핵심 기능 - 이름, 이번 주 한 일, 다음 주 할 일, 블로커 입력 - 제출 시 자동으로 날짜 저장 - 팀장 뷰: 주차별로 전원 요약 한 페이지에 표시 - CSV 내보내기 버튼 ## 제약사항 - 사내 서버 없음 → 정적 HTML만 사용 - 로그인 기능 없음 (팀 내부용) - 작업 시간 3시간 이내 ## 실행 순서 1. 입력 폼 HTML (submit.html) 2. 로컬스토리지에 데이터 저장 3. 조회 페이지 (view.html) 4. CSV 내보내기 기능 ## 하지 말 것 - 외부 라이브러리 설치 금지 (순수 HTML/JS만) - 사용자 확인 없이 로컬스토리지 초기화 금지 - 색상은 기존 회사 브랜드(#2A6FDB) 사용 ## 중요 메모 - 폰트: Pretendard - 모바일에서도 잘 보여야 함 (팀원 절반이 휴대폰 제출)
실행하기

Claude Desktop에서 Code 모드 실행

계획서가 준비됐으면 이제 실행입니다

1
Claude Desktop 앱 실행
claude.com/download에서 설치 후 실행합니다. Claude.ai 웹사이트와 다른 별도 앱입니다.
2
채팅창에서 Code 모드 선택
채팅 입력창 왼쪽 하단 "+" 버튼 또는 모델 선택 드롭다운에서 "Claude Code"를 선택합니다. 선택하면 입력창 상단에 "Code" 배지가 뜨고 파일 시스템 접근이 활성화됩니다.
3
작업 폴더 지정
Claude가 작업할 폴더를 지정합니다. PLAN.md를 이 폴더에 미리 넣어두세요.
4
자연어로 요청
"PLAN.md를 참고해서 만들어줘"라고 입력하면 됩니다. 코드를 전혀 몰라도 됩니다.
5
진행 확인 및 수정 요청
Claude가 작업하는 과정이 채팅에 표시됩니다. 방향이 다르면 "이 부분은 이렇게 바꿔줘"라고 바로 개입하면 됩니다.

자연어 요청 예시

이렇게 말하면 됩니다

예시 1 · 파일 정리 자동화
다운로드 폴더에 파일이 너무 많아. 확장자별로 폴더를 만들어서 정리하는 스크립트 만들어줘. pdf는 /문서, jpg와 png는 /사진, 나머지는 /기타 폴더로.
예시 2 · 간단한 웹 페이지 제작
PLAN.md를 참고해서 팀 주간 업무 보고용 간단한 HTML 페이지 만들어줘. 제출자 이름, 이번 주 한 일, 다음 주 할 일 입력란이 있으면 돼.

🚨 잘못됐을 때, 되돌리기

Claude가 예상 밖으로 파일을 바꿨을 때 대처 순서

1
즉시 멈추고 상태 확인
"작업 중단. 지금까지 수정한 파일 목록을 알려줘"라고 입력하세요. Claude가 변경 내역을 요약해 줍니다.
2
Git이 있다면 한 줄로 복구
"git으로 마지막 커밋 상태로 되돌려줘"라고 요청하세요. Git을 쓰고 있다면 안전하게 전부 원복됩니다.
3
Git이 없다면 백업 폴더에서 복구
작업 시작 전 백업한 폴더에서 원본을 꺼내 덮어쓰세요. macOS는 Time Machine, Windows는 파일 기록이 있다면 그걸로도 됩니다.
4
원인 반영 후 재시작
"이번엔 이렇게 하지 말아줘"를 PLAN.md의 하지 말 것에 추가하고 다시 요청하세요.
예방이 최선

작업 시작 전 "먼저 git commit으로 현재 상태 저장부터 해줘" 한 줄이면 돌이킬 수 없는 사고를 거의 막을 수 있습니다. Git을 몰라도 Claude가 알아서 처리합니다.

핵심 원칙

작게 시작해서 확인하며 확장하세요. 한 번에 너무 많이 요청하면 중간에 방향이 틀어져도 되돌리기 어렵습니다. "1단계만 먼저 만들어줘, 확인하고 이어서 진행할게"가 훨씬 안전합니다.

  • 백업 먼저: Claude Code는 파일을 직접 수정하거나 삭제합니다. 중요한 파일은 별도 폴더에 복사해두세요.
  • 구체적으로 요청: "예쁘게 만들어줘"보다 "배경은 흰색, 텍스트는 16px 이상, 버튼은 파란색"이 훨씬 정확합니다.
  • 중간에 멈추기: 예상과 다른 방향으로 가면 바로 "잠깐, 이 부분은 다시 생각해보자"라고 개입하세요.
  • PLAN.md 계속 업데이트: 작업 중 발견한 내용과 결정 사항을 PLAN.md에 계속 추가하세요. 멀티 페르소나로 주기적으로 재검토하면 방향이 흔들리지 않습니다.
