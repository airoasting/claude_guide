이름의 주인 01
Claude

클로드 섀넌 Claude Shannon

생몰1916~2001 국적미국 분야수학·정보 이론

섀넌은 0과 1로 정보를 다루는 토대를 세운 수학자입니다. 그가 만든 정보 이론은 오늘날 모든 디지털 통신과 AI의 밑바탕이 되었습니다. Anthropic은 이 이름을 따와 자사 AI에 붙였습니다.

Anthropic 인간 중심의 AI

Anthropic ← anthropos (그리스어: 사람)

회사 이름 Anthropic은 그리스어 anthropos(사람)에서 왔습니다. "인간 중심의"라는 뜻을 담아, 사람에게 안전한 AI를 만든다는 방향을 보여줍니다.

창업자 두 사람은 이탈리아계 미국인 남매입니다. 2021년 OpenAI에서 독립해 Anthropic을 세웠습니다.

Claude 핵심 기능

Claude가 업계 선두가 된 기능

Anthropic은 Claude를 대화창 안의 AI에서 개발자와 조직이 직접 쓰는 실행 플랫폼으로 확장해 왔습니다. 외부 도구를 연결하는 표준(MCP)을 먼저 세우고, 그 위에 터미널 기반 실행 환경(Claude Code)과 재사용 가능한 업무 패키지(Skills)를 차례로 쌓았습니다.

02 2025년 5월 공식 런칭

Claude Code터미널 기반 코딩 에이전트

개발자의 터미널에서 직접 돌아가며 레포 전체를 읽고·수정하고·테스트하고·커밋까지 자율적으로 수행합니다. Claude 4 공개와 함께 리서치 프리뷰를 거쳐 공식 제품으로 출시되었습니다.

에이전트 실행 환경의 기본값이 되었습니다. IDE에 의존하지 않고 CLI 하나로 구동되며, 하네스·서브 에이전트·MCP·Skills를 한데 묶어 쓰는 오케스트레이터 역할까지 겸합니다.

왜 Claude인가

왜 Claude인가?

비즈니스에서 중요한 6가지 기준으로 Claude를 평가했습니다. 문제 해결 · 문서 · 코딩 · 안전성에서 최상급이며, 토큰 비용은 상대적으로 높습니다.

01 최상

비즈니스 문제 해결력

복잡한 문제를 단계로 쪼개는 힘

구조화·추론에서 선두권입니다. 다만 GPT-5.5 시리즈·Gemini 3.1 Pro와 박빙이며 단독 1위는 아닙니다. 컨설팅형 분석·브리프 작성에 강점이 뚜렷합니다.

02 상

문서·데이터 처리력

장문 보고서와 데이터를 다루는 능력

장문 처리 품질은 최상급입니다. 단, 컨텍스트 크기는 Gemini(1–2M)에 열세이며 초대용량 레포·전체 코드베이스 리서치에는 불리할 수 있습니다.

03 최상

코딩·개발 역량

프로덕션급 코드를 작성하고 리팩터링하는 능력

Claude는 SWE-bench에서 최상위를 기록합니다. Claude Code는 자율 에이전트로 실제 레포를 직접 수정하고 테스트합니다. 6개 축 가운데 가장 명확한 우위입니다.

04 상

신뢰·안전성

프라이버시와 기업용 안전 수준

헌법형 AI 설계 철학은 최상이나, 환각이 완전히 제거된 것은 아닙니다. 컴플라이언스 인증(FedRAMP·산업별 인증) 폭은 MS·Google 대비 제한적입니다.

05 중

생태계·통합성

연동되는 외부 도구의 범위

MCP는 유망한 표준이지만 아직 초기입니다. ChatGPT(GPTs·MS Copilot)와 Gemini(Google Workspace 네이티브) 대비 커넥터 폭·깊이가 명확히 좁습니다.

06 하

토큰 비용 효율

같은 작업당 단가의 부담

Opus·Sonnet 단가는 Gemini Flash·GPT mini 대비 수 배 비쌉니다. Haiku가 있지만 경쟁사 저가 모델 대비 가격 우위가 약합니다. 대량 처리·저단가 자동화에는 부적합합니다.

🛡 사이버 보안
Mythos가 가장 잘합니다

Mythos가 두 시험에서 모두 1위를 차지했습니다. Cybench는 Opus 4.7이 거의 같은 점수를 냈고, CyberGym은 GPT-5.5가 Mythos를 바짝 따라붙었습니다.

</> 코딩
Opus 4.7과 GPT-5.5가 비슷합니다

SWE-bench Pro는 Opus 4.7이 GPT-5.5를 앞서고, SWE-bench Verified는 GPT-5.5가 Opus 4.7을 살짝 앞섭니다. Gemini는 두 시험 모두 확실히 뒤처집니다.

🖥 컴퓨터 사용
시험마다 1위가 다릅니다

Terminal-Bench는 GPT-5.5가 다른 모델을 큰 차이로 앞섭니다. BrowseComp는 GPT-5.5 Pro가 1위이고 Gemini가 그 뒤를 따릅니다. OSWorld는 Opus 4.7과 GPT-5.5가 거의 같은 점수를 받았습니다.

🔧 도구·지식
GPT-5.5가 도구를 가장 잘 씁니다

GDPval, Toolathlon, t2-bench까지 도구를 쓰는 시험은 모두 GPT-5.5가 1위를 차지했습니다. DeepSearchQA만 Opus 4.7이 Gemini를 큰 차이로 앞섭니다.

∑ 수학·학술
수학은 Mythos 다음으로 GPT-5.5 Pro가 강합니다

수학 난제와 도구를 쓰는 시험은 GPT-5.5 Pro가 다른 모델을 큰 차이로 앞섭니다. 도구 없이 푸는 추론 시험은 Opus 4.7이 가장 잘합니다. GPQA는 상위 모델끼리 점수가 거의 같습니다.

* OpenAI는 "Claude Opus 4.7이 SWE-bench Pro(Public) 문제를 학습 때 미리 외웠을 가능성이 있어, 64.3%는 실력보다 암기 덕일 수 있다"며 점수를 신뢰하기 어렵다는 주석을 달았습니다. 단, 이는 OpenAI 측 주장으로 증명된 사실은 아닙니다. 최종 업데이트: 2026.04

Claude1차 자체 검토
ChatGPT API타 모델 교차 검증
이 구조의 핵심. Claude Code가 컨테이너이자 지휘자, 서브 에이전트는 각자의 전문가입니다. PM 에이전트를 따로 두지 않고, 역할을 과하게 쪼개지 않습니다.
  • 리서처: 내 자료(NotebookLM), 외부(Liner), 수치(Google Sheet)를 한 에이전트에서 모읍니다.
  • 스토리텔러: 구조 설계와 본문 작성을 한 맥락에서 처리해 일관성을 지킵니다.
  • 디자이너: 이미지는 Gemini Nano Banana Pro에 넘깁니다.
  • 빌더: PPTX Skill이 템플릿·레이아웃·차트를 .pptx로 묶어냅니다.
  • 리뷰어: Claude 1차 + ChatGPT 교차 검증. 문제 발견 시 해당 단계로 피드백을 돌려보냅니다.
