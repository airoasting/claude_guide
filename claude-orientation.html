그래서 이 사이트는 Claude 하나에 집중합니다. 여러 AI를 얕게 쓰는 것보다, 한 도구를 깊게 익히는 편이 업무에 더 빠르게 쓰입니다.

※ 다만 이미지 생성은 Gemini의 Nano Banana Pro가, 코드 자동 실행·리팩터링은 OpenAI Codex가 더 적합합니다. 용도에 따라 보조 도구로 함께 씁니다.
그런데 Claude, 이름은 어디서 왔을까요? 정보 이론의 아버지 클로드 섀넌(Claude Shannon, 1916~2001, 미국)에서 따왔습니다. 0과 1로 정보를 다루는 토대를 세운 수학자로, 오늘날 모든 디지털 통신과 AI의 밑바탕이 된 인물입니다.

※ 발음은 '클로드' [klɔːd]입니다. 프랑스어에서 온 이름이라 '클라우드(cloud)'가 아니라 '클로드'로 읽습니다. 참고로 Anthropic의 창업자 다리오·다니엘라 아모데이 남매는 이탈리아계 미국인이며, 회사명 Anthropic은 그리스어 anthropos(사람)에서 온 '인간 중심의'라는 뜻입니다.

  • 스토리텔러: 구조 설계와 본문 작성을 한 맥락에서 처리해 일관성을 지킵니다.

  • 빌더: PPTX Skill이 템플릿·레이아웃·차트를 .pptx로 묶어냅니다.
  • 리뷰어: Claude 1차 + ChatGPT 교차 검증. 문제 발견 시 해당 단계로 피드백을 돌려보냅니다.
활용 사다리 · L0~L5

무엇을 배워야 하는가?

AI 활용에는 단계가 있습니다. 지금 위치에서 한 칸만 올라가도 업무가 달라집니다.

L0
수동 운전 · 입문자기본 채팅
Claude.ai 웹·앱에서 질문하고 답을 받습니다. "이미 AI를 쓰고 있다"는 단계입니다. 기본 프롬프트와 모델 선택만 활용합니다.
출발점
L1
운전자 보조 · 세팅자맞춤 프로젝트 세팅
Claude Projects로 프로젝트 지침과 파일, 멀티 페르소나를 설정합니다. 매번 같은 상황을 설명하는 피로가 사라집니다.
여기서 시작 →
L2
부분 자동화 · 검수자디자인 · 플러그인
Claude Design과 Claude in Chrome, Office 플러그인을 연결합니다. 쓰던 도구 안에서 AI가 일합니다.
여기서 시작 →
L3
조건부 자동화 · 연결자Cowork (파일·업무 연결)

여기서 시작 →

여기서 시작 →
L5
완전 자동화 · 설계자멀티 에이전트 오케스트레이션
