이 코스는 AI 자동화의 4가지 도구를 먼저 읽은 뒤 진행하면 더 잘 이해됩니다.

STEP 1

설계 템플릿

5가지 질문에 답하면, 나만의 AI 팀이 설계됩니다

목적과 기준

무엇을, 왜 자동화하고 싶은지 먼저 정리합니다.

Q1. 나는 왜 이것을 하고 싶은가?
예: "매주 3시간 걸리는 주간보고를 30분으로 줄이고 싶다"
Q2. 좋은 결과물의 기준은 무엇인가?
예: "팀장이 바로 승인할 수 있는 형식의 보고서"
역할과 도구

누가, 어떻게 일할지 정합니다.

Q3. 팀장(에이전트)의 역할은?
에이전트 (Agent)
전체를 총괄하고 최종 판단을 내리는 AI
예: 보고서 전체 구성 결정
Q4. 팀원(서브에이전트)은?
서브에이전트 (Sub-Agent)
각 분야를 담당하는 전문가 AI
예: 커밋 분석, 이슈 분석 담당
Q5. 공통 도구(슬래시 명령)는?
슬래시 명령 (Slash Command)
한 줄로 전체 워크플로우를 호출하는 단축 명령
예: /weekly-report

자동 저장됩니다. 다음에 다시 와도 그대로 보입니다.

TIP

5가지 질문을 다 채울 필요는 없습니다. Q1, Q3만 정해도 첫 자동화를 시작할 수 있습니다.

STEP 2

기본 구조 패턴

AI에게 일을 맡기는 4가지 방법

자세히 보기 →
패턴 1 SOLO
단독으로 수행하기
나 → A
하나의 AI가 처음부터 끝까지 혼자 처리합니다.

예시: 이메일 초안 작성, 문서 요약
패턴 2 PIPELINE
순차적 업무
나 → A → B → C
한 에이전트가 끝나면 다음 에이전트가 시작합니다.

예시: 데이터 수집 → 분석 → 보고서 작성
패턴 3 TEAM
팀으로 나눠서
나 → [A, B, C] → 결과
여러 에이전트가 동시에 작업합니다.

예시: 매출, 고객, 재고 데이터를 동시 분석
패턴 4 ORCHESTRA
지휘자 세우기
나 → Main ⇄ [A, B, C]
메인 에이전트가 서브에이전트를 관리하고 조율합니다.

예시: Claude Code의 기본 작업 방식
TIP

처음에는 패턴 1(Solo)부터 시작하세요. 익숙해지면 패턴 2, 3, 4로 확장합니다.

STEP 3

도구 선택 가이드

"이걸 에이전트로 할까, 슬래시 명령으로 할까?" 헷갈릴 때 보는 결정 기준

슬래시 명령
한 줄 명령으로 자주 부르고 싶을 때. 사용자가 직접 호출하는 단축키 같은 역할입니다.
/weekly-report 매주 보고서 시작
스킬 (Skill)
특정 상황에서 Claude가 알아서 활성화할 도구가 필요할 때. SKILL.md에 트리거 조건을 적어두면 자동으로 켜집니다.
엑셀 파일을 열었을 때 자동 활성
서브에이전트
한 작업을 여러 전문 분야로 쪼개고 싶을 때. 메인 에이전트가 분야별로 일을 나눠 맡깁니다.
보안 전문가 + 성능 전문가 동시 검토
후크 (Hook)
"무슨 일이 일어나면 자동으로" 만들고 싶을 때. 사용자가 부르지 않아도 이벤트가 트리거합니다.
파일 저장 시 자동 포맷팅
결정 흐름
내가 직접 명령어를 칠 건가? → 예: 슬래시 명령
Claude가 상황 보고 알아서 켜야 하나? → 예: 스킬
일을 분야별로 쪼개야 하나? → 예: 서브에이전트
특정 이벤트(저장/완료/시작)에 자동 실행돼야 하나? → 예: 후크
TIP

네 가지가 서로 배타적이지 않습니다. 보통 슬래시 명령으로 시작 → 메인 에이전트가 서브에이전트 호출 → 후크가 마무리. 한 자동화에 다 들어갑니다.

STEP 4

실전 시나리오

실제 업무에 바로 적용할 수 있는 3가지 자동화 레시피

📌
용어 정리. 아래 표에서 /이름 형태는 슬래시 명령(사용자가 직접 호출)이고, 스킬은 SKILL.md로 자동 활성화되는 도구입니다. 후크는 PreToolUse, PostToolUse, Stop 같은 이벤트에 걸려 자동 실행됩니다.
시나리오 1 매주 월요일 주간보고 자동화
매주 반복되는 주간보고서 작성. 지난주 작업 내역을 자동으로 수집하고, 정해진 양식으로 정리합니다.
역할담당하는 일
에이전트보고서 총괄전체 구성 결정, 최종 문서 생성
서브에이전트 1활동 수집지난주 git 커밋·캘린더·이메일에서 활동 요약
서브에이전트 2이슈 분석완료/진행 중 이슈 정리
슬래시 명령/weekly-report한 줄로 전체 프로세스 시작
후크Stop 완료 알림보고서 생성 끝나면 Slack 알림 발송
복사해서 바로 쓰는 프롬프트
매주 월요일마다 실행할 수 있는 주간보고 자동화를 만들어줘. - /weekly-report 슬래시 명령으로 시작 - 지난주 git 커밋·캘린더·이메일에서 활동을 자동 수집 - 완료된 이슈와 진행 중인 이슈를 분류 - 주간 보고서 양식(날짜, 완료 항목, 진행 중, 다음 주 계획)으로 정리 - reports/ 폴더에 markdown으로 저장

→ Claude가 슬래시 명령 파일과 보고서 템플릿을 자동으로 만들어 줍니다

시나리오 2 회의록 → 액션아이템 자동 분배
팀 회의 녹취록이나 메모를 받으면, 결정사항·액션아이템·담당자·마감일을 자동으로 추출해 정리합니다. 비개발 직군에 가장 잘 맞는 자동화입니다.
역할담당하는 일
에이전트회의록 총괄회의 흐름 파악, 최종 요약본 작성
서브에이전트 1액션아이템 추출"누가/무엇을/언제까지" 형태로 분리
서브에이전트 2결정사항 정리합의된 결정과 보류 항목 구분
슬래시 명령/meeting-notes회의록 파일 경로 받아 시작
후크Stop 자동 분류완료 시 담당자별 폴더로 액션아이템 자동 분배
복사해서 바로 쓰는 프롬프트
회의록을 액션아이템으로 분해하는 자동화를 만들어줘. - /meeting-notes 슬래시 명령으로 시작 - $ARGUMENTS로 회의록 파일(.txt, .md, .docx)을 받음 - 결정사항, 액션아이템(담당자/마감일), 보류항목을 분리 - 액션아이템은 actions/{담당자명}.md에 누적 추가 - 회의 요약본은 meetings/{날짜}.md로 저장

→ /meeting-notes 2026-04-26-주간회의.txt 한 줄로 모든 정리 끝

시나리오 3 데이터 정리 자동화
엑셀이나 CSV 파일을 받으면 자동으로 정리하고, 요약 보고서를 만듭니다.
역할담당하는 일
에이전트분석 총괄데이터 구조 파악, 분석 방향 결정
서브에이전트 1데이터 정제중복 제거, 형식 통일, 빈 값 처리
서브에이전트 2통계 분석핵심 지표 계산, 트렌드 파악
슬래시 명령/data-clean파일명 지정해서 정리 시작
후크PostToolUse 결과 저장분석 도구 호출 후 자동으로 결과 파일 저장
복사해서 바로 쓰는 프롬프트
데이터 정리를 자동화하는 슬래시 명령을 만들어줘. - /data-clean 으로 시작 - $ARGUMENTS로 CSV나 Excel 파일 경로를 받아서 - 중복 행 제거, 빈 셀 처리, 날짜 형식 통일 - 핵심 통계(합계, 평균, 최대, 최소)를 자동 계산 - 정리된 파일 + 요약 보고서를 같은 폴더에 저장

→ /data-clean sales_2024.csv 로 바로 실행 가능

STEP 5

워크플로우 조합

4가지 도구를 연결하면 완전 자동화가 됩니다

에이전트 + 서브에이전트 + 스킬 + 후크. 이 4가지를 조합하면 "명령 하나로 시작부터 끝까지 자동 실행"되는 워크플로우를 만들 수 있습니다. 마치 공장의 자동화 라인처럼요.

1
스킬로 시작
/project:feature 명령어 한 줄로 전체 프로세스 시작
2
에이전트가 계획
어떻게 만들지 전략을 먼저 세움
3
서브에이전트가 동시 작업
여러 전문가가 동시에 분석하고 구현
4
후크가 자동 검사
파일이 수정될 때마다 자동으로 품질 검사
5
후크가 알림
모든 작업이 끝나면 자동으로 결과 알림

설계 원칙

멀티 에이전트를 성공적으로 운영하기 위한 4가지 원칙

🌱
작은 것부터 시작
에이전트 1개에서 2개로, 2개에서 조합으로. 한 번에 복잡한 구조를 만들지 마세요.
🎯
역할을 명확히 분리
각 에이전트에 하나의 명확한 역할을 부여하세요. 역할이 겹치면 혼란이 생깁니다.
🛡️
실패에 대비
한 에이전트가 실패해도 전체가 멈추지 않도록 설계하세요.
👤
사람이 감독
최종 확인과 중요한 결정은 항상 사람이 합니다.
STEP 6

프롬프트 모음

Claude Code에 복사해서 바로 쓸 수 있는 자동화 프롬프트

스킬 생성 요청

매주 월요일마다 실행할 수 있는 스킬을 만들어줘. - /weekly-report 라고 치면 실행되게 - 이번 주 git 커밋 내역을 자동으로 읽어서 - 주간 보고서 양식으로 정리해줘 - reports/ 폴더에 저장해줘

후크 설정 요청

파일을 저장할 때마다 자동으로 실행되는 후크를 만들어줘. - .js 파일이 수정되면 ESLint 자동 실행 - 에러가 있으면 바로 알려줘 - 에러가 없으면 "통과"라고만 표시

멀티 에이전트 설계 요청

아래 업무를 자동화하는 멀티 에이전트 구조를 설계해줘. 업무: [여기에 자동화하고 싶은 업무를 설명] 조건: - 에이전트(팀장), 서브에이전트(팀원), 스킬(도구), 후크(알람) 역할을 나눠줘 - 각 역할이 하는 일을 구체적으로 설명해줘 - 실행할 수 있는 스킬 파일도 함께 만들어줘
STEP 7

운영 점검 (ROI)

자동화를 만들었으면 끝이 아닙니다. "정말 효과가 있는지" 2주마다 점검하세요

Q1
시간이 실제로 줄었는가?
자동화 전 vs 후 소요 시간을 측정합니다. 주 1시간 이상 절감되지 않으면 단순화하거나 폐기합니다.
Q2
결과물 품질은 유지되는가?
처음 4회 결과물을 사람이 검수합니다. 3회 이상 손볼 필요가 없어야 운영 안정 단계에 들어갑니다.
Q3
실패할 때 어떻게 알 수 있는가?
후크의 Stop 이벤트로 결과 알림을 받습니다. 알림이 오지 않으면 조용한 실패입니다.
Q4
사람이 빠져도 되는 단계인가?
중요한 결정(외부 발송, 비용 집행, 인사)은 사람 승인 단계를 명시적으로 남겨 둡니다. 자동화의 합격선은 "감독 가능한 자동화"입니다.
TIP

2주 후에도 Q1~Q4를 통과하면 그 자동화는 "운영 안정"입니다. 통과 못 하면 더 단순한 형태로 다시 설계합니다.
