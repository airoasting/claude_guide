STEP 1

설계 템플릿

5가지 질문에 답하면, 나만의 AI 팀이 설계됩니다

목적과 기준

무엇을, 왜 자동화하고 싶은지 먼저 정리합니다.

Q1. 나는 왜 이것을 하고 싶은가?

직접 작성해보세요

예: "매주 3시간 걸리는 주간보고를 30분으로 줄이고 싶다"
Q2. 좋은 결과물의 기준은 무엇인가?

직접 작성해보세요

예: "팀장이 바로 승인할 수 있는 형식의 보고서"
역할과 도구

누가, 어떻게 일할지 정합니다.

Q3. 팀장(에이전트)의 역할은?
에이전트 (Agent)
전체를 총괄하고 최종 판단을 내리는 AI
예: 보고서 전체 구성 결정
Q4. 팀원(서브에이전트)은?
서브에이전트 (Sub-Agent)
각 분야를 담당하는 전문가 AI
예: 커밋 분석, 이슈 분석 담당
Q5. 공통 도구(스킬)는?
스킬 (Skill)
모든 팀원이 사용하는 공유 도구
예: /weekly-report 명령어
TIP

5가지 질문을 다 채울 필요는 없습니다. Q1, Q3만 정해도 첫 자동화를 시작할 수 있습니다.

STEP 2

기본 구조 패턴

AI에게 일을 맡기는 4가지 방법

자세히 보기 →
패턴 1 SOLO
단독으로 수행하기
나 → A
하나의 AI가 처음부터 끝까지 혼자 처리합니다.

예시: 이메일 초안 작성, 문서 요약
패턴 2 PIPELINE
순차적 업무
나 → A → B → C
한 에이전트가 끝나면 다음 에이전트가 시작합니다.

예시: 데이터 수집 → 분석 → 보고서 작성
패턴 3 TEAM
팀으로 나눠서
나 → [A, B, C] → 결과
여러 에이전트가 동시에 작업합니다.

예시: 매출, 고객, 재고 데이터를 동시 분석
패턴 4 ORCHESTRA
지휘자 세우기
나 → Main ⇄ [A, B, C]
메인 에이전트가 서브에이전트를 관리하고 조율합니다.

예시: Claude Code의 기본 작업 방식
TIP

처음에는 패턴 1(Solo)부터 시작하세요. 익숙해지면 패턴 2, 3, 4로 확장합니다.

STEP 3

실전 시나리오

실제 업무에 바로 적용할 수 있는 3가지 자동화 레시피

시나리오 1 매주 월요일 주간보고 자동화
매주 반복되는 주간보고서 작성. 지난주 작업 내역을 자동으로 수집하고, 정해진 양식으로 정리합니다.
역할담당하는 일
에이전트보고서 총괄전체 구성 결정, 최종 문서 생성
서브에이전트 1커밋 분석지난주 git 커밋 내역 수집 및 요약
서브에이전트 2이슈 분석완료/진행 중 이슈 정리
스킬/weekly-report한 줄 명령어로 전체 프로세스 시작
후크완료 알림보고서 생성 후 Slack 알림
복사해서 바로 쓰는 프롬프트
매주 월요일마다 실행할 수 있는 주간보고 자동화 스킬을 만들어줘. - /weekly-report 라고 치면 실행되게 - 지난주 git 커밋 내역을 자동으로 읽어서 요약 - 완료된 이슈와 진행 중인 이슈를 분류 - 주간 보고서 양식(날짜, 완료 항목, 진행 중, 다음 주 계획)으로 정리 - reports/ 폴더에 markdown으로 저장

→ Claude가 스킬 파일과 보고서 템플릿을 자동으로 만들어 줍니다

시나리오 2 코드 리뷰 자동화
PR(Pull Request)이 올라오면 자동으로 코드를 분석하고, 개선 사항을 제안합니다.
역할담당하는 일
에이전트리뷰 총괄전체 코드 품질 판단, 최종 리뷰 작성
서브에이전트 1보안 검사보안 취약점 탐색
서브에이전트 2성능 검사성능 이슈 분석
스킬/code-review대상 파일 지정해서 리뷰 시작
후크자동 포맷팅파일 저장 시 코드 스타일 자동 정리
복사해서 바로 쓰는 프롬프트
코드 리뷰를 자동화하는 스킬을 만들어줘. - /code-review 라고 치면 실행 - $ARGUMENTS로 대상 파일이나 폴더를 받아서 - 보안 취약점, 성능 이슈, 코드 스타일을 동시에 검사 - 결과를 심각도(높음/중간/낮음)로 분류해서 보여줘 - 개선 제안도 함께 포함

→ /code-review src/login.js 로 바로 실행 가능

시나리오 3 데이터 정리 자동화
엑셀이나 CSV 파일을 받으면 자동으로 정리하고, 요약 보고서를 만듭니다.
역할담당하는 일
에이전트분석 총괄데이터 구조 파악, 분석 방향 결정
서브에이전트 1데이터 정제중복 제거, 형식 통일, 빈 값 처리
서브에이전트 2통계 분석핵심 지표 계산, 트렌드 파악
스킬/data-clean파일명 지정해서 정리 시작
후크결과 저장분석 완료 시 자동으로 결과 파일 저장
복사해서 바로 쓰는 프롬프트
데이터 정리를 자동화하는 스킬을 만들어줘. - /data-clean 이라고 치면 실행 - $ARGUMENTS로 CSV나 Excel 파일 경로를 받아서 - 중복 행 제거, 빈 셀 처리, 날짜 형식 통일 - 핵심 통계(합계, 평균, 최대, 최소)를 자동 계산 - 정리된 파일 + 요약 보고서를 같은 폴더에 저장

→ /data-clean sales_2024.csv 로 바로 실행 가능

STEP 4

워크플로우 조합

4가지 도구를 연결하면 완전 자동화가 됩니다

에이전트 + 서브에이전트 + 스킬 + 후크. 이 4가지를 조합하면 "명령 하나로 시작부터 끝까지 자동 실행"되는 워크플로우를 만들 수 있습니다. 마치 공장의 자동화 라인처럼요.

1
스킬로 시작
/project:feature 명령어 한 줄로 전체 프로세스 시작
2
에이전트가 계획
어떻게 만들지 전략을 먼저 세움
3
서브에이전트가 동시 작업
여러 전문가가 동시에 분석하고 구현
4
후크가 자동 검사
파일이 수정될 때마다 자동으로 품질 검사
5
후크가 알림
모든 작업이 끝나면 자동으로 결과 알림

설계 원칙

멀티 에이전트를 성공적으로 운영하기 위한 4가지 원칙

🌱
작은 것부터 시작
에이전트 1개에서 2개로, 2개에서 조합으로. 한 번에 복잡한 구조를 만들지 마세요.
🎯
역할을 명확히 분리
각 에이전트에 하나의 명확한 역할을 부여하세요. 역할이 겹치면 혼란이 생깁니다.
🛡️
실패에 대비
한 에이전트가 실패해도 전체가 멈추지 않도록 설계하세요.
👤
사람이 감독
최종 확인과 중요한 결정은 항상 사람이 합니다.
STEP 5

프롬프트 모음

Claude Code에 복사해서 바로 쓸 수 있는 자동화 프롬프트

스킬 생성 요청

멀티 에이전트 설계 요청

아래 업무를 자동화하는 멀티 에이전트 구조를 설계해줘. 업무: [여기에 자동화하고 싶은 업무를 설명] 조건: - 에이전트(팀장), 서브에이전트(팀원), 스킬(도구), 후크(알람) 역할을 나눠줘 - 각 역할이 하는 일을 구체적으로 설명해줘 - 실행할 수 있는 스킬 파일도 함께 만들어줘
