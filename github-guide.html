비개발자를 위한 단계별 안내

Git이 뭔가요?

Git은 파일의 변경 이력을 기록하고 관리하는 도구입니다. 구글 독스에서 "버전 기록"을 보면 누가, 언제, 뭘 바꿨는지 볼 수 있죠? Git이 바로 그 역할을 합니다. 다만 코드 파일에 특화되어 있고, 훨씬 더 강력합니다.

저장소

Repository (레포)

프로젝트의 모든 파일과 변경 기록이 담긴 폴더입니다.

비유: 프로젝트 전용 서류함

커밋

Commit

현재 상태를 "이 시점의 버전"으로 저장하는 것입니다.

비유: 문서의 "다른 이름으로 저장"

브랜치

Branch

원본에 영향 주지 않고 따로 작업할 수 있는 공간입니다.

비유: 원본 문서를 복사해서 수정하기

머지

Merge

따로 작업한 내용을 원본에 합치는 것입니다.

비유: 수정본을 원본에 반영하기

GitHub

공간

쉽게 말해, Git이 일기장이라면 GitHub은 구글 드라이브입니다. Git으로 내 컴퓨터에서 버전 관리를 하고, GitHub에 올려서 백업하거나 다른 사람과 함께 작업할 수 있습니다.

GitHub에서 할 수 있는 것

  • 코드를 안전하게 백업 (컴퓨터가 고장나도 OK)
  • 팀원과 함께 코드 작업
  • 변경 사항을 검토하고 승인 (코드 리뷰)
  • 누가 뭘 바꿨는지 전체 기록 확인

GitHub 시작하기

1

GitHub 가입하기

github.com에 접속해서 Sign up 버튼을 클릭합니다. 이메일, 비밀번호, 사용자명을 입력하면 끝입니다.

2

새 저장소 만들기

로그인 후 오른쪽 상단 + 버튼 → New repository를 클릭합니다.

# 저장소 설정 Repository name: my-project ← 프로젝트 이름 입력 Public / Private 선택 ← 공개 or 비공개 [Create repository] 클릭

※ Private(비공개)으로 만들면 나만 볼 수 있습니다.

3

Claude Code에서 연결하기

이미 내 컴퓨터에 프로젝트가 있다면:

# Claude에게 이렇게 말하세요 > 이 프로젝트를 GitHub에 올려줘 > GitHub 주소는 https://github.com/내아이디/my-project 야

GitHub에 있는 프로젝트를 내려받고 싶다면:

# Claude에게 이렇게 말하세요 > https://github.com/내아이디/my-project 클론해줘

※ Claude가 알아서 git 명령어를 실행해줍니다. 명령어를 외울 필요 없습니다.

Claude Code와 함께 쓰기

명령어를 외울 필요 없습니다. 한국어로 말하면 Claude가 알아서 실행합니다.
하고 싶은 것Claude에게 이렇게 말하세요
변경사항 저장하기"지금까지 바꾼 거 커밋해줘"
GitHub에 올리기"GitHub에 푸시해줘"
최신 코드 받기"GitHub에서 최신 코드 받아줘"
변경 내역 보기"뭐가 바뀌었는지 보여줘"
이전 버전으로 돌리기"마지막 커밋 전으로 되돌려줘"
새 브랜치 만들기"새 브랜치 만들어줘, 이름은 feature/login"
브랜치 합치기"이 브랜치를 main에 머지해줘"

Claude Code를 쓰면 git 명령어를 직접 입력할 일이 거의 없습니다. "커밋해줘", "푸시해줘" 같은 자연스러운 한국어로 지시하면, Claude가 적절한 git 명령어를 자동으로 실행합니다.

꼭 알아야 할 용어

용어뜻쉽게 말하면
push (푸시)내 컴퓨터 → GitHub으로 올리기구글 드라이브에 업로드
pull (풀)GitHub → 내 컴퓨터로 받기구글 드라이브에서 다운로드
clone (클론)GitHub 저장소를 통째로 복사공유 폴더를 내 컴퓨터에 복사
PR (풀 리퀘스트)코드 변경을 검토 요청"이거 확인해주세요" 요청서
fork (포크)남의 저장소를 내 계정으로 복사남의 레시피를 내 노트에 복사
conflict (충돌)같은 부분을 다르게 수정했을 때두 사람이 같은 문장을 다르게 고침
.gitignoreGit이 무시할 파일 목록"이 파일들은 추적하지 마" 목록
main / master기본 브랜치 (원본)원본 문서
originGitHub에 있는 원격 저장소구글 드라이브 속 원본 폴더
