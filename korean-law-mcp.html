INTRO

이 실습에서 만들 것

Claude Desktop에 한국 법령 검색 MCP를 커넥터로 붙여, 법령과 판례, 조례를 대화 중에 바로 찾아오게 만듭니다

⚖️

실행 환경은 Claude Desktop입니다. 별도 설치 없이 Customize 메뉴에서 커넥터 URL 하나만 입력하면 끝납니다. 법제처 Open API의 OC 키 한 줄을 발급받고, 그 키를 붙인 URL을 Claude Desktop에 등록하면 9개의 법령 검색 도구가 대화 안에서 동작합니다.

한국 법령과 판례를 ChatGPT나 Claude에 물어보면 그럴듯한 조문을 지어내는 경우가 자주 있습니다. 존재하지 않는 조항 번호, 잘못된 항·호 인용, 폐지된 옛 조문을 현행처럼 답하는 식입니다. 한국 법령 검색 MCP는 법제처 공식 Open API 42개를 묶어 9개의 도구로 노출해, 모델이 답하기 전에 실제 법령 데이터베이스를 조회하도록 만듭니다.

완성하면 흐름은 이렇게 됩니다.

STEP 1 · 법제처 Open API 가입과 OC 키 발급

open.law.go.kr에서 1분 안에 키 받기

법제처가 운영하는 공식 Open API에 가입하면 OC라는 짧은 키 한 줄을 줍니다. 이 키가 MCP 커넥터의 신분증입니다

한국 법령 MCP는 법제처 Open API를 그대로 호출합니다. 누구나 무료로 가입할 수 있고, 신청 즉시 OC 키가 발급됩니다. OC는 보통 본인 이메일 아이디(@ 앞부분)와 같은 영문 짧은 문자열입니다.

🔗 법제처 Open API 공식 사이트 open.law.go.kr
1
법제처 Open API 회원가입
open.law.go.kr에 접속해 우측 상단의 회원가입을 누릅니다. 본인 인증과 이메일 인증을 거치면 됩니다.
2
활용 신청
로그인 후 OPEN API > 활용 신청 메뉴에서 활용 목적을 간단히 적고 신청합니다. 개인 학습이나 업무 활용이라고 적으면 즉시 승인됩니다.
3
OC 키 확인
마이페이지 > 활용 현황으로 들어가면 본인 OC 값이 나옵니다. 보통 이메일 아이디와 같은 영문 문자열입니다. 이 값을 메모해 두세요. STEP 3에서 URL에 붙입니다.
참고

OC 키는 비밀번호가 아니지만 외부에 그대로 노출하면 다른 사람이 본인 한도를 같이 소모할 수 있습니다. GitHub 같은 공개 저장소에 OC가 박힌 URL을 올리지 않도록 주의합니다.

STEP 2 · Claude Desktop에서 Customize 메뉴 열기

왼쪽 사이드바의 Customize 메뉴 누르기

Claude Desktop은 왼쪽 사이드바에 커스텀 커넥터 전용 진입점을 두고 있습니다

커스텀 MCP를 붙이려면 Claude Desktop 최신 버전이 필요합니다. 무료 플랜에서도 커스텀 커넥터 추가는 가능합니다. 사내 정책으로 막아둔 환경이라면 관리자에게 문의해야 할 수 있습니다.

1
Claude Desktop 열기
앱을 실행하면 화면 왼쪽 사이드바에 대화 목록과 함께 메뉴 항목들이 보입니다. 사이드바가 접혀 있다면 좌상단의 메뉴 아이콘을 눌러 펼칩니다.
2
왼쪽 메뉴에서 Customize 클릭
왼쪽 사이드바 메뉴 중 Customize 항목을 누릅니다. 누르면 커스터마이즈 화면이 열리고, 그 안에 Connectors(커넥터) 영역이 보입니다.
3
Connectors 영역에서 추가 버튼 찾기
Customize 화면 안 Connectors 섹션에 기존 연결 목록이 보입니다. 그 아래 또는 우측의 Add custom connector(커스텀 커넥터 추가) 버튼을 눌러 STEP 3으로 넘어갑니다.
📌 진입 경로 요약

왼쪽 사이드바 Customize 클릭 → 화면 안의 Connectors 영역으로 이동 → Add custom connector 버튼 누르기. 이 순서만 기억하면 됩니다. 버전에 따라 메뉴 이름이 커스터마이즈, 커넥터로 한글화돼 있을 수 있지만 위치는 같습니다.

STEP 3 · 커넥터에 한국 법령 MCP URL 등록

URL 한 줄을 붙여 넣고 저장

레포지토리 운영자가 호스팅 중인 공개 엔드포인트가 있어, 별도 서버를 띄울 필요가 없습니다

커스텀 커넥터 추가 화면이 열리면 이름과 URL을 입력하는 칸이 보입니다. 이름은 알아보기 쉽게 적고, URL에는 다음 형식을 그대로 넣습니다.

https://korean-law-mcp.fly.dev/mcp?oc=YOUR_OC_KEY

맨 뒤 YOUR_OC_KEY 부분만 STEP 1에서 받은 본인 OC 값으로 바꿉니다. 예를 들어 OC가 honggildong이라면 최종 URL은 https://korean-law-mcp.fly.dev/mcp?oc=honggildong이 됩니다.

1
Name 입력
알아보기 쉬운 이름을 적습니다. 예: Korean-Law. 이 이름이 채팅창 도구 목록에 그대로 표시됩니다.
2
URL 입력
위 URL을 그대로 붙여 넣고 YOUR_OC_KEY를 본인 OC 값으로 교체합니다. 프로토콜은 반드시 https이어야 합니다.
3
Add 또는 저장
Add 또는 저장을 누르면 연결 시도를 합니다. 몇 초 안에 도구 목록(search_law, get_law_text, legal_analysis 등)이 화면에 펼쳐지면 성공입니다.
로컬로 띄우고 싶다면

회사 정책 때문에 외부 호스팅을 못 쓰는 경우, 같은 레포지토리를 npm install -g korean-law-mcp로 직접 설치한 뒤 로컬 stdio 방식으로 등록할 수 있습니다. 이 경우 Customize 화면 대신 claude_desktop_config.json을 편집합니다. 자세한 절차는 레포지토리 README의 Installation 섹션을 참고합니다.

STEP 4 · 첫 검색으로 연결 확인

대화창에서 자연어로 한 번 물어보기

연결이 잘 됐는지 가장 빠르게 확인하는 방법은 평소처럼 한국어로 질문하는 것입니다

Settings를 닫고 새 대화를 열어 다음 질문을 그대로 붙여 넣습니다.

근로기준법 제60조 연차유급휴가 조문 본문을 그대로 보여줘. 출처는 법제처 데이터로 검증해서.

정상적으로 연결돼 있으면 Claude가 응답 도중에 search_law, get_law_text 같은 도구를 호출했다는 안내가 뜨고, 그 결과로 실제 조문이 인용됩니다. 도구를 사용하지 않고 답이 바로 나오면 커넥터가 비활성화 상태일 수 있습니다.

📌 동작 확인 체크리스트
  • 응답 안에 "도구를 사용했습니다" 또는 Tool use 표시가 보이는가
  • 조문 본문에 제1항, 제2항 같은 항 번호가 정확히 붙어 있는가
  • 마지막에 "이어서 할 수 있는 조회" 같은 추가 검색 안내가 따라오는가

세 개 모두 보이면 연결이 끝났습니다. 하나라도 빠지면 트러블슈팅 섹션을 봅니다.

간단한 추가 테스트 질문을 몇 개 더 준비해 뒀습니다. 본인이 자주 다루는 분야를 골라 바로 던져 봅니다.

STEP 5 · 실전 활용. 인용 검증, 영향 그래프, 단계별 안내

이 MCP가 특별한 이유

단순 검색을 넘어서, AI가 법률 답변을 만들 때 가장 자주 사고를 치는 지점을 막아주는 핵심 도구는 둘입니다. 정밀분석(legal_analysis)과 다단계 리서치(legal_research). 아래 세 가지가 그 대표 활용입니다

✅ 인용 검증. AI 답변의 환각 잡기

ChatGPT나 Claude가 작성한 법률 답변을 그대로 붙여 넣고 사실 여부만 검증할 수 있습니다. 정밀분석 도구 legal_analysis의 verify_citations 모드가 조문 인용을 법제처 DB와 교차 확인합니다.

다음 답변에 등장하는 모든 조문 인용이 실제로 존재하는지 법제처 데이터로 검증해줘. "민법 제750조에 따라 손해배상을 청구할 수 있고, 근로기준법 제60조 제1항은 연차유급휴가를 규정하며, 상법 제401조의2 제7항은 이사 책임을 정한다."

실존 조항은 ✓, 존재하지 않는 항 번호는 ✗로 표시되고, 어디까지 실제 조항이 있는지(예: "최대 제2항까지 존재")까지 알려줍니다. 법률 AI 답변을 다른 사람에게 전달하기 전에 마지막 안전망으로 쓰면 좋습니다.

🗺️ 영향 그래프. 조문 한 줄의 파급효과

어떤 조문 하나를 인용한 판례, 헌재 결정, 자치법규, 행정심판을 한 번에 끌어와 그래프로 그립니다. 같은 legal_analysis 도구의 impact_map 모드입니다.

민법 제103조를 인용한 대법원 판례와 헌재 결정을 그래프로 그려줘.

결과 끝에 mermaid 코드가 붙어 나오므로, claude.ai에서 그대로 시각화해 볼 수 있습니다. 법 개정이나 조례 영향 분석을 할 때 한 번에 전체 영향 범위를 잡아낼 수 있습니다.

📋 단계별 안내. 평소 말투를 5단계 실행안으로

법률 용어를 모르는 상태에서 평소 말투로 던져도, 관련 법령과 절차를 5단계로 정리해줍니다. 다단계 리서치 도구 legal_research가 여러 법제처 API를 엮어 답을 구성합니다.

전세금 못 받았어. 어떻게 해야 하는지 단계별로 정리해줘.

결과는 STEP 1 상황 진단(주택임대차보호법 자동 식별) STEP 2 권리와 구제수단(판례) STEP 3 신청 기관과 기한(행정규칙) STEP 4 필요 서류와 양식 STEP 5 함정과 주의(시효, 법률구조공단)로 나옵니다. 비법조인이 바로 행동할 수 있는 형태로 변환됩니다.

활용 팁

도구 이름이나 모드를 굳이 외울 필요는 없습니다. "이 답변 환각 검증해줘", "이 조문 파급효과 그려줘", "내 상황에서 뭐부터 해야 하는지 단계로 정리해줘"처럼 자연어로 던지면 Claude가 알아서 legal_analysis·legal_research 중 맞는 도구와 모드를 골라 호출합니다.

FAQ · 자주 막히는 지점

연결이 안 될 때 점검 순서

에러 메시지별 원인과 해결 방법을 정리했습니다

Q1. Customize 또는 Connectors 메뉴가 보이지 않습니다

원인 후보:

  • Claude Desktop 버전이 낮음. 메뉴에서 Claude > Check for Updates로 최신 버전을 받습니다.
  • 개발자 옵션 비활성. Settings > Advanced에서 Enable custom connectors 또는 Developer mode를 켭니다.
  • 관리자 계정으로 제한된 환경. 회사 SSO 계정이라면 관리자에게 커스텀 커넥터 허용을 요청해야 할 수 있습니다.
Q2. 사용자 정보 검증에 실패하였습니다라는 응답이 나옵니다

원인: URL의 oc= 뒤에 붙은 OC 키가 잘못됐거나, 법제처에서 활용 신청 승인이 안 끝났습니다.

해결:

  • open.law.go.kr에 로그인해 마이페이지에서 정확한 OC 값을 다시 복사합니다.
  • 활용 현황 페이지에서 신청 상태가 승인인지 확인합니다. 대기면 잠시 기다립니다.
  • OC 키에 공백이나 줄바꿈이 섞이지 않았는지 다시 봅니다.
Q3. 커넥터는 추가됐는데 Claude가 도구를 호출하지 않습니다

원인 후보:

  • 도구 활성화 토글이 꺼져 있음. 새 대화 좌하단의 도구 아이콘을 눌러 Korean Law MCP가 켜져 있는지 확인합니다.
  • 질문이 너무 일반적임. "근로기준법 알려줘"보다 "근로기준법 제60조 조문 보여줘"처럼 구체적인 법령명과 조항을 짚어주는 게 도구 호출 확률이 올라갑니다.
  • 모델 선택 문제. Sonnet 또는 Opus 계열에서 안정적으로 동작합니다. Haiku는 도구 호출 빈도가 낮을 수 있습니다.
Q4. fetch failed 또는 EXTERNAL_API_ERROR가 뜹니다

원인: 호스팅 서버가 일시적으로 법제처 API에 접근하지 못하는 상태입니다. 법제처 API의 점검 시간이거나, 봇 차단 정책 변경 때문일 수 있습니다.

해결:

  • 5분 뒤 다시 시도합니다. 대부분 자동 복구됩니다.
  • 지속되면 레포지토리 Issues에 환경을 적어 보고합니다.
  • 긴급하게 써야 하면 npm install -g korean-law-mcp로 로컬에 직접 설치해 stdio 방식으로 돌립니다.
Q5. 일일 호출 한도를 넘었습니다

원인: 법제처 Open API는 OC 키당 일일 호출 한도가 있습니다. 한 질문에 여러 도구가 연쇄 호출되면 한도를 빠르게 소진할 수 있습니다.

해결:

  • 한국 시간 자정에 한도가 초기화됩니다. 다음 날 다시 사용합니다.
  • 한도를 자주 넘긴다면 마이페이지에서 한도 증액을 신청합니다.
  • 같은 OC 키를 여러 사람이 공유하지 않습니다. 키마다 한도가 따로 잡힙니다.
