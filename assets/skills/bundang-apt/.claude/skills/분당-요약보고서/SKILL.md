# 분당 요약보고서 스킬

## 목적

국토부 실거래가 CSV 한 묶음을 받아, 의사결정에 바로 쓸 수 있는 **분당구 아파트 매매 요약보고서 PDF**를 한 번에 만든다.
사람이 매번 표를 다시 그리지 않도록 분석과 PDF 조판을 `scripts/`의 파이썬 스크립트로 묶어 둔 것이 이 스킬의 핵심이다.
숫자 계산과 PDF 렌더링은 모두 스크립트가 담당하고, 이 문서는 "언제·어떤 순서로·어떤 결과를 기대하며" 그 스크립트를 호출하는지를 정의한다.

PDF가 아니라 인터랙티브 HTML 대시보드가 필요하면 `scripts/dashboard.py`, Gmail 주간 메일 초안이 필요하면 `scripts/weekly_report.py`가 별도로 있으니 그쪽을 쓴다. 이 스킬은 PDF 보고서만 책임진다.

## 사전 확인

실행 전 두 가지를 먼저 점검한다. 둘 중 하나라도 빠지면 스크립트가 중단되므로 미리 사용자에게 알린다.

1. `data/` 폴더에 `아파트(매매)_실거래가_*.csv`가 최소 1개 있는가. (rt.molit.go.kr에서 분당구 매매 실거래를 받아 넣는다)
   `data/` 폴더 자체가 없으면 스크립트가 폴더를 자동으로 만들고, CSV를 어디에 넣을지 안내한 뒤 중단한다. 안내대로 CSV를 넣고 다시 실행하면 된다.
2. 한글 PDF 폰트는 **OS에 맞게 자동 탐지**된다. macOS는 AppleGothic, Windows는 맑은 고딕, Linux는 나눔/Noto를 자동으로 찾으므로 보통은 손댈 필요가 없다. 자동 탐지에 실패하면 명확한 설치 안내와 함께 중단한다(폰트 처리 절 참고).

## 실행 순서

### 1. 의존성 확인
`reportlab`과 `pandas`가 필요하다. 둘 다 `requirements.txt`에 포함돼 있으므로 아래 한 줄이면 갖춰진다.

```bash
pip install -r requirements.txt
```

(개별 설치하려면 `pip install reportlab pandas`)

### 2. 보고서 생성 (엔트리포인트 1개)
`scripts/pdf_report.py`만 실행하면 분석과 PDF 생성이 한 번에 끝난다. 이 스크립트가 내부에서 `analyze.run()`을 호출한다.

```bash
python scripts/pdf_report.py
```

선택 인자(기본값으로도 동작한다):

```bash
python scripts/pdf_report.py --data data/ --output output/ --min-deals 5 --top-n 5
```

- `--data`: CSV 폴더 (기본 `data/`)
- `--output`: PDF 저장 폴더 (기본 `output/`, 없으면 자동 생성)
- `--min-deals`: 주목할 단지 선정 시 제외할 최소 거래건수 (기본 5, 소량 거래 이상치 방지)
- `--top-n`: 주목할 단지 TOP N (기본 5)

### 3. 결과 확인
`output/분당 아파트 매매_요약보고서_YYYYMMDD.pdf`가 생겼는지 확인하고, 경로를 사용자에게 알린다.
스크립트가 콘솔에 분석 요약(동별 변동률·전년 동기비 등)도 출력하므로, 사용자가 PDF를 열기 전에 핵심 수치를 함께 전달하면 좋다.

## 분석 로직 (스크립트가 수행하는 계산)

세부 수식·점수 가중치·기간 정의는 `references/analysis-spec.md`에 정리해 두었다. 요약은 다음과 같다.

- **입력**: `data/`의 매칭 CSV를 **모두 읽어 합산**한다(최신 한 파일만 고르지 않는다). 상단 15행 안내문은 `skiprows=15`로 건너뛰고, euc-kr·cp949·utf-8-sig 순으로 인코딩을 시도한다. "해제" 거래와 필수값 결측 행은 제외한다.
- **평당가**: `거래금액(만원) / (전용면적(㎡) / 3.3058)`
- **변동률**: 전반기(2025.06.15~2025.12.13) 평균 평당가 대비 후반기(2025.12.14~2026.06.14) 평균 평당가
- **이 달의 거래**: 동 평균 변동률과의 편차(절댓값)가 가장 큰 단지를 동별로 선정
- **단지 종합점수(100점)**: 입지(40) + 평형/구조(20) + 학군(15) + 재건축(10) + 단지규모(10) + 동·층·조망(5). `scores_override.json`에 값이 있으면 해당 단지 점수를 그 값으로 덮어쓴다.
- **주목할 단지**: `점수당가격 = 평당가 / 종합점수` 최저 TOP N (낮을수록 저평가)

## 출력 — 보고서 구성

`scripts/pdf_report.py`는 다음 순서의 PDF 섹션을 생성한다. 표현은 스크립트에 고정되어 있으니, 구성을 바꾸려면 스크립트의 해당 섹션 빌더를 수정한다.

1. **다크 헤더 배너** — 분당구 전체 평균 변동률과 최고·최저 동을 한 줄로 압축
2. **KPI 요약 카드(3칸)** — 총 거래건수 / 평균 1평당가 / 최고 상승 동
3. **동별 거래 동향** — 줄글 요약 + 수평 바차트, 변동률 내림차순
4. **이 달의 주목 거래 TOP 5** — 동 평균 대비 편차 최대 단지, 변동률·편차(%p) 표기
5. **주목할 단지 TOP 5** — 점수당가격(저평가지수) 최저 순, 점수·평당가 포함
6. **매매 관점 인사이트** — 강세 권역·조정 동향·저평가 포인트·대출 기준 요약
7. **고지 사항** — 국토부 실거래가 기반 자동 분석, 추가 검토 권장

## 톤 원칙

보고서 문장(인사이트 단락 등)을 손보거나 보강할 때 지킨다.

- **두괄식**: 핵심 결론을 먼저, 근거를 뒤에.
- **가독성 우선**: 변동률은 소수점 1자리, 큰 금액은 천 단위 구분.
- **근거 기반**: 실거래 수치를 명시하고, 추정치는 "약"으로 표기.
- **과도한 단정 지양**: "상승 추세로 전환"보다 "후반기 평당가가 상승했다"처럼 데이터가 말하는 범위만 쓴다.

## PDF 한글 폰트 처리

`pdf_report.py`는 한글 폰트를 임베딩하며, **OS별로 적절한 폰트를 자동 탐지**한다. 자동 탐지에도 폰트를 못 찾으면 **대체 폰트를 쓰지 않고 명시적으로 중단**한다(깨진 글자가 든 PDF가 나오는 것을 막기 위함).

### 자동 탐지 (기본 동작 — 보통 손댈 필요 없음)
스크립트가 현재 OS에 맞춰 아래 순서로 한글 폰트를 찾는다.

- **macOS**: `/System/Library/Fonts/Supplemental/AppleGothic.ttf`(기본 설치) → 나눔고딕(`/Library/Fonts`, `~/Library/Fonts`)
- **Windows**: `C:\Windows\Fonts\malgun.ttf`/`malgunbd.ttf`(맑은 고딕) → 나눔고딕
- **Linux**: 나눔고딕(`/usr/share/fonts/truetype/nanum/...`) → Noto Sans CJK → `fc-list :lang=ko`로 설치된 한글 폰트 자동 탐색

Bold용 폰트가 따로 없으면 Regular 폰트를 굵은 글씨에도 그대로 쓴다. 실행 시 콘솔에 `✓ 한글 폰트: <파일명>`이 출력되므로 어떤 폰트가 잡혔는지 확인할 수 있다.

### 수동 지정 (override — 특정 폰트를 강제할 때)
다른 한글 폰트를 강제하고 싶으면 `pdf_report.py` 상단의 `FONT_REG`·`FONT_BOLD`(기본 `None`)에 `.ttf` **절대경로**를 직접 넣는다. 값이 채워져 있으면 자동 탐지를 건너뛰고 그 폰트를 그대로 쓴다.

```python
# pdf_report.py 상단
FONT_REG  = "/Users/me/Library/Fonts/NanumGothic.ttf"
FONT_BOLD = "/Users/me/Library/Fonts/NanumGothicBold.ttf"  # 별도 Bold가 없으면 Regular와 동일 경로
```

- **`.ttc`(폰트 컬렉션) 처리**: `AppleSDGothicNeo.ttc`처럼 `.ttc`를 지정하면 스크립트가 `subfontIndex=0`으로 첫 폰트를 읽는다. 로드에 실패하면 "단일 `.ttf` 경로를 쓰라"는 명확한 안내와 함께 중단한다. 가장 안전한 선택은 여전히 **단일 `.ttf`**다.
- **Linux**: 폰트가 전혀 없으면 `sudo apt install fonts-nanum` 후 재실행하면 자동 탐지가 잡는다.

## 오류 처리 / 중단 조건

| 상황 | 조치 |
|---|---|
| `data/` 폴더 자체가 없음 | 폴더를 **자동 생성**하고 "여기에 CSV를 넣고 다시 실행" 안내 후 중단 |
| `data/`에 매칭 CSV 없음 | "data/ 폴더에 아파트(매매)_실거래가_*.csv를 넣어주세요." 안내 후 중단 |
| 한글 폰트 자동 탐지 실패 | OS별 설치 안내(예: Linux `apt install fonts-nanum`)와 함께 중단. 수동 지정 안내 |
| `.ttc` 폰트 지정 시 | `subfontIndex=0`으로 첫 폰트 로드. 실패하면 "단일 .ttf 사용" 안내 후 중단 |
| `reportlab` 미설치 | `pip install -r requirements.txt` 안내 후 재시도 |
| CSV 인코딩 인식 실패 | 해당 파일 건너뛰고 경고 출력, 나머지로 계속 진행 |
| 필터 후 유효 거래 0건 (전부 '해제'·결측 등) | "분석할 유효 거래가 없습니다" 명확히 중단 (빈 PDF 생성 안 함) |
| 분석 기간 양쪽에 거래 있는 동이 0개 | "변동률을 계산할 거래가 없습니다" 명확히 중단 |
| 동/단지 거래건수 < `--min-deals` | 주목할 단지 후보에서만 제외, 다른 분석은 계속 진행 |
| 전반기·후반기 중 한쪽 거래가 없는 단지 | 변동률 계산에서 자동 제외(dropna) |
| `output/` 폴더 없음 | 자동 생성 후 계속 진행 |

## 자가 점검

작업을 마치기 전 확인한다.

- PDF가 `output/`에 생성되었고 파일명에 오늘 날짜(YYYYMMDD)가 들어갔는가.
- 헤더 배너·KPI 카드·동별 바차트·TOP5 표 2종·인사이트·고지가 모두 보이는가.
- 한글이 깨지지 않았는가(폰트 임베딩 성공).
- 콘솔에 출력된 동별 변동률 수치와 PDF의 수치가 일치하는가.

## 사용 예시

- 사용자: "분당 보고서 만들어줘" → 사전 확인 2가지 점검 → `python scripts/pdf_report.py` → 생성된 PDF 경로와 핵심 수치 전달.
- 사용자: "저평가 단지 10개까지 보고 싶어" → `python scripts/pdf_report.py --top-n 10`.
- 사용자(맥/리눅스 사용): "폰트 오류 나는데" → 보통은 OS 자동 탐지가 처리한다. 그래도 안 잡히면 한글 `.ttf`를 설치(예: Linux `sudo apt install fonts-nanum`)하거나, `pdf_report.py`의 `FONT_REG`/`FONT_BOLD`에 단일 `.ttf` 절대경로를 직접 지정 후 재실행. (.ttc보다 단일 .ttf 권장)

## 관련 파일

- `scripts/pdf_report.py` — **이 스킬의 실행 엔트리포인트** (PDF 생성, 내부에서 analyze.run 호출)
- `scripts/analyze.py` — 핵심 분석 파이프라인. `run(data_dir, min_deals, top_n)` → dict 반환
- `scripts/dashboard.py` — 인터랙티브 HTML 대시보드 (PDF 대신 웹 결과물이 필요할 때)
- `scripts/weekly_report.py` — Gmail 주간 메일 초안 생성 (참고용)
- `scores_override.json` — 단지별 종합점수 수동 보정값 (`"행정동_단지명": {"종합점수": N}`)
- `references/analysis-spec.md` — 변동률·점수·기간 등 분석 세부 명세
- `examples/` — 생성물 예시(PDF·대시보드 HTML·미리보기 이미지)
- `output/` — PDF 저장 폴더
