```

[3] DATA_CONST    — 별자리 정의 (선 연결, 이름, 설명)

[6] DATA_MAP      — 한반도 SVG 패스
[7] DATA_STARDUST — 별가루 사전 생성 좌표 (RA/Dec 2500개, 빌드 타임 생성)
[8] ASTRO         — 천문 좌표 변환 (순수 함수)
[9] PLANETS       — 행성 위치 계산 (케플러)
[10] RENDERER     — Canvas 렌더링 (메인 + 오프스크린 trailCanvas)
[11] AUDIO        — 앰비언트 사운드 (Web Audio API)
[12] UI           — 사용자 입력 처리
[13] APP          — 초기화, 오프닝 시퀀스 상태 머신, 메인 루프
```

### ASTRO 모듈 함수 시그니처
```javascript
dateToJD(year, month, day, utHours) → number           // 율리우스 일
jdToGMST(jd) → number                                   // 그리니치 항성시 (°)
gmstToLST(gmst, longitude) → number                     // 지방 항성시 (°)
equatorialToHorizontal(ra, dec, lst, latitude) → {alt, az}  // 적도→지평
horizontalToScreen(alt, az, viewAlt, viewAz, fov, cx, cy, scale) → {x, y, visible}  // 지평→화면
```

## 첫 로드 경험: 오프닝 시퀀스

시연 시 첫 5초가 전체 인상을 결정한다. 별이 "만들어지는" 순간을 보여준다.

### 타임라인
```
[0ms]       검정 화면 중앙에 "🌌" 이모지 표시 (60px, opacity pulse 0.3↔0.7, 1초 주기)
            — 로딩이 아닌 "무언가 시작되는" 느낌
[200ms]     "🌌" 이모지 fade-out (200ms)
            별가루(미세 점)가 먼지처럼 서서히 나타남 (opacity 0→0.3, 1초간)
[600ms]     밝은 별부터 하나씩 "톡, 톡" 나타남:
            - 1등급 이상 밝은 별 ~20개가 0.8초에 걸쳐 순차 등장
            - 각 별: scale(0→1.3→1.0) + opacity(0→1), 개별 딜레이 30~50ms 간격
            - 나타나는 순간 글로우가 확 퍼졌다 수축 (반경 30px→정상)
[1.4s]      나머지 별 전체가 한 번에 페이드인 (opacity 0→1, 0.5초)
            앰비언트 사운드 fade-in 시작 (0→목표볼륨, 2초간)
[1.9s]      별자리 선이 별에서 별로 그어짐:
            - 주요 별자리 5~8개가 동시에 드로잉 시작
            - 각 선: 시작별→끝별로 0.3초간 길이가 늘어남 (stroke-dashoffset 애니메이션)
            - 선이 완성되면 별자리 이름이 fade-in (0.3초)
[3.0s]      헤더 "📍 [도시명]의 밤하늘" 페이드인
            컨트롤 바 하단에서 슬라이드업 (translateY(20px)→0, ease-out 400ms)
[3.5s]      오프닝 완료. 일반 인터랙션 모드 진입.
            별 반짝임 시작.
```

### 오프닝 제어
- **스킵:** 화면 아무 곳 클릭 또는 아무 키 입력 시 즉시 완료 상태로 점프
- **오프닝 중 입력 처리:** 마우스 드래그, 휠, 키보드 단축키 등 모든 인터랙션은 무시. 클릭/키 입력은 오직 스킵으로만 처리. 오프닝 완료 후 일반 인터랙션 활성화.
- **반복 시연:** 키보드 `I`(Intro) 키로 오프닝 재실행 가능
- **IP API:** 오프닝과 병렬 실행. 오프닝 중 응답 오면 조용히 반영. 3초 타임아웃, 실패 시 삼성역 기본값.

### 구현 가이드
- 오프닝 상태를 `introPhase` 변수로 관리 (0: 이모지, 1: 별가루, 2: 밝은별, 3: 전체별, 4: 별자리, 5: UI등장, 6: COMPLETE)
- 각 단계의 시작 시각을 `introStartTime + offset`으로 계산
- requestAnimationFrame 루프 안에서 현재 시각에 따라 분기
- 스킵 시 `introPhase = COMPLETE`로 설정, 모든 요소 즉시 최종 상태, 사운드 즉시 목표 볼륨
- **별가루 좌표는 DATA_STARDUST에 사전 생성 데이터로 내장** (런타임 랜덤 생성 아님). 초기화 시 화면 좌표 변환만 수행. 2500개 × 삼각함수 4회 ≈ 10000회 연산, 50ms 이내 완료 보장.

**등급별 시각 표현:**

**글로우 렌더링 (Canvas 2D):**
```

2. 코어: fillArc, 반지름 [값]px, 별 색상 100%
3. 하이라이트: fillArc, 1px, #ffffff (0등급 이상만)
```

**별 색상 (스펙트럼 타입):**


**별 반짝임:**
- 대상: 2등급 이상
- `alpha = base * (1 + amp * sin(time * freq + phase))`
- 각 별: freq 0.3~0.8Hz, phase 0~2π 랜덤, amplitude ±15%
- requestAnimationFrame 기반 상시 실행

### 별가루 레이어 (Stardust)

밤하늘의 "깊이"를 만드는 배경 레이어. 육안으로는 보이지 않는 5~7등급 별을 미세한 점으로 표현.

- **개수:** 2500개 (DATA_STARDUST에 사전 내장)
- **데이터:** RA/Dec 좌표를 코드에 직접 내장 (런타임 랜덤 생성 아님, 매 실행 동일한 하늘)
- **초기화 성능:** 2500개 × 좌표 변환(삼각함수 4회) ≈ 10000회 연산. 50ms 이내 완료.
- **크기:** 0.5~1px, 균일
- **색상:** `rgba(180, 200, 240, 0.08~0.15)` — 거의 안 보이지만 밀집되면 밤하늘 질감이 됨
- **반짝임:** 없음 (정적 점)
- **성능:** 화면 좌표는 시간/위치 변경 시에만 재계산, 매 프레임은 캐시된 좌표로 drawPoint만
- **지평선:** 고도 0° 이하 제외 (별과 동일)
- **광해 연동:** 광해 단계에 따라 별가루 표시 비율 변경 (도심: 0%, 완전 암흑: 100%)

### 별자리

**최소 30개 필수 포함:**
큰곰(북두칠성), 작은곰(북극성), 카시오페이아, 오리온, 사자, 전갈, 백조, 거문고, 독수리, 페가수스, 안드로메다, 쌍둥이, 황소, 처녀, 물병, 물고기, 양, 게, 궁수, 염소, 천칭, 목동, 왕관, 용, 큰개, 작은개, 마차부, 페르세우스, 세페우스, 까마귀

**데이터 형식:**
```javascript
{ abbr, name, nameEn, lines: [[starId1, starId2], ...], myth }
```

**시각:**
- 선: `rgba(100, 149, 237, 0.35)`, 1px
- 호버: `rgba(100, 149, 237, 0.7)`, 1.5px, 소속 별 글로우 증가
- 이름: 한글, 12px, weight 300, `rgba(180, 200, 240, 0.6)`, 중심 배치
- 클릭 히트 영역: 볼록 껍질(convex hull) + 패딩 10px

**별자리 선 드로잉 애니메이션:**
- 트리거: 오프닝 시퀀스, 또는 별자리 선 토글 OFF→ON 시
- 방식: 각 선분이 시작별→끝별로 길이가 늘어남
  - Canvas: 시작점에서 끝점까지 `t` 비율(0→1)로 보간, 300ms/선분
  - 같은 별자리 내 선분은 50ms 간격으로 순차 시작 (이전 선 완료 안 기다림)
  - 여러 별자리는 동시에 드로잉 시작
- 선이 완성된 후 별자리 이름 fade-in (opacity 0→0.6, 200ms)
- 토글 ON→OFF 시: 역순 애니메이션 없이 즉시 사라짐 (빠른 토글 대응)
- **빠른 토글 방지:** OFF→ON 드로잉 애니메이션 진행 중 다시 OFF→ON 시, 진행 중인 애니메이션 즉시 취소(cancel) → 새 애니메이션 처음부터 시작. 동시에 두 개의 드로잉 애니메이션이 실행되지 않도록 보장.

**별 호버 효과:**
- 대상: 2등급 이상 밝은 별
- 마우스가 별 중심 기준 15px 이내 접근 시:
  - 글로우 반경 1.5배 확대 (200ms ease-out)
  - 별 이름 라벨 fade-in (opacity 0→0.9, 150ms), 별 우측 상단 8px 오프셋
  - 영문명 + 한글명 (예: "Sirius 시리우스"), 11px
- 마우스 이탈 시: 역순으로 원복 (150ms)

### 좌표 변환

**파이프라인:**
```
RA, Dec, lat, lng, Date
  → JD → GMST → LST → HA → (Alt, Az) → (screenX, screenY, visible)
```

**남반구 처리:**
- 위도(lat)가 음수일 때 좌표 변환 공식은 동일하게 작동 (sin/cos가 자동 처리)
- 단, 기본 뷰 방향을 변경: 남반구에서는 기본 뷰를 **북쪽(Az=0°)**으로 (하늘이 뒤집혀 보이는 효과)
- 천구 회전 방향: 남반구에서는 별이 동→서가 아닌 시계 반대 방향으로 회전하는 것처럼 보임 → 이건 좌표 변환이 자동 처리하므로 별도 로직 불필요. 단, 시점 드래그 방향 감각은 그대로 유지.
- 검증: 시드니에서 북극성이 보이지 않고, 남십자성(적위 -60°)이 남쪽 높이 보여야 함

**투영:** 등거리 방위 투영 (azimuthal equidistant). 천정=화면 중심, 지평선=원 경계.

**배경 그라데이션과 시점 회전:**
- 배경 그라데이션(천정 어두움 → 지평선 밝음)은 **화면 고정** (시점과 무관)
- 이유: 시점이 천정을 향하면 화면 전체가 어두워지고, 지평선을 향하면 하단이 밝아지는 게 자연스러움
- 구현: 화면 중심에서 외곽으로 radialGradient, 시점의 고도(viewAlt)에 따라 중심 밝기 조절

**검증:**
- 북극성: 방위 ≈ 0°, 고도 ≈ 37.5° (±0.5°)
- 1시간 경과: ~15° (±0.5°) 이동

### 지평선과 방위

- 고도 < 0°: 렌더링 제외
- 고도 0°~5°: alpha = alt/5 선형 감쇠
- 지평선 원: `rgba(60, 80, 120, 0.3)`, 1px
- 방위 라벨: N E S W, 지평선 원 외곽
- 기본 뷰: 남쪽(Az=180°), 고도 45° (남반구: 북쪽 Az=0°)

### 시점 조작

- **드래그:** 1px = 0.3° 회전
- **휠:** FOV 30°~150°, 기본 90°, 1틱 = ±5°
- **더블클릭:** 해당 지점을 중심으로 이동 (ease-out 300ms)

## P1: 인터랙션

### 시간 컨트롤

- 슬라이더: 당일 18:00 ~ 익일 06:00 KST
- 시각 표시: "2026년 3월 2일 22:30 KST", 슬라이더 상단
- 재생 ▶/⏸: 클릭 토글
- 속도: ×1 → ×5 → ×30 → ×60 (클릭 순환)
- 매 프레임 좌표 재계산, 보간 없음

### Star Trail (별 궤적) — 타임랩스의 와우 포인트

장노출 사진처럼 별이 호를 그리며 흘러가는 궤적을 남긴다.

**트리거:** 자동 재생(▶) 중에만 활성화 (토글 컨트롤 바 버튼)
**기본:** off (재생 시작해도 자동 켜지지 않음, 수동 토글)

**구현:**
- **별도 오프스크린 Canvas** (trailCanvas)를 메인 Canvas 뒤에 레이어
- 재생 중 매 프레임: trailCanvas에 현재 별 위치를 매우 낮은 alpha(0.03~0.05)로 점 찍기
- trailCanvas는 클리어하지 않음 → 시간이 지나면 호가 누적
- 밝은 별(2등급 이상)만 trail 대상 (성능 + 시각적 선명도)
- trail 색상: 별 고유 색상, alpha 0.03
- trail 비활성화(토글 OFF) 시: trailCanvas를 fade-out(0.5초) 후 클리어
- 재생 정지 시: trail 유지 (정지 상태에서 궤적 감상 가능)
- 시간 슬라이더 수동 조작 시: trail 클리어 (새 시점에서 다시 시작)

**시각 효과:**
- 북극성 근처 별: 짧은 동심원 호 (거의 안 움직임) → 극축 회전 시각화
- 적도 근처 별: 긴 호 → 하늘의 회전 속도 체감
- ×60 속도로 5분(실시간) 재생 시 약 5시간 분량 = 75° 회전 → 선명한 호 형성

**성능:**
- 오프스크린 Canvas는 메인 렌더링에 영향 없음
- trail 대상 별 ~50개 × 60fps = 프레임당 50개 점 → 부담 없음

**시점 드래그와의 상호작용:**
- Trail은 **화면 좌표 기준**으로 찍힘 (하늘 좌표 아님)
- 시점 드래그 시: trail이 같은 화면 위치에 남아있으므로 "하늘이 흘러간 자국"처럼 보임 → 자연스러움
- 시점 드래그 중에는 trail 누적 일시 정지 (드래그 중 찍으면 궤적이 깨짐)
- 드래그 종료 후 trail 누적 재개

### 광해 슬라이더

| 단계 | 라벨 | 한계 등급 | 배경 밝기 | 글로우 강도 | 별가루 |

| 2 | 교외 | 3.0 | +8% | ×0.7 | 25% |
| 3 | 시골 | 3.8 | 기본 | ×1.0 | 50% |
| 4 | 산간 | 4.2 | -5% | ×1.2 | 80% |
| 5 | 완전 암흑 | 4.5 | -10% | ×1.5 | 100% |

- 기본값: 3 (시골) — 별이 적당히 많이 보이는 상태
- UI: 5단계 스텝 슬라이더, 각 단계 라벨

**광해 변경 시 별 등장/퇴장 애니메이션:**
- 광해를 낮출 때 (도심→암흑 방향): 새로 보이게 되는 별들이 오프닝과 동일한 "톡" 애니메이션으로 등장
  - scale(0→1.2→1.0) + opacity(0→1), 200ms/별
  - 동시에 전부가 아닌, 30~50ms 간격으로 순차 등장 (별이 "쏟아지는" 느낌)
  - 별가루도 해당 비율까지 fade-in (opacity 0→목표, 500ms)
- 광해를 높일 때 (암흑→도심 방향): 사라지는 별은 fade-out (opacity→0, 300ms), 순차 아님 동시
- **시연 킬링 포인트:** "도심"에서 "완전 암흑"으로 한 번에 바꾸면 수백 개의 별이 쏟아지며 나타남

### 유성 효과

- 토글 (기본 off)
- 빈도: 10~30초 랜덤 간격
- **생성 위치: 현재 화면 뷰포트 내에서만 생성** (화면 밖 유성 낭비 방지)
- 시작점: 화면 내 랜덤 (고도 30°~80° 범위 내)
- 궤적: 15~25° 직선, 대체로 지평선 방향, ±30° 편차
- 시각: 두께 2→0px 테이퍼 + 꼬리 페이드 500ms

- 우하단: "AI 에이전트를 고용하라 · 아르고스AI Lab", 12px, alpha 0.35, 글래스 없음

### 앰비언트 사운드

은은한 배경음으로 밤하늘 몰입감을 극대화한다.

**구현: Web Audio API (OscillatorNode + 노이즈)**
- 외부 오디오 파일 없음. 코드로 생성하는 프로시저럴 사운드.
- **바람 소리:** 브라운 노이즈 기반, 저주파 필터(200Hz cutoff), 볼륨 미세 변동(LFO 0.1Hz)
- **귀뚜라미:** 고주파 사인파(4000~5000Hz) 짧은 버스트, 2~5초 간격 랜덤, 좌우 패닝 랜덤
- **전체 볼륨:** 매우 낮음 (마스터 게인 0.03~0.05). 대화를 방해하지 않는 수준.

**제어:**
- 토글: 컨트롤 바 "🔊" 버튼 (기본: on, 오프닝에서 자동 fade-in)
- 키보드: `A` (Audio) 토글
- 오프닝 시: 별 전체 등장(1.4s) 시점부터 2초간 fade-in (0→목표볼륨)
- 오프닝 스킵 시: 즉시 목표 볼륨
- **시연 주의:** 발표장 마이크와 간섭 가능. 기본 ON이지만 한 번 탭으로 즉시 OFF 가능해야 함.

**성능:** Web Audio API는 별도 오디오 스레드에서 실행, 렌더링 성능 영향 없음.

### 별자리 포커스 모드

별자리를 더블클릭하면 해당 별자리로 줌인하며 주변이 어두워지는 프레젠테이션 모드.

**트리거:** 별자리 영역 더블클릭 (단일 클릭은 기존 정보 패널)

**전환 애니메이션 (600ms ease-out):**
```
1. 시점이 해당 별자리 중심으로 부드럽게 이동
2. FOV가 현재값 → 별자리를 화면 60%로 채우는 값으로 줌인
3. 해당 별자리 외 영역: 오버레이 dimming (rgba(0,0,0,0.6)) 서서히 적용
4. 해당 별자리 선: alpha 0.35→0.9, 두께 1→2.5px
5. 소속 별: 글로우 ×2 확대
6. 각 별 옆에 이름 라벨 fade-in (영문 + 한글, 12px)
7. 별자리 이름: 상단에 크게 표시 (24px, weight 300, 글래스 배경)
```

**포커스 해제:**
- ESC 키 또는 dimming 영역 클릭
- 역순 애니메이션 (400ms ease-in)으로 원래 시점/FOV 복귀
- dimming fade-out, 별 크기 원복

**용도:** 시연 시 특정 별자리를 설명할 때 "이게 오리온자리입니다" 하면서 더블클릭 → 프레젠테이션처럼 깔끔하게 보여줌

## 구현 순서 — 단계별 분할 구현 (필수)

⚠️ **단일 HTML 파일을 한 번에 작성하지 마라.** 출력 토큰 제한에 걸린다.
아래 순서대로 파일을 생성하고, 단계마다 병합한다.

### Phase 1: 뼈대 + 데이터
```
1-1. constellation.html 생성 — HTML 구조 + CSS + 빈 Canvas + 글래스모피즘 UI 껍데기만
1-2. data_stars.js 생성 — 별 카탈로그 500개 배열 (DATA_STARS + DATA_STARDUST)
1-3. data_constellations.js 생성 — 별자리 30개 정의 (DATA_CONST)
1-4. data_planets.js 생성 — 행성 궤도 요소 (DATA_PLANETS)
1-5. data_map.js 생성 — 한반도 SVG 패스 + 도시 룩업 (DATA_MAP + DATA_CITIES)
→ constellation.html에서 <script src="..."> 로 로드
```

### Phase 2: 핵심 엔진 (P0)
```
2-1. astro.js 생성 — 좌표 변환 함수 5개 (ASTRO 모듈)
2-2. renderer.js 생성 — 별/별자리/지평선 렌더링 (RENDERER 모듈)
2-3. app.js 생성 — 초기화 + 메인 루프 + 시점 조작 (APP 모듈)
→ 이 시점에서 기본 밤하늘이 화면에 표시되어야 함
→ 5인 페르소나 평가
```

### Phase 3: 인터랙션 (P1)
```
3-1. ui.js 생성 — 시간 슬라이더, 토글, 정보 패널 (UI 모듈)
3-2. renderer.js 수정 — 은하수, 별자리 드로잉 애니메이션, 별 호버, Star Trail 추가
3-3. app.js 수정 — UI 이벤트 연결
→ 5인 페르소나 평가
```

### Phase 4: 확장 (P2)
```
4-1. app.js 수정 — IP 위치 감지, 날짜 변경
4-2. ui.js 수정 — 한국 지도 미니맵, 장소 전환 애니메이션
4-3. planets.js 생성 — 행성 위치 계산 (PLANETS 모듈)
4-4. renderer.js 수정 — 행성 렌더링 추가
→ 5인 페르소나 평가
```

### Phase 5: 프리미엄 마감 (P3)
```
5-1. audio.js 생성 — 앰비언트 사운드 (AUDIO 모듈)
5-2. renderer.js 수정 — 광해 별 등장 애니메이션, 유성, 포커스 모드
5-3. app.js 수정 — 오프닝 시퀀스, 키보드 단축키 전체
5-4. constellation.html 수정 — 워크샵 브랜딩
→ 5인 페르소나 최종 평가
```

### Phase 6: 단일 파일 병합
```
6-1. 모든 .js 파일 내용을 constellation.html의 <script> 태그 안에 순서대로 인라인
6-2. 외부 .js 파일 참조 제거
6-3. 최종 단일 HTML 파일 출력: constellation_final.html
6-4. 동작 검증
```

### 파일 구조 (Phase 1~5 작업 중)
```
/project
├── constellation.html      ← 메인 HTML (CSS + UI 구조)
├── data_stars.js            ← 별 500개 + 별가루 2500개
├── data_constellations.js   ← 별자리 30개
├── data_planets.js          ← 행성 궤도 요소
├── data_map.js              ← 한반도 SVG + 도시 20개
├── astro.js                 ← 좌표 변환
├── planets.js               ← 행성 위치 계산
├── renderer.js              ← Canvas 렌더링
├── audio.js                 ← 앰비언트 사운드
├── ui.js                    ← 사용자 입력
├── app.js                   ← 초기화 + 메인 루프

```

### 핵심 규칙

- Phase 2 완료 시점에서 브라우저로 constellation.html 열어 동작 확인
- Phase 6에서 병합할 때 모듈 순서: CONFIG → DATA_* → ASTRO → PLANETS → RENDERER → AUDIO → UI → APP
