STEP 1 · Claude Cowork에 Google Workspace 연결

Cowork의 Google Workspace 커넥터 활성화

Claude Cowork는 Google Drive · Sheets · Calendar · Gmail을 별도 API 없이 한 번에 연결합니다

Claude Cowork에는 Google Workspace 커넥터가 기본으로 준비되어 있습니다. 사용자의 구글 계정을 한 번만 인증하면 Cowork 세션 안에서 시트의 셀 데이터를 바로 읽을 수 있습니다. 별도의 API 키나 서비스 계정은 필요하지 않습니다.

1
Claude Desktop 실행 후 Cowork 선택
Claude Desktop(macOS/Windows 데스크톱 앱)을 실행하고, 새 대화의 모델/도구 선택 영역에서 Cowork를 켭니다.
2
커넥터 설정 열기
Claude Desktop의 Customize > Connectors로 들어가면 사용 가능한 커넥터 목록이 보입니다. 거기에서 Google Workspace 항목을 찾습니다.
3
Google 계정 인증
Connect 버튼을 누르면 구글 OAuth 화면이 열립니다. 분석할 시트가 있는 계정으로 로그인하고, Drive · Sheets 읽기 권한을 승인합니다.
4
연결 확인
커넥터 옆에 Connected 표시가 뜨면 완료입니다. 이제 Cowork 대화창에서 내 드라이브의 시트 URL을 그대로 붙여넣고 분석을 요청할 수 있습니다.
TIP · 회사 계정과 개인 계정 분리

회사 워크스페이스 계정과 개인 구글 계정을 함께 쓰는 경우, Cowork 커넥터는 한 번에 한 계정만 인증할 수 있습니다. 분석 대상 시트가 있는 쪽 계정으로 연결하세요. 계정을 바꿀 때는 Disconnect를 누른 다음 다시 Connect하는 방식이 가장 안전합니다.

STEP 2 · 예제 시트를 내 드라이브로 복사

예제 시트를 내 드라이브에 복사하기

Cowork 커넥터는 내 드라이브 안의 파일을 읽습니다. 따라서 공용 예제 시트를 먼저 내 드라이브로 복사합니다

Cowork 커넥터가 읽을 수 있는 시트는 커넥터에 연결한 구글 계정의 드라이브에 들어 있는 파일입니다. 공용 예제 시트를 그대로 쓰면 권한 문제가 생기기 쉽기 때문에, 먼저 내 드라이브로 사본을 만들어 둡니다. 사본을 한 번 만들어 두면 다음부터는 그 시트만 갱신하면 됩니다.

1
예제 시트 열기
아래 링크로 예제 시트를 브라우저에서 엽니다. 이 시트는 누구나 뷰어로 접근할 수 있도록 열어 두었습니다.
2
사본 만들어 내 드라이브에 저장
상단 메뉴에서 파일 > 사본 만들기를 선택합니다. 저장 위치는 내 드라이브(혹은 원하는 폴더)로 지정하고, 파일 이름은 기본값을 그대로 둬도 좋습니다.
3
사본의 URL 복사
사본이 열리면 주소창의 URL을 복사합니다. gid=... 부분이 붙어 있으면 그대로 두세요. 특정 탭을 가리키는 값입니다.
4
계정 일치 확인
사본을 만든 계정과 STEP 1에서 Cowork에 연결한 구글 계정이 같아야 합니다. 다르면 Cowork가 시트를 찾지 못합니다.
실습용 예제 시트

아래 시트를 열고 파일 > 사본 만들기로 내 드라이브에 옮기세요. STEP 3부터는 사본의 URL을 씁니다.

왜 내 드라이브로 복사부터 하나

Cowork가 "시트에 접근할 수 없습니다"라고 답하는 가장 흔한 이유가 권한 누락입니다. 내 드라이브의 사본은 처음부터 내 권한으로 만들어지기 때문에 권한 문제가 생기지 않습니다. 그 다음부터는 자연어 한 줄로 작업이 끝납니다.

STEP 3 · 탐색적 분석 (EDA) 실행

한 줄 프롬프트로 EDA 시작하기

복잡한 지시는 필요하지 않습니다. 시트 주소와 한 문장만으로 충분합니다

EDA는 거창한 절차가 아닙니다. Cowork 대화창에 내 드라이브의 시트 주소를 붙여 넣고, "이 데이터를 살펴봐 줘" 정도로만 부탁해도 Claude가 알아서 컬럼을 정리하고 결측치와 이상치, 핵심 지표를 찾아 줍니다.

EDA 프롬프트

아래처럼 한 줄로 던지면 됩니다. 시트 URL은 STEP 2에서 복사한 내 드라이브 사본의 주소를 씁니다.

아래 구글 시트를 EDA로 분석해줘. https://docs.google.com/spreadsheets/d/내-시트-ID/edit 어떤 데이터가 들어 있는지, 흥미로운 패턴이 뭔지 편하게 알려줘.
결과로 받게 되는 것

한 줄 요청만으로도 다음이 한 번에 정리됩니다.
· 데이터 개요. 행 수와 컬럼 수, 각 컬럼의 의미와 단위.
· 데이터 품질. 결측치와 이상치 위치.
· 핵심 KPI. 총 매출, 영업이익률, 매장별 순위 같은 요약 지표.
· 인사이트. 사람이 놓치기 쉬운 패턴 3~5가지.

STEP 4 · 뉴모피즘 5탭 인터랙티브 대시보드 생성

대시보드 한 장으로 압축하기

EDA에서 잡아낸 인사이트를 뉴모피즘 라이트 디자인 · 5개 탭의 단일 HTML 파일로 패키징합니다

차트를 따로따로 만들어 두면 보는 사람이 흐름을 잃습니다. 모든 인사이트를 한 페이지 안에 모으고 탭과 필터로 좁혀 가며 보는 인터랙티브 대시보드로 만들 때 의사결정 속도가 크게 빨라집니다. 디자인은 뉴모피즘(라이트 톤 배경에 부드러운 그림자로 카드가 떠 있는 듯한 소프트 UI)을 씁니다. 정보 밀도가 높아도 눈이 덜 피로하고, 표·차트·KPI 카드가 서로 같은 톤으로 묶입니다.

설계 원칙 · 5개 탭으로 시야 나누기

경영 대시보드는 한 화면에 모든 걸 우겨넣기보다 주제별 5개 탭으로 나눠 두는 편이 빠릅니다.
1. 종합 개요 · 핵심 KPI 6개 + 월별 매출/이익 + 카테고리 믹스 + 매장 랭킹.
2. 매장별 실적 · 매장 칩 토글, 매장별 월 매출 라인, 이익률 막대, 상위 3개 매장 레이더.
3. P&L 분석 · 월별 비용 누적 바, 비용 구성 도넛, 영업이익률 추이.
4. 채널 & 플랫폼 · 매장별 홀/배달/포장 믹스, 배민·쿠팡이츠·요기요 점유, 월별 플랫폼 트렌드.
5. 품질 & 고객 · QSC·리뷰 추이, 재방문율, 매장별 품질 테이블.

대시보드 생성 프롬프트

EDA를 마친 직후 같은 세션에서 이 한 줄을 던집니다. 디자인 톤과 인터랙션 방식 정도만 정해 주면 나머지는 Claude가 알아서 채워 줍니다.

방금 분석한 데이터로 EDA로 탐색 가능한 인터랙티브 대시보드를 단일 HTML 한 파일로 만들어줘. - 뉴모피즘 라이트 디자인 (소프트 그림자) - 연도 셀렉터로 기간 좁히기 + 매장 칩 토글 - 데이터는 Google Sheets CSV(gviz)에서 직접 fetch - 차트는 Chart.js 사용 - 연도 셀렉터·헤더의 기간 표시·"최신: YYYY-MM" 배지는 데이터에서 자동 생성 (시트에 새 달이 추가되면 코드 수정 없이 그대로 반영되도록) - 기본값은 가장 최근 연도
왜 Google Sheets에서 직접 CSV로 읽나
  • 중간 JSON 파일이 필요 없습니다. 시트만 갱신하면 대시보드가 같이 갱신됩니다.
  • 로컬에 데이터 사본을 두지 않아 "시트 따로, 대시보드 따로"가 어긋날 일이 없습니다.
  • 엔드포인트는 https://docs.google.com/spreadsheets/d/{ID}/gviz/tq?tqx=out:csv&gid={GID}. CORS가 열려 있어 GitHub Pages 등 정적 호스팅에서 바로 fetch됩니다.
  • 시트가 "링크가 있는 모든 사용자(보기)"로 공개돼 있어야 합니다. 비공개면 401이 떨어집니다.
  • 한국어 헤더 → 내부 키 매핑을 코드에 한 번만 정의해 두면, 컬럼 순서가 바뀌어도 동작합니다.
뉴모피즘이 경영 대시보드에 맞는 이유

경영자가 매일 보는 화면은 화려함보다 안정감이 중요합니다. 뉴모피즘은 라이트 톤 배경에 카드가 부드럽게 떠 있는 형태라 정보 밀도를 높여도 눈이 피로하지 않고, 표·차트·KPI 카드의 톤이 통일됩니다. 강렬한 컬러가 필요한 차트는 카드 안쪽에서만 강조되기 때문에 화면 전체가 흐트러지지 않습니다.

STEP 5 · 배포와 Sheets 라이브 연동

완성한 대시보드를 매달 꾸준히 쓰기

시트만 갱신하면 대시보드가 같이 갱신되는 구조 — 운영 비용이 거의 0에 가깝습니다

대시보드는 만든 순간보다 운영하는 동안 쌓이는 가치가 더 큽니다. 이번 구조는 데이터를 시트에서 직접 읽기 때문에, 매월 따로 갱신 스크립트를 돌릴 필요가 없습니다. 시트 한 곳만 업데이트하면 다음 방문자에게 새 숫자가 그대로 보입니다.

1
시트 공개 권한 확인
시트의 공유 > 일반 액세스를 "링크가 있는 모든 사용자(뷰어)"로 열어 둡니다. 비공개 상태면 gviz/tq?tqx=out:csv 엔드포인트가 401/302로 막힙니다. 민감 데이터가 같이 들어 있으면 별도 시트로 분리해 두세요.
2
로컬에서 동작 확인
생성된 HTML을 브라우저로 열어 CSV fetch가 성공하는지, 5개 탭이 모두 정상으로 그려지는지, 연도 셀렉터·매장 칩 토글이 동작하는지 점검합니다. 콘솔에 CORS 오류가 뜨면 시트 권한부터 의심하세요.
3
GitHub Pages로 무료 배포
HTML 한 파일을 GitHub 저장소에 푸시하고 Pages를 켜면 24시간 접근 가능한 URL이 생깁니다. 데이터는 원격(Sheets)에서 가져오니, 페이지 자체는 캐시가 잘 먹어 빠르게 뜹니다.
4
월간 갱신은 "시트만 업데이트"
이번 구조에서 매월 해야 할 일은 단 하나 — 시트에 새 달 데이터를 한 줄 더 추가하는 것입니다. 사용자가 다음 번 페이지를 열면 새 숫자가 자동으로 반영됩니다. 별도 JSON 변환, 재배포, refresh 스킬 모두 필요하지 않습니다.
5
의사결정자에게 링크 한 줄
대시보드는 만들었다고 끝이 아닙니다. "매월 첫째 주 월요일, 이 링크 한 번씩만 봐 주세요"까지 같이 약속해 두세요. 그래야 대시보드가 계속 살아 있는 도구로 남습니다.
의사결정자가 실제로 보게 만드는 한 줄

대시보드를 만들어 두고도 아무도 보지 않으면 만든 의미가 없습니다. 만든 다음 날, 의사결정자에게 짧은 한 문장으로 알리세요. "이번 달 가장 큰 변화는 X였습니다. 자세한 내용은 이 링크에서 보실 수 있습니다." 한 번 클릭해 본 페이지는 다음 달에도 다시 들여다보게 됩니다.

이 실습이 끝나면 다음 네 가지가 손에 남습니다.

  • 재사용 가능한 대시보드 틀. 뉴모피즘 HTML 한 파일 안에서 Google Sheets CSV를 직접 읽고 집계하는 구조라, 다른 데이터에도 시트 ID·gid만 바꿔 끼우면 그대로 쓸 수 있습니다.
  • 한 줄짜리 EDA 프롬프트. "이 시트를 EDA로 분석해줘" 한 줄이 앞으로 모든 시트 분석의 출발점이 됩니다.
  • 5탭 경영 대시보드 구성 감각. 종합 / 매장 / P&L / 채널 / 품질로 시야를 나누는 흐름이 앞으로 차트를 고를 때 기준이 됩니다.
  • "시트만 갱신" 운영 모델. 따로 JSON을 갈아끼우거나 재배포할 필요 없이, 시트 한 줄만 추가하면 다음 방문자에게 새 숫자가 보입니다.
