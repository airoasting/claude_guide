1 📝 마크다운 파일 1종 · .md

CLAUDE.md가 낯설다면 마크다운을 모르기 때문입니다. Claude Code에서 가장 많이 쓰이는 파일입니다.

.md Markdown 텍스트 서식

기호로 서식을 표현하는 순수 텍스트. AI가 가장 잘 이해하는 형식이며, CLAUDE.md도 이 형식입니다.

# 제목 (H1)
## 소제목 (H2)

**굵게** / *기울임*

- 목록 항목 1
- 목록 항목 2

> 인용 블록
`CLAUDE.md`  ← 파일명 강조
AI가 마크다운을 선호하는 이유
형식 vs .docx / .pdf
✗.docx, .pdf는 이진 파일이라 AI가 바로 읽지 못하고, 별도의 변환 과정이 필요합니다.
✓.md는 순수 텍스트라 그대로 읽고 활용합니다.
효율 vs .html
✗.html은 태그가 많아 같은 내용에 토큰이 더 많이 소요됩니다.
✓.md는 기호가 최소화되어 더 많은 내용을 담을 수 있습니다.
표현 vs .json / .py
✗.json, .py는 데이터·코드 전용이라 "왜 이렇게 했는지"를 적기 어렵습니다.
✓.md는 자연어 문장으로 의도와 배경을 그대로 적을 수 있습니다.

2 🌐 웹 3총사 파일 3종 · .html · .css · .js

Claude가 만든 웹 페이지를 수정하려면 세 파일의 역할을 알아야 합니다. 하나만 잘못 건드려도 레이아웃이 무너집니다.

.html HyperText Markup Language 구조

웹 페이지의 뼈대. 보이는 모든 요소를 태그로 정의합니다.

<h1>안녕하세요</h1>
<p>단락 텍스트입니다</p>
<button id="btn">클릭</button>
.css Cascading Style Sheets 스타일

웹 페이지의 외모. 색상, 글꼴, 간격, 애니메이션을 정의합니다.

h1 {
  color: #D97757;
  font-size: 24px;
}
button:hover {
  background: #333;
}
.js JavaScript 동작·로직

웹 페이지의 동작. 클릭 처리, 데이터 계산, API 통신을 담당합니다.

// 버튼 클릭 이벤트
document.getElementById('btn')
  .addEventListener('click', () => {
    alert('클릭됨!');
  });

// API에서 데이터 불러오기
const data = await fetch('/api/data')
  .then(r => r.json());

.sh Shell Script 터미널 자동화

터미널 명령을 순서대로 자동 실행합니다. 빌드·배포·파일 정리에 씁니다.

#!/bin/bash
# 빌드 후 배포 자동화
echo "빌드 시작..."
npm run build

echo "오래된 파일 정리..."
rm -rf ./dist/old

echo "완료!"

.json JavaScript Object Notation 데이터·설정

  "model": "claude-sonnet-4-6",
  "permissions": {
    "allow": ["Bash", "Edit"]
  },
  "theme": "dark"
}
.ts TypeScript 타입 안전 JS

JavaScript에 타입 검사를 더한 언어. 큰 프로젝트에서 오류를 미리 잡아줍니다.

// 타입을 미리 정의
interface User {
  name: string;
  age: number;
}

function greet(user: User) {
  return `안녕, ${user.name}!`;
}

ABOUT

"개발 말고, 비즈니스용으로는 Claude를 어떻게 써야 하나요?"

AI 이야기를 나누면서 가장 자주 받는 질문입니다.

시중에 나와 있는 Claude 자료는 대부분 개발자를 대상으로 합니다. 그래서 이 가이드는 비즈니스 리더가 의사결정과 보고, 자동화에 바로 활용할 수 있도록 같은 도구를 비즈니스 관점으로 다시 정리한 자료입니다.

코드를 한 줄도 모르는 리더도 진단부터 자동화까지 한 번에 따라올 수 있도록 전체 내용을 5단계로 구성했습니다.

01 왜 만들었나요

AI 강의에서 가장 자주 받는 질문이 있습니다. "프롬프트는 어떻게 써야 잘 쓰나요?", "Claude Code는 어떻게 쓰나요?" 같은 질문입니다.

매번 같은 답을 반복하는 대신, 한 권으로 정리된 표준 레퍼런스를 만들기로 했습니다. 이 가이드가 그 시작입니다.

02 선정 기준

모든 기능을 담지는 않았습니다. 다음 세 가지 기준을 모두 만족하는 내용만 골라 담았습니다.

  • I코드나 터미널 지식 없이도 바로 시작할 수 있을 것
  • II비즈니스 리더가 곧장 실무에 적용할 수 있을 것
  • III한국의 업무 환경(보고, 결재, 미팅, 문서)에 맞을 것
