1. 토큰·컨텍스트 / 세션 관리
/context시스템 프롬프트·메시지·파일별 토큰 사용량을 시각화하고 최적화 힌트를 보여줍니다.어디서 토큰을 많이 쓰는지 분석하고, 압축·삭제 대상을 찾고 싶을 때.
/compact긴 대화 이력을 AI가 요약해 컨텍스트/토큰 사용량을 줄이고, 절감 전·후 토큰 수를 보여줍니다.예시: /compact 핵심만 남겨줘같은 작업을 이어가되 컨텍스트가 너무 커졌을 때, 토큰을 아끼고 싶을 때.
/clear현재 세션의 대화·도구·파일 컨텍스트를 모두 지우고 완전히 새 대화로 시작합니다.완전히 다른 작업으로 전환할 때, 긴 디버깅 후 리셋하고 싶을 때, 민감 정보가 섞였을 때.
/status현재 모델, 컨텍스트 크기, 세션 설정 등 전반 상태를 요약해서 보여줍니다.지금 어떤 모델/설정으로 돌고 있는지 한눈에 확인하고 싶을 때.
/help사용 가능한 모든 명령어와 간단한 설명을 목록으로 보여줍니다.새로 환경을 익힐 때, 명령어 전체 지도를 보고 싶을 때.
/resume이전에 종료한 세션의 히스토리를 불러와 대화를 이어서 진행합니다.예전에 하던 작업을 다시 이어서 계속할 때.
/undoClaude가 마지막으로 수행한 코드/파일 변경을 취소합니다.방금 적용된 수정이 마음에 들지 않을 때, 직전 변경만 되돌리고 싶을 때.
/exit현재 Claude Code 세션을 종료하고 기본 쉘로 돌아갑니다.더 이상 이 세션에서 작업하지 않을 때, 깔끔하게 종료하고 싶을 때.
2. 모델·리소스 / 비용 관리
/model현재 사용 중인 AI 모델과 컨텍스트 윈도우 크기를 확인·변경합니다.예시: /model Haiku, /model Sonnet, /model Opus가벼운 작업은 저렴한 모델로, 깊은 분석은 강한 모델·큰 컨텍스트로 바꾸고 싶을 때.
/cost현재 세션에서 누적된 입력·출력 토큰 수와 예상 비용을 보여줍니다.특정 작업/세션에서 얼마나 토큰과 비용을 썼는지 모니터링할 때.
/usage계정 전체 기준 토큰 사용량·남은 한도(플랜, Extra usage 등)를 보여줍니다.월별 한도에 얼마나 가까워졌는지, 추가 사용량이 얼마나 남았는지 확인할 때.
/effort추론 강도와 응답 속도 사이의 균형(깊이 vs 속도)을 조절합니다.예시: /effort low, /effort medium, /effort high, /effort max빠르게 러프한 답이 필요할 때 vs 시간이 걸려도 깊은 분석이 필요할 때.
/stats최근 토큰 사용량·모델 사용 패턴을 간단한 통계로 요약합니다.장기간 사용 패턴을 감으로 보고 싶을 때.
3. 파일·컨텍스트 / 프로젝트 스코프 관리
/add특정 파일을 현재 세션 컨텍스트에 추가합니다.예시: /add src/index.html한두 개 파일을 기반으로 분석·리팩토링·요약을 받고 싶을 때.
/add-dir디렉터리 전체를 컨텍스트에 추가합니다.예시: /add-dir ./src프로젝트 전체 구조를 이해시키고, 모노레포 단위로 작업 지시를 하고 싶을 때.
/files세션에 추가된 파일 목록과 크기를 보여줍니다.어떤 파일들이 올라가 있는지, 덩치 큰 파일이 무엇인지 확인하고 싶을 때.
/drop특정 파일·디렉터리를 컨텍스트에서 제거합니다.예시: /drop src/index.html더 이상 필요 없는 파일이 토큰·컨텍스트를 차지하지 않게 정리할 때.
/snapshot현재 코드베이스 상태를 스냅샷으로 저장합니다.예시: /snapshot before-refactor리팩토링·대규모 변경 전 기준점을 남겨두고, 이후 상태와 비교하고 싶을 때.
4. 코드 변경·리뷰 / 품질·보안
/diffClaude가 수행한 코드 변경 사항을 diff 형식으로 보여줍니다.어떤 파일이 어떻게 바뀌었는지, 리뷰 전에 변경 내역을 빠르게 훑고 싶을 때.
/review선택한 파일 또는 변경 사항에 대한 코드 리뷰를 요청합니다.예시: /review src/auth.js코드 품질·구조·버그 가능성·테스트 관점에서 피드백이 필요할 때.
/fix에러 로그·실패한 테스트를 기반으로 버그 수정을 시도합니다.이미 드러난 오류를 중심으로 자동 수정 제안을 받고 싶을 때.
/test테스트 스크립트를 실행하거나 테스트 실행을 요청합니다.변경 후 기존 테스트가 깨지지 않았는지 빠르게 확인하고 싶을 때.
/security-review코드의 보안 취약점, 시크릿 노출, 취약한 패턴 등을 리뷰합니다.외부에 릴리즈되기 전, 보안 관점으로 한 번 더 턴을 돌리고 싶을 때.
/pr-commentsGitHub PR 코멘트를 기반으로 변경 내용을 요약하거나 답변 초안을 작성합니다.복잡한 PR에서 코멘트들을 한 번에 처리할 전략·요약·응답 초안이 필요할 때.
5. 계획·자동 편집 / 워크플로 설계
/plan코드를 바로 수정하지 않고 단계별 작업 계획을 먼저 세우는 플랜 모드로 전환합니다.예시: /plan 로그인 기능 만들기대규모 리팩토링/신규 기능처럼 다단계 작업 로드맵을 먼저 설계하고 싶을 때.
/auto-acceptClaude가 제안한 코드 변경을 사용자 확인 없이 자동 적용하는 모드로 전환합니다.포맷팅, 간단 리팩토링 등 신뢰 가능한 반복 작업을 빠르게 처리하고 싶을 때.
/rewind세션을 특정 시점 이전 상태로 되돌립니다.예시: /rewind, /rewind 3잘못된 방향으로 길게 진행했을 때, 과감하게 일정 시점으로 롤백하고 싶을 때.
/btw메인 흐름을 유지한 채 사이드 질문·노트를 던집니다.예시: /btw Node 버전 뭐야?현재 문맥을 유지한 채 보조 질문/메모를 남기고 싶을 때.
6. 프로젝트 설정·메모리 / CLAUDE.md
/init새 프로젝트용 CLAUDE.md(설정 파일)를 생성하고 기본 설정을 구성합니다.새로운 레포에서 Claude 사용 규칙·컨텍스트 전략·토큰 가이드라인을 초기 세팅할 때.
/memory프로젝트 메모리/CLAUDE.md를 열어 편집합니다.도메인 지식, 금지 규칙, "이 레포에서는 항상 지켜야 할 것들"을 장기 메모리처럼 저장하고 싶을 때.
/config프로젝트 설정 파일을 열어 모델, 컨텍스트 정책, 자동화 옵션 등을 직접 수정합니다.이 레포에서만 적용되는 고정 정책(예: 자동 compact 여부)을 정하고 싶을 때.
/rules.claude/rules 아래 규칙 파일을 확인하고 적용 상태를 점검합니다.팀 룰(코딩 스타일, 금지 패턴 등)이 제대로 적용·인식되고 있는지 확인할 때.
/skills사용 가능한 스킬 목록과 옵션, 대략적인 사용 특성을 보여줍니다.반복 작업을 스킬로 자동화할 때, 어떤 스킬이 있는지 카탈로그를 보고 싶을 때.
/mcpMCP 서버 설정 및 상태를 관리합니다.예시: /mcp list, /mcp add, /mcp remove외부 도구·API 연동을 추가/삭제하거나 연결 상태를 점검하고 싶을 때.
/settings세션/프로젝트 레벨 설정을 확인하고 변경합니다.기본 모델, 자동 행동, 색상/테마 등 전반적인 환경 설정을 조정하고 싶을 때.
/todosClaude가 파악한 "할 일 목록"을 생성·관리합니다.큰 기능·리팩토링 중 남아 있는 서브태스크를 자동으로 정리하고 싶을 때.
7. 스킬·자동화·에이전트 / 플러그인
/run특정 스킬 또는 워크플로를 실행합니다.예시: /run weekly-report자주 반복되는 분석·리포트·정리 작업을 미리 정의해두고 버튼 한 번에 실행하고 싶을 때.
/optimize최적화 스킬을 실행해 코드·설정을 개선합니다.성능·구조 개선이 필요한 코드에 대해 자동 최적화 제안을 받고 싶을 때.
/buddy서브 에이전트를 호출해 특정 역할(리뷰어, QA, PM 등)을 맡깁니다.예시: /buddy reviewer, /buddy qa, /buddy pm메인 대화와 별도로, 역할 분리된 에이전트와 병렬로 작업하고 싶을 때.
/agents서브 에이전트 목록을 보고 관리합니다.여러 역할(성능 튜너, 보안 리뷰어 등)을 구분해서 쓰고 싶을 때.
/plugin플러그인 설치/삭제 및 마켓 탐색 인터페이스를 엽니다.예시: /plugin list, /plugin install, /plugin remove이슈 트래커·모니터링·문서 도구 등 외부 서비스 연동 플러그인을 찾고 관리하고 싶을 때.
/install-github-appGitHub PR 자동 리뷰용 앱 설치를 도와줍니다.PR 생성 시 자동 리뷰·요약을 GitHub에 붙이고 싶을 때.
8. 출력·결과물
/copy [N]Claude의 마지막 N개 응답(또는 기본 1개)을 복사합니다.예시: /copy 3방금 생성된 코드·요약을 IDE, 문서, 이슈 트래커에 바로 붙여 넣고 싶을 때.
/summarize긴 텍스트나 변경 내역을 요약합니다.긴 로그·문서·PR 설명을 한눈에 볼 수 있는 요약본이 필요할 때.
/simplify복잡한 코드·텍스트를 더 단순한 형태로 재작성합니다.복잡한 로직·설명을 팀원이나 비개발자도 이해할 수 있게 정리하고 싶을 때.
/export결과물을 파일·패치 등으로 내보냅니다.예시: /export changes.patch변경 내용을 patch 파일로 저장하거나, 리뷰·배포용 산출물을 아카이브할 때.
9. 보안·운영·환경
/permissions파일/명령 실행 권한 등 Claude가 할 수 있는 작업 범위를 제어합니다.삭제·배포·외부 호출 같은 위험한 액션을 제한하고 싶을 때.
/doctor설치·환경·연동 상태를 진단하고, 성능/컨텍스트 관련 문제를 점검합니다.토큰 사용량이 이상하게 높거나, MCP/플러그인 때문에 문제가 생긴 것 같을 때.
/privacy-settings로그·데이터 수집·텔레메트리 등 개인정보 설정을 관리합니다.기업/규제 환경에서 데이터 사용 범위를 명확히 제한해야 할 때.
/statusline상태줄에 표시되는 정보(모델, 컨텍스트, 경로 등)를 설정합니다.상태줄에 모델·컨텍스트 정보를 띄워 두고 토큰 상황을 상시 감시하고 싶을 때.
/vim입력 모드를 Vim 스타일로 전환합니다.Vim 키맵에 익숙해서 동일한 편집 경험으로 작업하고 싶을 때.
/terminal-setup터미널 입력(Shift+Enter 줄바꿈 등) 관련 설정을 도와줍니다.멀티라인 입력, 엔터 동작 등을 나에게 맞게 커스터마이즈하고 싶을 때.
/sandbox격리된 샌드박스 환경에서 명령을 실행합니다.예시: /sandbox npm install실제 레포에 영향 없이 실험적인 명령/코드를 테스트하고 싶을 때.
/login / /logout계정 로그인/로그아웃을 처리합니다.예시: /login, /logout여러 계정을 번갈아 쓰거나, 공유 머신에서 계정을 분리하고 싶을 때.
/bugClaude Code 관련 버그 리포트를 제출합니다.재현 가능한 버그·오작동을 팀에 바로 공유하고 싶을 때.
/release-notes최신 릴리즈 노트를 확인합니다.새 버전에서 어떤 명령어·기능이 추가·변경되었는지 빠르게 훑고 싶을 때.
10. CLI·외부
claude authCLI에서 계정 인증을 수행하고 액세스 토큰을 관리합니다.예시: claude auth login, claude auth logout, claude auth status새로운 머신/환경에서 Claude Code CLI를 처음 설정하거나 토큰을 갱신해야 할 때.
claude updateClaude Code를 최신 버전으로 업데이트합니다.새 기능·버그 픽스가 포함된 최신 버전으로 올리고 싶을 때.
claude mcpMCP 서버 관련 설정을 CLI에서 일괄 관리합니다.예시: claude mcp list, claude mcp add, claude mcp remove여러 MCP 서버를 한 번에 설정·점검하고 싶을 때.
