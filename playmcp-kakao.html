INTRO

이 실습에서 만들 것

클로드 코드가 작업 결과를 정리한 뒤, PlayMCP 도구로 카카오톡 나와의 채팅방에 보내는 연결을 만듭니다

💬

실행 환경은 클로드 코드입니다. PlayMCP에서 카카오톡 도구함 하나를 만들고, 거기서 받은 MCP 서버 URL을 claude mcp add 명령으로 등록하면 끝납니다. 그 뒤로는 클로드 코드에게 자연어로 부탁하면 작업 요약을 카카오톡 나와의 채팅방으로 보냅니다.

긴 작업을 끝내고 나면 결과를 다시 복사해 휴대폰으로 옮기는 일이 번거롭습니다. 회의 메모, 오늘 바꾼 파일 목록, 다음 액션을 클로드 코드가 직접 정리해 카카오톡으로 보내면, 자리를 떠도 휴대폰에서 바로 확인할 수 있습니다. Kakao PlayMCP는 카카오가 공식으로 운영하는 MCP 플랫폼이고, 그 안의 카카오톡 나와의 채팅방 도구를 클로드 코드에 붙이는 것이 이 실습의 핵심입니다.

⚠️ 먼저 범위를 짚고 갑니다

이 페이지가 안전하게 설명하는 범위는 카카오톡 나와의 채팅방입니다. 친구 채팅방으로 보내기, 단체 채팅방 읽기, 기존 대화 자동 답장은 나와의 채팅방 도구와 다른 문제입니다. 그런 작업은 카카오 공식 API 권한, 비즈니스 채널, 상담톡, 알림톡 같은 별도 영역으로 봐야 합니다.

STEP 1 · 카카오 계정, PlayMCP, 클로드 코드 준비

연결 전에 세 가지만 확인하면 됩니다

계정, PlayMCP 접속, 클로드 코드 MCP 지원이 준비되어 있으면 설정이 빠르게 끝납니다

PlayMCP는 카카오 계정 권한으로 카카오톡 나와의 채팅방에 글을 남깁니다. 그래서 먼저 어느 계정으로 보낼지 정하고, 클로드 코드가 MCP 서버를 받아들일 준비가 됐는지 확인합니다.

🔗 Kakao PlayMCP 공식 사이트 playmcp.kakao.com
1
카카오 계정 로그인 준비
PlayMCP는 카카오 계정 권한을 씁니다. 업무 계정과 개인 계정이 나뉘어 있다면 어느 계정의 나와의 채팅방으로 보낼지 먼저 정합니다.
2
PlayMCP 접속
playmcp.kakao.com에 접속해 로그인합니다. 화면 안에서 도구함 또는 Toolbox 메뉴를 찾습니다.
3
클로드 코드 MCP 확인
터미널에서 claude를 실행한 뒤 /mcp를 입력하면 현재 연결된 MCP 서버를 볼 수 있습니다. 새 서버는 claude mcp add 명령으로 추가하거나, 프로젝트 루트의 .mcp.json에 직접 적습니다.
핵심 기준

PlayMCP가 클로드 코드 전용 버튼을 따로 보여주지 않아도 괜찮습니다. Streamable HTTP 방식의 MCP 서버 URL만 있으면, claude mcp add --transport http로 같은 URL을 넣어 연결할 수 있습니다.

STEP 2 · PlayMCP 도구함 만들기

카카오톡 나와의 채팅방 도구 추가

PlayMCP에서 카카오톡 도구를 도구함에 넣고, 외부 AI 서비스 연결 정보를 확인합니다

PlayMCP는 여러 카카오 도구를 한 도구함에 담아 두고, 그 도구함을 하나의 MCP 서버 URL로 노출합니다. 업무에 필요한 도구만 골라 담아두면 클로드 코드가 호출할 수 있는 범위가 명확해집니다.

1
도구함 생성
PlayMCP에서 새 도구함을 만들고 이름을 정합니다. 예: claude-kakao-work. 업무용으로 쓸 도구만 넣어두면 됩니다.
2
카카오톡 나와의 채팅방 추가
도구 목록에서 카카오톡 나와의 채팅방을 찾아 추가합니다. 권한 확인 화면이 나오면 어떤 정보와 동작을 허용하는지 읽고 승인합니다.
3
MCP 연결 정보 확인
도구함의 연결 또는 공유 메뉴에서 MCP 서버 URL과 인증 안내를 확인합니다. 화면에 표시된 값은 계정 권한과 연결되므로 공개 문서나 GitHub에 올리지 않습니다.
📌 화면 이름은 바뀔 수 있습니다

카카오가 PlayMCP를 계속 확장하고 있으므로, 버튼 이름이 연결, Connect, Toolbox, 도구함처럼 다르게 보일 수 있습니다. 핵심은 같습니다. 카카오톡 나와의 채팅방 도구를 도구함에 넣고, MCP 서버 URL을 받는 것입니다.

STEP 3 · claude mcp add로 PlayMCP 서버 등록

터미널 명령 한 줄로 MCP 서버 붙이기

클로드 코드는 MCP 서버를 CLI 명령이나 .mcp.json 파일로 등록합니다

PlayMCP가 서버 URL을 보여주면, 터미널에서 다음 명령으로 등록합니다. https://playmcp.kakao.com/your-mcp-url 자리에는 STEP 2에서 복사한 실제 URL을 넣습니다.

claude mcp add --transport http playmcp-kakao https://playmcp.kakao.com/your-mcp-url

여기서 playmcp-kakao는 서버 이름입니다. 이 이름이 나중에 /mcp 목록과 도구 이름 앞에 그대로 붙습니다. 인증 토큰을 헤더로 넘겨야 하는 서버라면 --header 옵션을 함께 씁니다.

claude mcp add --transport http playmcp-kakao https://playmcp.kakao.com/your-mcp-url \ --header "Authorization: Bearer YOUR_TOKEN"

팀과 함께 쓰는 프로젝트라면 명령 대신 프로젝트 루트의 .mcp.json에 직접 적어 버전 관리에 올릴 수도 있습니다. 토큰은 파일에 그대로 박지 말고 환경 변수로 빼는 편이 안전합니다.

{ "mcpServers": { "playmcp-kakao": { "type": "http", "url": "https://playmcp.kakao.com/your-mcp-url", "headers": { "Authorization": "Bearer ${PLAYMCP_TOKEN}" } } } }
1
등록 범위 정하기
개인 환경 전체에서 쓰려면 claude mcp add에 --scope user를, 특정 프로젝트에서만 쓰려면 --scope project를 붙입니다. 별도 지정이 없으면 현재 프로젝트(local) 범위로 들어갑니다.
2
명령 실행
위 claude mcp add 명령을 실행합니다. URL과 서버 이름이 맞으면 등록 완료 메시지가 뜹니다. claude mcp list로 등록된 서버 목록을 다시 확인할 수 있습니다.
3
연결 확인
클로드 코드를 새로 실행하거나 새 세션에서 /mcp를 입력합니다. playmcp-kakao와 카카오톡 관련 도구가 connected 상태로 보이면 연결된 것입니다.
보안 팁

토큰을 명령이나 파일에 직접 적어야 하는 방식이라면 환경 변수로 빼는 편이 낫습니다. .mcp.json은 ${PLAYMCP_TOKEN} 형태로 환경 변수를 참조할 수 있습니다. PlayMCP가 OAuth 로그인을 지원하면 /mcp 화면의 인증 흐름을 우선 사용합니다.

STEP 4 · 첫 테스트로 연결 확인

나와의 채팅방에 짧은 메시지 보내기

처음에는 민감한 정보가 없는 짧은 문장으로 도구 호출을 확인합니다

클로드 코드 세션에서 다음처럼 요청합니다. 도구 사용 승인 화면이 뜨면 카카오톡 나와의 채팅방 도구를 쓰도록 허용합니다.

카카오톡 나와의 채팅방으로 테스트 메시지를 보내줘. 메시지는 "클로드 코드와 PlayMCP 연결 테스트 완료"라고 적어줘.

정상적으로 연결돼 있으면 클로드 코드가 응답 도중에 playmcp-kakao의 카카오톡 도구를 호출하겠다는 승인 요청을 띄웁니다. 승인하면 휴대폰의 나와의 채팅방에 메시지가 도착합니다. 도구를 거치지 않고 "보냈습니다"라고만 답하면 실제로는 보내지지 않은 것이니 연결 상태를 다시 확인합니다.

📌 동작 확인 체크리스트
  • 클로드 코드가 playmcp-kakao 도구를 호출하겠다는 승인 요청을 띄우는가
  • 승인한 뒤 도구 호출 결과가 성공으로 표시되는가
  • 휴대폰 또는 데스크톱 카카오톡의 나와의 채팅방에 메시지가 실제로 도착했는가

세 개 모두 확인되면 연결이 끝났습니다. 하나라도 빠지면 트러블슈팅 섹션을 봅니다.

연결을 확인했다면, 실무에서 바로 쓰는 요청으로 넘어갑니다. 본인 업무에 맞는 형태를 골라 던져 봅니다.

오늘 이 프로젝트에서 바꾼 내용을 5줄로 요약하고, 내가 내일 확인해야 할 항목 3개를 붙여서 카카오톡 나와의 채팅방으로 보내줘.

STEP 5 · 업무용으로 쓸 때의 운영 기준

메신저 연결의 안전 장치

메신저 연결은 편하지만, 한 번 보내면 되돌리기 어렵습니다

나와의 채팅방은 본인만 보는 공간이라 사고가 나도 외부로 퍼지지는 않습니다. 그래도 토큰이 노출되거나 보내면 안 되는 정보가 섞이는 일은 막아야 합니다. 운영 기준을 미리 정해두면 자동화 범위를 넓힐 때도 안심할 수 있습니다.

1
승인 후 전송
처음 운영할 때는 모든 카카오톡 전송 도구를 승인 후 실행으로 둡니다. 클로드 코드에서 이 도구를 always allow로 바꾸기 전에 메시지 형식과 민감 정보 필터를 충분히 확인합니다.
2
보낼 수 없는 정보 정하기
주민등록번호, 고객 전화번호, 계약 금액, 내부 비밀번호, 비공개 링크처럼 메신저로 보내면 안 되는 항목을 CLAUDE.md나 프로젝트 지침에 적어둡니다. 클로드 코드는 이 파일을 읽고 행동 기준으로 삼습니다.
3
짧은 보고 형식 고정
나와의 채팅방에는 긴 원문보다 제목, 핵심 요약, 다음 액션, 링크만 보내는 편이 좋습니다. 상세 내용은 GitHub, 문서, 대시보드 같은 원본 위치에 둡니다.

FAQ · 자주 막히는 지점

연결이 안 될 때 점검 순서

PlayMCP, 카카오 권한, 클로드 코드 MCP 설정을 차례대로 확인합니다

Q1. PlayMCP 화면에 클로드 코드 버튼이 보이지 않습니다

클로드 코드 전용 버튼이 없어도 MCP 서버 URL만 있으면 연결할 수 있습니다. URL이 보이지 않으면 PlayMCP의 외부 AI 서비스 연결, 도구함 공유, MCP 연결 메뉴를 확인합니다. URL이 http 방식인지 sse 방식인지도 함께 봐 두면 --transport 옵션을 정확히 맞출 수 있습니다.

Q2. /mcp에 playmcp-kakao가 보이지 않습니다

원인: 등록이 안 됐거나, 다른 범위(scope)에 등록돼 현재 세션에서 안 보이는 경우입니다.

해결:

  • claude mcp list로 서버가 실제로 등록됐는지 먼저 확인합니다.
  • 프로젝트 전용으로 넣었다면 같은 프로젝트 폴더에서 클로드 코드를 실행했는지 봅니다.
  • 설정을 바꾼 뒤에는 새 세션을 열어야 반영됩니다.
Q3. 서버는 보이는데 failed 또는 인증 오류가 납니다

원인 후보:

  • 토큰 만료. PlayMCP에서 카카오 계정으로 다시 로그인하고 연결 정보를 새로 받습니다.
  • 헤더 형식 오류. Authorization: Bearer ...의 공백과 토큰 값을 다시 확인합니다.
  • 전송 방식 불일치. PlayMCP가 준 URL 방식과 --transport 값이 같은지 봅니다.
Q4. 도구는 보이는데 메시지가 보내지지 않습니다

카카오 계정 권한이 만료됐거나 PlayMCP 도구함에서 카카오톡 도구가 빠졌을 수 있습니다. PlayMCP에서 다시 로그인하고, 카카오톡 나와의 채팅방 도구가 도구함에 활성화돼 있는지 확인합니다. 클로드 코드 쪽에서는 도구 승인을 실수로 거부하지 않았는지도 봅니다.

Q5. 친구나 단톡방으로도 보낼 수 있나요?

이 페이지는 나와의 채팅방 기준입니다. 친구 발송, 단톡방 읽기, 자동 답장은 권한과 정책이 다른 영역입니다. 고객 알림은 알림톡이나 비즈메시지로, 상담 자동화는 카카오톡 채널과 상담톡으로 별도 검토해야 합니다.
