Level 1 기초 - 단계별로 천천히 8개 팁
1 IDE 설치 (VS Code, Cursor 등) brew install --cask visual-studio-code ▼
  • VS Code 또는 Cursor 중 하나를 설치
  • 설치 후 정상적으로 실행되는지 확인
  • 터미널(Terminal) 패널을 열어보기 (Ctrl + ` )
2 설치하기 sudo npm install -g @anthropic-ai/claude-code ▼
  • Node.js가 설치되어 있는지 확인 node -v
  • 터미널에서 Claude Code 패키지 설치
  • claude를 입력해서 처음 실행하기
  • Anthropic 계정으로 인증 완료
3 첫 프로젝트 만들기 mkdir my-project && cd my-project && claude ▼
  • 새로운 작업 폴더를 하나 만들기
  • "스탑워치 웹페이지를 만들어줘" 입력
  • 만들어진 결과물을 브라우저로 열어보기
  • 결과물이 정상 동작하는지 직접 클릭해서 테스트
4 CLAUDE.md 활용 /init ▼
  • /init을 입력하여 프로젝트 초기화
  • 자동으로 만들어진 CLAUDE.md 내용 살펴보기
  • 프로젝트에 맞는 규칙이 어떻게 설정되는지 파악
5 Esc로 중단/복구 Esc · Esc×2 ▼
  • 진행 중인 작업에서 Esc 키로 멈추기
  • 한 번 더 Esc를 눌러 이전 상태로 되돌리기
  • 되돌리기 후 파일이 원래대로 복구되었는지 확인
6 /doctor로 환경 점검 /doctor ▼
  • /doctor를 실행하여 환경 점검
  • 진단 결과에서 문제 여부 확인
  • 제안된 해결 방법이 있다면 적용
7 이미지 & 에러 입력 드래그 & 드롭 · 붙여넣기 ▼
  • 에러 메시지 복사 후 붙여넣기
  • 스크린샷을 드래그앤드롭으로 입력하기
  • Claude가 이미지를 인식하고 분석하는지 확인
8 작업 목록(To-Do) 확인 자동 생성 · "작업 목록 보여줘" ▼
  • 복잡한 요청을 하면 Claude가 자동으로 작업 목록을 생성하는 것 확인
  • 작업이 진행될 때마다 체크 표시가 업데이트되는 것 관찰
  • "작업 목록 보여줘"라고 요청해서 현재 진행 상황 확인
Level 2 중급 - 나도 바이브코딩 11개 팁
9 자동 승인 모드 Shift + Tab ▼
  • Shift + Tab으로 자동 승인 모드 전환
  • 하단 표시줄에서 현재 모드 확인
  • 파일 생성·수정이 자동 승인되어 빠르게 진행되는 것 확인
10 Plan 모드 활용 Shift + Tab ×2 ▼
  • Shift + Tab을 한 번 더 눌러 Plan 모드로 전환
  • 복잡한 기능에 대해 설계를 요청
  • 코드를 작성하지 않고 계획만 수립하는 과정 관찰
  • 제안된 계획을 확인하고 승인 또는 수정 지시
11 깊이 생각하기 (Extended Thinking) Option+T (Mac) · Alt+T (Win) ▼
  • 별도 설정 없이 기본 활성화되어 있는지 확인 (기본값: ON)
  • Option+T (Mac) / Alt+T (Win)으로 켜고 끄기 테스트
  • 간단한 질문 vs 복잡한 질문의 응답 품질 차이 비교
12 @ 파일 참조 @파일명 · @폴더명 ▼
  • @파일명을 입력하여 원하는 파일을 직접 지정
  • @폴더명으로 특정 폴더 전체를 참조
  • 필요한 파일을 명시하여 정확도 높이기
13 /model 모델 전환 /model ▼
  • /model을 입력하여 사용 가능한 모델 목록 확인
  • Haiku, Sonnet, Opus를 바꿔가며 테스트
  • 모델별 속도와 품질 차이를 직접 비교
14 /compact 대화 정리 /compact ▼
  • 대화가 길어지면 /compact로 요약
  • 대화 내용이 압축되어 정리되는 것 확인
  • /compact 뒤에 지시사항을 추가해서 원하는 방식으로 정리
15 PDF 문서 분석 드래그 & 드롭 ▼
  • PDF 파일을 끌어다 놓기
  • 문서 내용을 Claude가 읽고 분석하는지 확인
  • 기획서나 문서를 기반으로 구현 요청해보기
16 효과적인 프롬프트 구체적 서술 · 예시 첨부 ▼
  • 원하는 결과를 구체적으로 서술하는 연습
  • 참고할 예시나 스타일을 함께 전달
  • 수정할 파일 경로를 직접 명시
  • 출력 형식이나 구조를 미리 지정
17 작은 단위로 나누기 한 번에 한 가지씩 ▼
  • 큰 기능을 작은 조각으로 나누기
  • 한 번에 한 가지만 요청하기
  • 각 단계마다 결과를 확인하고 다음으로 진행
18 ! 즉시 실행 !ls · !npm start ▼
  • !ls 또는 !npm start처럼 느낌표(!) 뒤에 명령어를 붙여 실행
  • 별도 확인 과정 없이 곧바로 실행되는 것 확인
  • 반복적인 간단 명령에 적극 활용
19 메모리 활용 /memory · "이건 기억해줘" ▼
  • "이건 기억해줘"라고 말하면 다음 세션에도 유지되는지 확인
  • /memory로 현재 저장된 메모리 내용 확인
  • 반복되는 선호사항(코딩 스타일, 언어 등)을 메모리에 저장
Level 3 고급 - 완전 정복 4개 팁
20 대화로 계획 세우기 "코딩 전에 먼저 나한테 질문해줘" ▼
  • "코딩 전에 먼저 나한테 질문해줘"라고 요청
  • Claude의 질문에 답변하면서 요구사항을 구체화
  • 정리된 계획서를 확인하고 구현 시작 전에 최종 검토
21 /context 기억 상태 확인 /context · /clear ▼
  • /context를 실행하여 현재 상태 확인
  • Claude가 현재 기억하고 있는 양 확인
  • 기억 한도에 가까워지면 자동으로 정리되는 것 이해
  • 새 작업을 시작할 때 /clear로 깔끔하게 리셋
22 HANDOFF.md 인수인계 @HANDOFF.md ▼
  • "지금까지 내용을 HANDOFF.md로 정리해줘" 요청
  • 대화 초기화 후 @HANDOFF.md를 참조하여 작업 재개
  • 이전 내용을 기억한 채로 작업이 이어지는지 확인
23 나만의 명령어 만들기 mkdir -p .claude/commands/ ▼
  • .claude/commands/ 폴더 새로 만들기
  • 마크다운(.md) 파일에 나만의 명령어 정의
  • /project:명령어 형태로 직접 실행해보기
  • 개인 전용 명령어는 홈(~/.claude/commands/)에 저장
