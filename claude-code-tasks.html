04
GitHub 업로드
★★★☆☆
▼
프롬프트
1. Github.com 아이디 생성 2. New Repository 생성 3. API Key 생성 (프로필 → Settings → Developer Settings → Personal Access Tokens → Tokens Classic) 4. 해당 API Key를 클로드 코드에 주고 파일 업로드(Push) : 절대로 메모리에 저장시키면 안됨, 보안 관리 5. Github.com에서 해당 Repo 클릭 → Settings → Pages → Branch에서 main, /root 선택하고 Save 6. 아이디.github.io/Repository명 으로 웹사이트로 접근
기대 효과
git init → commit → push → Pages 배포까지 전 과정을 자연어 한 줄로 완료. 내 작품이 인터넷에 공개되는 순간을 경험합니다.

07
CEO 데일리 브리핑 대시보드
★★★★★
▼
프롬프트
CEO 아침 브리핑 대시보드를 만들어줘. 오늘의 날씨, 환율·마켓, AI 비서가 요약한 업계 뉴스, AI 명언, 오늘 일정을 한 화면에 모은 개인 대시보드야. 중요: API 키 없이도 샘플(mock) 데이터로 완전히 작동하게 만들어줘. 그리고 설정 패널에서 본인 API 키(뉴스·날씨·환율)를 입력하면 실시간 데이터로 전환되도록 해줘. 뉴모피즘 스타일로.
기대 효과
키 없이도 mock 데이터로 완성형 대시보드가 바로 작동하고, 본인 API 키를 연결하면 실시간으로 전환됩니다. "키 없이 완성, 키 있으면 실시간"이라는 실전 앱 설계 패턴을 익힙니다.

ABOUT

"개발 말고, 비즈니스용으로는 Claude를 어떻게 써야 하나요?"

AI 이야기를 나누면서 가장 자주 받는 질문입니다.

시중에 나와 있는 Claude 자료는 대부분 개발자를 대상으로 합니다. 그래서 이 가이드는 비즈니스 리더가 의사결정과 보고, 자동화에 바로 활용할 수 있도록 같은 도구를 비즈니스 관점으로 다시 정리한 자료입니다.

코드를 한 줄도 모르는 리더도 진단부터 자동화까지 한 번에 따라올 수 있도록 전체 내용을 5단계로 구성했습니다.

01 왜 만들었나요

AI 강의에서 가장 자주 받는 질문이 있습니다. "프롬프트는 어떻게 써야 잘 쓰나요?", "Claude Code는 어떻게 쓰나요?" 같은 질문입니다.

매번 같은 답을 반복하는 대신, 한 권으로 정리된 표준 레퍼런스를 만들기로 했습니다. 이 가이드가 그 시작입니다.

02 선정 기준

모든 기능을 담지는 않았습니다. 다음 세 가지 기준을 모두 만족하는 내용만 골라 담았습니다.

  • I코드나 터미널 지식 없이도 바로 시작할 수 있을 것
  • II비즈니스 리더가 곧장 실무에 적용할 수 있을 것
  • III한국의 업무 환경(보고, 결재, 미팅, 문서)에 맞을 것
