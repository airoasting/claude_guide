# 매장·배달 주문 관리 매뉴얼

가맹점 매장 주문(POS 입력·주방 전달·서빙·결제)과 배달 플랫폼 주문(배민·요기요·쿠팡이츠·땡겨요·배달특급) 통합 수신·연동·변경·취소·환불을 6단계로 표준화하여 POS 입력 정확도 99% 이상·배달앱 자동 연동률 100%·결제 1분 이내·환불 분쟁 월 1건 이하를 확보하기 위한 지침

소관부서 | 영업팀    분류 | 가맹점 운영 / 주문 관리

---

## [매뉴얼 목적]

### 제 1 조 (목적)

본 매뉴얼은 프랜차이즈 브랜드 가맹점 매장 주문(POS 시스템 입력·KDS 주방 전달·서빙·결제)과 배달 플랫폼 주문(배민·요기요·쿠팡이츠·땡겨요·배달특급 5종 통합 수신·POS 자동 연동) 그리고 변경·취소·환불 처리를 6단계로 표준화하여 POS 입력 정확도 99% 이상, 주문 후 주방 전달 60초 이내, 결제 처리 1분 이내, 배달앱 주문 자동 연동률 100%, 변경·취소 D-day 처리 100%, 환불 분쟁 월 1건 이하를 유지하기 위한 운영 기준을 확립하는 데 목적이 있다. Ch01 응대표준의 STEP 3 주문 접수 동선과 Ch02 클레임처리의 결제·이물 환불 동선, Ch03 테이블운영의 POS 시스템 테이블 매핑을 본 매뉴얼의 POS 입력·배달앱 연동·환불 처리 단계와 연계 운영하며, 본사 영업팀 SV(Supervisor, 슈퍼바이저)가 월 1회 정기 POS 로직·연동률 점검을 통해 이행 여부를 신속하고 정확하게 검증한다.

### 제 2 조 (개요·운영원칙)

1호. 주기: 매 영업일 오픈 30분 전 POS·메뉴 점검부터 마감 정산·이상치 분석까지 상시 적용. 매 주문 건별 POS 입력 즉시 반영, 신메뉴·단가 변경 시 D-1까지 POS 갱신, 변경·취소·환불 발생 즉시 처리, 배달앱 장애 발생 시 30분 단위 점검, 본사 SV는 월 1회 정기 POS 로직·연동률 점검을 시행한다.

2호. 대상: 프랜차이즈 브랜드 직영·가맹 전 매장 매장 주문(POS·KDS·결제) 및 배달 플랫폼 5종(배민·요기요·쿠팡이츠·땡겨요·배달특급) 주문, POS 시스템 메뉴로직, 배달앱 통합 수신기(BIDA·부릉·바로고 등) 연동 데이터, 결제·환불·취소·정산 데이터 일체.

3호. 담당자: 가맹점 홀 직원·아르바이트(POS 입력·서빙·결제 실무), 가맹점 홀 매니저(POS 시스템 검수·이상치 1차 대응), 가맹점주(메뉴·단가 변경 검수 및 5만원 이하 환불 자체 결재), 본사 영업팀 SV(POS 로직 점검·배달앱 연동 이슈 대응·5~20만원 환불 결재), 영업팀 팀장(20만원 초과 환불·시스템 장애 본사 이관 결재).

4호. 항목: POS 시스템 주문 입력 내역, KDS 송신 시각, 배달앱 5종 자동 수신 데이터, 메뉴 변경·옵션·취소 처리 기록, 결제 수단(현금·카드·간편결제·배달앱 선결제)·현금영수증·적립, 일일 매장·배달 매출 정산표, 환불·취소 사유 분류, 카카오톡 가맹점주 단체방 공유 기록.

5호. 보고방식: 홀 직원 → 홀 매니저 → 가맹점주(5만원 이하 환불 자체 결재) → 영업팀 SV(5~20만원 환불·POS 시스템 이슈) → 영업팀 팀장(20만원 초과 환불·배달앱 장애 본사 이관) 결재 라인을 따른다. 일일 정산표는 마감 직후 홀 매니저가 디지털 양식에 100% 입력하고, 가맹점주가 확인 서명한 뒤 D+1 09:00까지 SV에게 카카오톡으로 공유한다.

6호. 처리 시한: 오픈 30분 전 POS·메뉴 점검 완료, 주문 접수 후 60초 이내 KDS 주방 전달, 결제 1분 이내 처리, 배달앱 주문 수신 후 30초 이내 POS 자동 연동, 변경·취소 발생 후 3분 이내 POS·KDS 동기 처리, 환불 5만원 이하 10분 이내 집행, 마감 후 30분 이내 일일 정산표 입력 완료를 원칙으로 한다.

### 제 3 조 (구성)

본 매뉴얼은 총 6단계 업무 흐름(① 영업 사전 POS·메뉴 점검 → ② 매장 주문 접수·POS 입력 → ③ 배달앱 주문 통합 수신·연동 → ④ 메뉴 변경·옵션·취소 처리 → ⑤ 결제·영수증·적립 → ⑥ 일일 주문 정산·이상치 분석)으로 구성된다.

## [업무 상세작성]

### STEP 1  영업 사전 POS·메뉴 점검

① POS 시스템 로그인 및 메뉴 단가 검증

- 오픈 30분 전 홀 매니저가 POS 시스템에 매니저 권한으로 로그인하여 [메뉴관리] 화면에서 당일 판매 메뉴와 단가를 1건씩 표시 검증한다.
- 본사 ERP [주문관리] 공지사항과 카카오톡 가맹점주 단체방의 신메뉴·단가 변경 D-1 공지를 출근 직후 전 직원에게 공유하고 변경 항목을 칠판에 기록한다.
- POS 단가 표시값이 본사 확정 단가표(매장가·배달가 구분)와 1원이라도 불일치하면 영업 개시를 보류하고 가맹점주가 [메뉴관리]에서 단가를 수정한 뒤 시연 결제 1건으로 재검증한다.
- 품절 메뉴는 [품절처리] 메뉴에서 매장·배달 동시 ON 처리하여 5종 배달앱과 매장 POS의 노출을 일괄 차단한다.

※ POS 단가가 본사 확정 단가표와 1원 이상 불일치하면 가맹점·고객 분쟁 직접 사유에 해당하므로 영업 개시 전 시정 완료를 원칙으로 하고, 시정 지연 시 가맹점주가 즉시 SV에게 유선 보고한다.

② 배달앱 5종 영업 ON·통합 수신기 점검

- 배민·요기요·쿠팡이츠·땡겨요·배달특급 5종 사장님 앱 또는 셀프서비스 화면에서 [영업 시작] ON 상태를 1앱씩 확인하고 캡처를 카운터에 비치한다.
- 통합 수신기(BIDA 등)에서 5종 앱 연결 상태 5개 LED 또는 화면 표시가 모두 녹색인지 확인하고 1개라도 적색·점멸 시 SV에게 즉시 유선 보고한다.
- KDS, 무선 프린터, 카드 단말기, 호출벨을 1대씩 시연 호출하여 정상 작동을 검증하고 시연 영수증·KDS 캡처를 일일 점검표에 첨부한다.

※ 배달앱 5종 중 1종이라도 영업 ON 누락 또는 통합 수신기 적색 표시 시 자동 연동률이 100% 미만으로 떨어지므로 영업 개시 전 시정 완료한다.

### STEP 2  매장 주문 접수·POS 입력

① 주문 복창 및 POS 입력 더블체크

- Ch01 STEP 3 주문 접수 동선에 따라 메뉴명·수량·옵션 순으로 1회 복창하여 고객 확인을 받은 뒤 POS 시스템 [주문입력] 화면에 즉시 입력한다.
- POS 입력 직후 입력 내역을 화면에서 1회 더 확인하고 [주방전송] 버튼을 눌러 KDS 송신 여부를 60초 이내 검증한다.
- 알레르기·맵기 단계(0~5단계)·공기밥 추가·반찬 제외·아구 손질 형태(통째·손질) 옵션은 [비고란]에 텍스트로 별도 입력하여 KDS 화면 빨간 배너로 강조 송신한다.
- POS 입력 정확도 99% 미달 매장은 SV가 월간 점검 시 입력 누락·오입력 사례 상위 3건을 추출하여 익월 재교육 자료로 활용한다.

※ 주문 복창 1회·POS 입력 후 화면 재확인 1회의 더블체크를 생략하면 컴플레인 직접 사유에 해당하므로, 피크 타임에도 두 단계를 동시 누락하지 않는다.

② 주문 후 주방 전달 60초 시한

- POS [주방전송] 직후 KDS 화면에 메뉴·수량·옵션이 정상 표시되는지 60초 이내 1회 육안 확인한다.
- 60초 이내 KDS 미송신 시 무선 프린터 출력으로 백업 전달하고, 사유(통신·KDS 단말 오류 등)를 일일 점검표에 기재한다.
- 동일 KDS 미송신 사유가 1일 3건 이상 누적되면 가맹점주가 SV에게 유선 보고하고 통합 수신기·라우터 점검을 D+1 이내 요청한다.

※ KDS 미송신 백업 출력 누락 시 주문 누락으로 직결되므로 60초 시한을 절대 지표로 관리한다.

### STEP 3  배달앱 주문 통합 수신·연동

① 5종 배달앱 자동 연동 검증

- 배민·요기요·쿠팡이츠·땡겨요·배달특급 5종 주문은 통합 수신기를 거쳐 30초 이내 POS 시스템 [배달주문] 화면에 자동 연동되며, 30초 초과 시 통합 수신기 LED 또는 화면 표시를 즉시 확인한다.
- 자동 연동된 주문은 POS [배달주문] 화면에서 앱별 색상 태그(배민 청록·요기요 빨강·쿠팡이츠 분홍·땡겨요 연두·배달특급 파랑)로 구분 표시되어 KDS에 자동 송신된다.
- 메뉴 카테고리·옵션 매핑은 본사 영업팀이 작성한 [메뉴로직도] 기준으로 5종 앱이 동일 POS 코드로 매핑되어야 하며, 1건이라도 미매핑 메뉴 노출 시 가맹점주가 SV에게 즉시 유선 보고한다.

※ 통합 수신기 30초 자동 연동 미달 시 자동 연동률 100% 목표가 즉시 깨지므로, 5분 이내 수기 전환을 시행하지 않으면 SV 정기 점검 감점 항목으로 기록된다.

② 미연동 주문 수기 전환 처리

- 자동 연동 5분 초과 미수신 주문은 홀 매니저가 사장님 앱 화면에서 메뉴·수량·옵션을 직접 확인하여 POS 시스템 [수기 입력] 메뉴에 입력한다.
- 수기 입력 시 [출처] 필드에 5종 앱 중 해당 앱명을 선택하여 정산 시 자동 분류되도록 하고, 사유(통신 장애·앱 점검 등)를 비고란에 기재한다.
- 배달앱 장애 30분 초과 시 가맹점주가 SV에게 카카오톡 [긴급] 표기로 즉시 보고하고 SV가 본사 영업팀 통합 수신기 담당자에게 이관한다.

※ 수기 입력 누락분은 일일 정산 시 매장·배달 매출 불일치로 직결되므로 미연동 주문 100% 수기 전환을 원칙으로 한다.

### STEP 4  메뉴 변경·옵션·취소 처리

① 조리 시작 전 변경·취소 동기 처리

- 고객 또는 배달앱 측 변경·취소 요청 접수 후 KDS [조리 시작] 표시 전 3분 이내에 한해 POS 시스템 [주문 변경] 또는 [주문 취소] 메뉴로 동기 처리한다.
- 변경 처리 시 POS·KDS·통합 수신기 3개 화면에 동일 변경 내역이 반영됐는지 1건씩 육안 확인한 뒤 고객 또는 배달앱 측에 변경 완료 회신한다.
- 매장 주문 취소는 POS [주문 취소] 메뉴에서 사유(고객 변심·재료 소진·오입력)를 3구분으로 기재하고, 배달앱 취소는 사장님 앱에서 [업주 취소]를 선택하여 페널티 사유 코드(재료 소진·매장 사정 등)를 명시한다.

※ 조리 시작 후 메뉴 변경·취소는 식자재 손실로 직결되므로 가맹점주 결재 없이 임의 처리하지 않으며, 가맹점주 결재 시각·사유를 POS 비고란과 일일 점검표에 동시 기록한다.

② 옵션 변경 및 KDS 재송신

- 맵기 단계·공기밥 추가·반찬 제외 등 옵션 변경은 POS [옵션 수정] 메뉴에서 처리하고 KDS 우선순위 1번으로 재송신하여 주방이 즉시 인지하도록 한다.
- 옵션 변경 후 주방이 새 옵션을 KDS 화면에서 [확인] 처리하면 POS 화면에 변경 시각이 자동 기록되어 사후 분쟁 시 증거로 활용한다.
- 동일 주문 옵션 변경이 3회 이상 누적되면 홀 매니저가 가맹점주에게 즉시 보고하고 추가 변경은 가맹점주 직접 결재로 전환한다.

※ KDS 재송신 누락 시 주방이 구 옵션으로 조리하여 컴플레인 직접 사유가 되므로, 옵션 변경은 POS 입력 → KDS 재송신 → 주방 확인 3단계를 1건도 생략하지 않는다.

### STEP 5  결제·영수증·적립

① 결제 1분 이내 처리 및 사전 청취

- Ch01 STEP 4 결제 동선과 동일하게 결제 요청 접수 후 1분 이내 카드 단말기 또는 POS 시스템 [결제] 메뉴로 처리를 완료한다.
- 결제 직전 현금영수증 발행 여부, 법인카드 여부, 간편결제(카카오페이·네이버페이·삼성페이) 사용 여부, 적립·쿠폰 사용 여부를 1회 일괄 청취하여 결제 후 정정 발생을 차단한다.
- 배달앱 선결제 주문은 별도 결제 절차 없이 사장님 앱 [조리 완료] 처리만 시행하고, 후불 주문(배달특급 일부)은 라이더 도착 시 카드 단말로 결제한다.

※ 결제 후 현금영수증 추가 발행은 POS [영수증 재발행] 메뉴에서 D+1 영업일 24:00 이내에 한해 가능하므로, 결제 직전 사전 청취를 통해 D-day 발행을 원칙으로 한다.

② 환불·재발행·적립 처리

- 5만원 이하 환불은 가맹점주 자체 결재로 POS 시스템 [결제 취소] 메뉴에서 원결제 카드 동일 채널로 10분 이내 처리하고, 영수증과 취소 영수증을 함께 보관한다.
- 5~20만원 환불은 영업팀 SV 유선 결재 후 집행하며, 20만원 초과는 영업팀 팀장 결재 후 집행한다(Ch02 부록 1 보상 전결권과 동일 기준).
- 적립·쿠폰은 결제 직전 [적립/쿠폰] 메뉴에서 휴대폰 번호 또는 QR 스캔으로 사전 적용하고, 결제 후 소급 적용은 D+3 영업일 이내에 한해 가맹점주 결재로 처리한다.

※ 현금 환불은 위변조 영수증 위험으로 원칙적으로 금지하며, 카드 결제 취소 또는 모바일 상품권으로 대체한다(Ch02 STEP 3과 동일 기준).

### STEP 6  일일 주문 정산·이상치 분석

① 매장·배달 매출 일치 검증

- 마감 30분 이내 홀 매니저가 POS 시스템 [일일 마감] 메뉴에서 매장 매출·배달 매출(앱별 5종 분류)·환불·취소 합계를 출력하여 디지털 양식에 100% 입력한다.
- 5종 배달앱 사장님 앱 [정산 내역]과 POS [배달주문] 매출이 1원이라도 불일치하면 미연동·수기 입력 누락·환불 미반영을 우선 점검하고 사유를 비고란에 기재한다.
- 매장 현금 매출은 카운터 시재와 POS [현금 매출]의 차이가 ±5천원 이내여야 하며, 초과 시 가맹점주가 시재 재확인 후 원인 사유서를 작성한다.

※ 매장·배달 매출 불일치 1만원 이상 또는 시재 차이 ±5천원 초과 시 익일 SV 정기 점검 감점 항목으로 기록되며, 누적 2회 발생 시 가맹점 운영 평가 감점 처리된다.

② 이상치 분석 및 SV 보고

- 일일 환불·취소 사유를 3구분(고객 변심·매장 사정·시스템 오류)으로 분류하여 일일 정산표에 첨부하고, 시스템 오류 건은 통합 수신기 로그를 캡처하여 별첨한다.
- 가맹점주가 일일 정산표를 확인 서명한 뒤 D+1 09:00까지 영업팀 SV에게 카카오톡으로 캡처 공유한다.
- POS 입력 정확도 99% 미달, 자동 연동률 100% 미달, 환불 분쟁 월 1건 초과 매장은 SV가 D+7 이내 현장 방문하여 POS 로직·수기 전환 동선·환불 결재 라인을 1:1 코칭한다.

※ D+1 09:00 보고 미준수 또는 이상치 사유 미기재 시 SV 정기 점검 감점 항목으로 즉시 기록되며, 누적 2회 누락 시 가맹점 운영 평가 감점 처리된다.

③ 신메뉴·단가 변경 D-1 검수 및 본사 이관

- 본사 영업팀이 본사 ERP [메뉴로직] 또는 카카오톡 단체방으로 신메뉴·단가 변경을 D-3 공지하면 가맹점주는 D-1 18:00까지 POS 시스템 단가 갱신과 5종 배달앱 단가 적용을 완료한다.
- D-1 갱신 누락 매장은 SV가 본사 관리자 권한으로 강제 적용을 검토하고, 가맹점주 동의서 수신 후 본사 대행 적용을 시행한다.
- 단가 적용률 100% 미달, 5종 앱 카테고리 매핑 오류 발생 매장은 영업팀 팀장 보고 후 D+3 영업일 이내 푸드테크(MATE) 측에 메뉴로직 재발송을 요청한다.

※ POS 단가와 배달앱 단가가 1원이라도 불일치하면 정산 오류·고객 분쟁 직접 사유가 되므로 D-1 18:00 시한을 절대 지표로 관리한다.
