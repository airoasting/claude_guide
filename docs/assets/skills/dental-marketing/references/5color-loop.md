# 5-Color Harness 반복 개선 루프

이 문서는 SKILL.md의 STEP 7에서 호출한다. 대상 파일은 `BrandLab_브랜드전략_제안서.docx` 단 하나다. 합격선 달성 또는 최대 10회까지 자동 수정·재생성·재평가한다.

---

## 1. 평가 기준 (매 라운드 동일 적용)

각 평가자에게 0~10점(0.5 단위)을 부여한다.

| 평가자 | 관점 | 합격 기준 행동 | 합격선 |
|--------|------|--------------|--------|
| **RED** | 이성·건조. BrandLab 차별성 근거의 인과 정합성. "왜 광고대행사가 아닌 BrandLab인가"가 수치·사례로 뒷받침되는가. | 차별성 근거를 한 번 읽고 반박 없이 다음 페이지로 넘어간다 | 9.5 |
| **SILVER** | 전문가·깐깐. 실행 가능성, 의료광고법 준수 명시, 가격 투명성, AEO·브랜딩·P&L 효과 타임라인. | 추가 질문 없이 계약 검토 단계로 넘어간다 | 9.5 |
| **BLUE** | 공감·차분. 원장이 "맡겨도 되겠다"는 결정이 가격·일정 의문 없이 생기는가. 단순 대행사가 아닌 파트너로 느껴지는가. | 가격 페이지에서 멈추지 않고 상담 예약으로 넘어간다 | 9.5 |
| **GOLD** | 치과 원장, 평일 오후 3시, 진료 사이 5분. 첫 3페이지에서 차별성과 가격을 납득하고 상담 예약 버튼을 누른다. | 첫 3페이지 내 납득 및 CTA 클릭 | 9.5 |

**합격 기준:** RED·SILVER·BLUE 평균 ≥ 9.5 **AND** GOLD ≥ 9.5

---

## 2. 자동 감점 항목 (라운드마다 먼저 점검)

아래 항목이 하나라도 있으면 해당 평가자 점수를 **자동 -1.0**한다. 감점 후 점수를 산출한다.

| 항목 | 감점 평가자 |
|------|------------|
| "별도 협의", "문의 주세요", "상담 후 결정" 같은 모호한 가격 표현 | SILVER |
| "~인 것 같다", "~로 보인다" 등 추측 어미 | RED |
| em dash(—, –)가 문장 중간에 사용된 경우 | RED |
| 실행 일정이 없는 서비스 섹션 | SILVER |
| 자사 자랑 문장이 차별성 근거 문장보다 많은 섹션 | RED |
| CTA 또는 연락처 누락 | GOLD |
| 경쟁사 수치 나열이 원장 위협감을 조성하는 방식 | BLUE |
| "홍보 대행사" 표현이 "브랜드 전략 파트너" 대신 사용된 경우 | RED |
| P&L·수익 구조 관점 없이 마케팅 실행만 언급된 섹션 | SILVER |

---

## 3. 라운드 진행 방식 (N = 1 ~ 10)

```
1. [평가] 현재 make_proposal.js의 텍스트를 읽어 §1 기준으로 각 평가자 점수 산출.
          §2 자동 감점 항목을 먼저 점검한 뒤 점수에 반영.
          → 점수표 출력: RED / SILVER / BLUE / 평균 / GOLD

2. [판정]
   - 평균 ≥ 9.5 AND GOLD ≥ 9.5 → 합격. 루프 종료 → SKILL.md STEP 8로 이동.
   - N = 10이고 미달            → 10회 한도 도달. 최고 점수 버전으로 저장 후
                                  최종 점수와 미달 이유를 사용자에게 보고.
   - 그 외 미달                 → 3단계(수정)로 진행.

3. [수정] 미달 평가자 중 점수가 가장 낮은 항목부터 우선 수정한다.
          수정 우선순위:
          a) 자동 감점 항목 제거 (즉각 효과 최대)
          b) RED 미달:    수치·출처 없는 주장 → 구체 근거 문장으로 교체.
                          BrandLab 차별성 근거 강화 (패션 C-레벨 실적 수치 활용).
          c) SILVER 미달: 모호한 가격 → 상한/하한 범위 명시 / 실행 일정 추가.
                          P&L 효과 타임라인 구체화.
          d) BLUE 미달:   위협성 경쟁사 나열 → 기회 중심 표현으로 전환. "파트너" 언어 강화.
          e) GOLD 미달:   첫 3페이지 요약·차별성 재배치 / CTA 앞당기기.

4. [재생성] node scripts/make_proposal.js [저장 폴더]
          (STEP 6과 동일한 저장 폴더를 넘겨 같은 파일을 덮어쓴다.
           스킬 폴더에서 실행하므로 docx 모듈 경로가 항상 해석된다.)

5. N += 1. 1단계로 복귀.
```

> 중요: make_proposal.js를 수정할 때는 반드시 Python으로 파일 전체를 재작성한다. Edit 도구는 한국어 UTF-8 멀티바이트 문자가 잘릴 수 있어 사용 금지.

## 4. 라운드별 누적 점수표 (매 라운드 후 출력)

```
| 라운드 | RED  | SILVER | BLUE | 평균 | GOLD | 판정   |
|--------|------|--------|------|------|------|--------|
|   1    |  8.0 |   7.5  |  8.5 |  8.0 |  8.0 | 미달   |
|   2    |  8.5 |   8.5  |  9.0 |  8.7 |  8.5 | 미달   |
|   N    |  9.5 |   9.5  |  9.5 |  9.5 |  9.5 | 합격   |
```

## 5. 루프 종료 후 보고 형식

**합격 시:**
```
[라운드 N] 5color 합격
RED: X.X / SILVER: X.X / BLUE: X.X / 평균: X.X / GOLD: X.X
주요 개선 내역: [라운드 1→N 동안 수정된 핵심 항목 3가지]
```

**10회 한도 도달 시:**
```
10회 반복 후 최고 점수 버전 저장 (라운드 N 버전)
최종: RED: X.X / SILVER: X.X / BLUE: X.X / 평균: X.X / GOLD: X.X
미달 이유: [예: SILVER -X.X 가격 범위 불명확]
추가 수동 수정이 필요한 항목 목록 제공
```

---

## 6. 무한 루프·정체 방지 (stop conditions)

- 점수가 직전 라운드 대비 3회 연속 0.5 미만으로만 개선되면(정체), 남은 자동 감점 항목과 가장 낮은 평가자 1명에 수정을 집중한다. 그래도 2라운드 더 정체되면 10회를 기다리지 말고 조기 종료하고, 최고 점수 버전과 미달 사유를 보고한다.
- make_proposal.js 재생성이 오류로 실패하면 직전 정상 버전으로 롤백하고 사용자에게 오류를 보고한다. 깨진 docx를 합격 처리하지 않는다.
- 점수가 하락하면 직전 라운드의 텍스트로 되돌린 뒤 다른 수정안을 시도한다.
