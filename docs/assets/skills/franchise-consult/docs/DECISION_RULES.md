# 판정 규칙 정본 (Decision Rules Reference)

SKILL.md가 참조하는 정밀 규칙 모음이다. 본문이 한 줄로 요약한 항목의 **운영 가능한 정의**를 여기 둔다. 충돌 시 이 문서가 SKILL.md 본문보다 우선한다(본문은 요약, 이 문서가 정본).

---

## 1. 발동 경계 — 점주 vs 본사 (C1)

이 스킬은 **본사가 예비 가맹점주(후보)를 적격성으로 평가**하는 도구다. "누가, 누구를, 무엇을" 기준으로 가른다.

| 요청 화자 | 평가 대상 | 요청 예시 | 처리 |
|---|---|---|---|
| 본사 담당자 | 예비 점주 | "이 점주 후보 적격성 평가해줘" | **발동** (정상 케이스) |
| 본사 담당자 | 예비 점주 | "상담 메모 붙여넣을게, 진행할지 봐줘" | **발동** |
| 본사 담당자 | 같은 점주 2차 | (같은 사람 추가 메모) | **발동 — 재분석 모드** |
| 예비 점주 본인 | 자기 자신 | "내가 가맹점 할 자격 되는지 봐줘" | **발동** (자료가 적격성 평가 대상이면 화자 무관) |
| 점주/본사 | **점주의 매장 운영 매뉴얼** | "내 가맹점 운영 매뉴얼 써줘", "이 점포 SOP 만들어줘" | **비발동 → `franchise-manual`** |
| 본사 | **본사·브랜드 자체** | "우리 본사 경쟁력 평가해줘" | **비발동** (점주 적격성이 아님) |
| 누구든 | 일반 상권/경영 | "이 동네 매출 데이터 뽑아줘" | **비발동** |

**핵심 분기점**: *"점주 후보를 채점/판정하는가"* = 발동. *"매장을 어떻게 운영할지 문서를 만드는가"* = 매뉴얼 영역(비발동).

**구체 대비 케이스**
- 점주: "내 가맹점 매뉴얼 써줘" → **매뉴얼 영역**. 적격성 채점 아님. `franchise-manual`로 안내.
- 본사: "이 점주 적격성 평가해줘" → **이 스킬**. 정상 발동.
- 혼합: "설문 응답 붙여넣음 + 이 점주용 SOP도 써줘" → **적격성만 채점**하고, SOP는 `franchise-manual` 영역임을 한 줄 안내 후 수행하지 않는다(한 번에 섞어 만들지 않음).

## 2. 응답 모순(응답 모순) 감지 범위 — 숫자형 + 논리형

레드가 분석 **선행 단계**에서 점검한다. 두 종류 모두 탐지 대상이다.

### 2.1 숫자형 모순 (산술 충돌)
서로 다른 응답의 숫자가 산술적으로 양립 불가한 경우.
- 예: 자기자본 3천 < 6개월 예비자금 6천 (예비자금이 자본을 초과 — case02).
- 예: 즉시현금 + 대출 < 총 창업 비용인데 "자금 충분" 주장.
- 예: 회수기간 × 기대 월수익 ≠ 합리적 총투자 규모 (case03 모순 1).
- 예: 홀+포장+배달 비중 합계 ≠ 100%.

### 2.2 논리형 모순 (태도·사실 비정합)
숫자는 맞지만 진술 간 논리/태도가 충돌하는 경우.
- 예: "입지로 망했다"고 진단 + 입지를 전부 본사에 전가 (case03 모순 2 — 태도-행동 모순).
- 예: "매뉴얼 전적 준수" + "본인 레시피로 차별화하고 싶다".
- 예: "전업 전일 운영" + "주당 투입 가능 10시간".
- 예: 무경험 + "직원 10명 규모로 시작" 등 경험-규모 비정합.

### 2.3 모순 처리 절차 (필수)
1. 분석 선행 단계에서 포착 → 레드 진단의 `응답 모순:` 줄에 명시.
2. 해당 영역은 **보수적으로 채점**하되 "모순 해소 시 격상 가능"을 8점 조건/로드맵에 적는다.
3. **블랙의 §5(다음 상담 질문)로 최우선 격상** — 이것이 자가 점검을 "통과 도장"이 아니라 실제 출력으로 만드는 핵심. (시연: case03 모순 1→§5 레드 질문, 모순 2→§5 블루 질문.)
4. 재분석 모드에서는 §0에 이전 모순의 해소/잔존 여부를 명시 (시연: case04).

## 3. "8점 도달 조건" 루브릭 (자의성 제거)

"8점"은 채점 루브릭의 **7~8 구간(양호·신뢰 가능)의 상단**을 뜻한다. 막연한 "더 잘하면 8점"이 아니라, 각 영역에서 **8점이 충족하는 구체 상태**를 아래 기준으로 도출한다.

| 영역 | 8점이 의미하는 상태 (충족 조건) |
|---|---|
| **레드(자금)** | 총 창업 비용 대비 즉시현금 + 예비자금이 명확히 충분 + **현실적 회수기간(24~36개월대) 인식** + 실패 시 자금 마지노선을 본인이 진술 가능 + 미해소 응답 모순 0건. |
| **골드(운영)** | 직접운영 의지·시간 확보 + 관련 경험 또는 본사 교육 이수 동의 + 본인이 부담스러워하는 업무에 대한 보완책(인력/교육) 존재 + 운영 의지·체력 거부권 사유 0건. |
| **블루(상권)** | 희망 행정동 명시 + 직접 답사 + 경쟁 매장 3곳 인지 + 임대료 예산 범위 제시 + 매출 채널(홀:포장:배달) 인식. (정보 50% 미만이면 N/A이며, 8점 조건은 "N/A 탈출 후 위 항목 충족"으로 적는다.) |
| **화이트(브랜드)** | 표준(메뉴·가격·정책) 준수 의향 확인 + 경쟁 브랜드 비교 경험 + 수익 외 선택 동기 1문장 + 슈퍼바이저·클레임 응대 협업 태도 양호. |

**작성 규칙**: 각 페르소나의 `8점 도달 조건:`은 위 표에서 *현재 미충족 항목만* 골라 "X 하면 8점"으로 구체화한다. 막연한 표현("준비를 더 갖추면") 금지.

## 4. STOP / 분석 중단 조건 (Abort)

아래에 해당하면 일반 채점 흐름을 멈추고 지정된 결론만 낸다. 무리하게 4영역을 채워 평균을 내지 않는다.

| 조건 | 처리 |
|---|---|
| **전체 정보의 절반 이상(≈50%+)이 공백** (A~E 핵심 항목 기준) | 채점 시도 중단 → **보류-정보부족** + "분석에 필요한 최소 정보" 목록만 출력. 종합점수 N/A. |
| **채점 가능 영역이 2개 이하** | 종합점수 N/A → **보류-정보부족** (표본 과소로 평균 무의미). 단 §5 우선순위는 작성. |
| **레드·골드 둘 다 거부권(≤4점 또는 결정적 결격)** | 즉시 **부적합**으로 종결. 강점 영역 점수는 표기하되 적합으로 흐리지 않음. (시연: case01.) |
| **명확한 부적합 사유(자금 결정적 부족·운영 의지 부재·가족 반대 등)** | 정보가 부족해도 **부적합 우선**(거절에 추가 정보 불필요). |
| **대조 짝 불명확**(재분석에서 누구의 1차인지 모호) | 채점 전 "어느 상담과 비교할지" 1회 확인. 추측으로 짝짓지 않음. |
| **요청이 매뉴얼/SOP 작성** | 적격성 채점만 수행, 매뉴얼은 `franchise-manual` 안내 후 미수행 (§1 참조). |

## 5. 4.0~5.9 판정 구간 모호성 해소 (vs 2영역 이하 규칙)

SKILL.md 본문의 "4.0~5.9 = 부적합 또는 보류-정보부족"은 아래 결정 트리로 일의적으로 가른다. **거부권 점검을 항상 먼저** 한다.

```
[1] 레드/골드 거부권 또는 명확한 부적합 사유 있는가?
      예 → 부적합 (점수 무관). 끝.
      아니오 → [2]

[2] 채점 영역이 2개 이하인가?
      예 → 종합 N/A → 보류-정보부족. 끝.
      아니오 → [3]

[3] 종합점수 구간으로 판정:
      8.0~10.0 → 적합
      6.0~7.9  → 보류-개선조건부
      4.0~5.9  → [4]
      0~3.9    → 부적합

[4] 4.0~5.9 구간 세부 분기 (거부권 없음·채점 3개 이상 전제):
      핵심 결정 영역(레드·골드)이 정보 부족(N/A 또는 근거 절반 이하)으로
      점수가 낮은가?
          예 → 보류-정보부족 (정보 확보가 우선)
          아니오(정보는 충분한데 실제로 낮음) → 부적합
```

**요지**: 4.0~5.9에서 "정보가 없어서 낮은가, 실제로 부족해서 낮은가"가 갈림길이다. 전자는 보류-정보부족(추가 정보로 바뀔 수 있음), 후자는 부적합(정보를 더 받아도 결론 동일).

## 6. 거부권 vs 보류-정보부족 우선순위 (요약)

- **명확한 부적합 사유에 따른 거부권** (예: 자금 결정적 부족) → **부적합 우선**. 거절에 추가 정보가 필요 없으므로 채점 2개 이하라도 부적합으로 종결.
- **점수만으로 걸린 거부권**(레드/골드 ≤4)인데 **그 판단 근거 자체가 절반 이하**(추측 채점 위험) → 거부권을 확정하지 말고 **보류-정보부족**으로 두고, 그 근거를 §5 최우선 질문으로 올린다.
