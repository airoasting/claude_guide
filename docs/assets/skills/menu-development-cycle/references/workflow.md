# 메뉴개발 사이클 — 진행 원칙 및 사용자 확인 게이트 (v3.0)

> v3.0 핵심 전환
> 이 스킬의 목적은 **새 레시피를 개발할 때 사용자가 각 단계를 직접 확인·점검하며 과정 누락 없이 진행**하도록 돕는 것이다.
> 자동 진행이 아니라, **단계마다 사용자에게 의견을 묻고 그 답변을 주요 안건으로 반영**하는 사용자 주도형 프로세스다.

---

## 대원칙

1. **사용자가 결정자다.** 페르소나 토의(가상 5인)는 **참고용**으로만 제시하고, **모든 단계의 최종 결정은 사용자 확인**으로 확정한다.
2. **단계 누락 금지.** 각 단계의 ✓점검표를 사용자와 함께 확인하고, 빠진 항목이 있으면 다음으로 넘어가지 않는다.
3. **사용자 의견은 주요 안건.** 사용자가 낸 의견·제약·선호는 해당 단계 산출물과 HTML에 **"사용자 의견" 안건으로 명시 기록**한다.
4. **확인 없이 다음 단계로 넘어가지 않는다.** 각 단계 끝에는 반드시 사용자 동의 게이트가 있다.

---

## 사용자 확인 게이트 (공통 패턴)

각 단계는 다음 순서로 진행한다:

```
① (필요 시) 웹 리서치 / 사용자 입력 수집
② 작업 수행 → 산출물 초안 생성
③ 페르소나 토의 요약 (참고용, 짧게)
④ ✓점검표 사용자와 확인
⑤ 【사용자 확인 게이트】
   "이대로 진행할까요? 수정할 부분이 있으면 말씀해 주세요."
   - 동의 → HTML 조각 채움·병합·전달 → 다음 단계
   - 수정 의견 → 의견을 안건에 반영해 산출물 재작성 → 다시 ⑤
```

게이트 질문은 `ask_user_input_v0` 또는 평문 질문으로 제시하되, **사용자가 자유 의견을 낼 수 있어야 하므로** 선택지만 강요하지 않는다. 예: "① 이대로 진행 ② 수정 의견 있음(직접 입력)".

## 1단계: 기획 — 상세 흐름 (★ 사용자 주도)

### 1-1 컨셉기획·시장조사

```
1. 브랜드 + 컨셉 확인 (없으면 질문)
2. 웹 리서치 (트렌드·경쟁사·식재료, 최근 6~12개월)
3. 메뉴 후보 도출 → 사용자에게 제시
   【게이트 A】 "이 메뉴로 진행할까요? 다른 의견 있으면 말씀해 주세요."
     - 사용자가 다른 의견 → 그 의견을 토대로 합당한 새 메뉴를 다시 리서치·도출 → 【게이트 A】 반복
     - 사용자 동의 → 다음
4. 확정 메뉴로 【신메뉴 기획서】 생성 (HTML 조각, 인쇄/엑셀화 가능)
   - 기획서에는 사용자 의견을 "주요 안건"으로 명시 포함
5. 기획서를 토대로 경쟁사 최소 3곳~5곳 조사 → 【경쟁사 분석표】 작성 (HTML 조각)
6. ✓점검표 확인
   【게이트 B】 "신메뉴 기획서와 경쟁사 분석표 확인해 주세요. 이대로 2단계로 갈까요?"
     - 동의 → HTML 병합·전달 → 2-1 진행
     - 수정 → 반영 후 재확인
```

**1-1 산출물 (★ 별도 HTML 문서 2종, A4 3~4장 분량)**

두 문서는 사이클 HTML에 *링크*로 연결되며, 각각 독립 HTML 파일로 저장한다. 문서 상단 툴바로 **엑셀(.xls)·워드(.doc) 저장, PDF·인쇄**가 가능하다.

1. **신메뉴 기획서** (`assets/doc_planning.html` 기반, A4 3장)
   - ① 기획 개요: 기획 배경, 핵심 전략, 기획 범위, 기획 핵심 원칙
   - ② 매장 인근 시장 분석: 인근에 유사 메뉴 판매 매장이 있는지 → 있으면 매장명·메뉴명·가격·특징 + **우리 메뉴와의 차별점**, 없으면 그 의미 기술
   - ③ 유사 브랜드 운영 상황: 유사 메뉴 운영 브랜드가 있는지 → 있으면 브랜드명·메뉴명·가격·특징 + **우리 메뉴와의 차별점**, 없으면 그 의미 기술
2. **경쟁사 분석표** (`assets/doc_competitor.html` 기반, A4 3장)
   - STEP 1 경쟁사 선정: 동종/유사 컨셉 브랜드 3~5개 선정 + 사유
   - STEP 2 정보 수집: 배달앱·SNS·블로그 등 리뷰 데이터 확보 (채널·핵심 데이터)
   - STEP 3 분석표: 경쟁사/브랜드 · 유사메뉴 · 가격 · 구성(특징) · **대표매장 링크**
   - STEP 4 인사이트 도출: 강점 · 약점 · 자사 도입 가능 요소 · 전략

**문서 생성 방법**
- `assets/doc_planning.html`·`assets/doc_competitor.html`을 작업 폴더에 복사 → `{{...}}` 자리표시자를 실제 내용으로 채운다.
- 두 파일 끝의 `/*__DOC_TOOLBAR_JS__*/` 자리에 `assets/doc_toolbar.js` 전체 내용을 인라인 삽입(단일 파일 portable)하거나, 같은 폴더에 두고 `<script src="doc_toolbar.js"></script>`로 연결한다.
- 파일명 예: `어화락_장어양념구이정식_신메뉴기획서.html`, `..._경쟁사분석표.html`.
- 사이클 HTML 1-1 조각의 `{{기획서 파일명}}`·`{{분석표 파일명}}`에 위 파일명을 넣어 링크 연결한다.
- `present_files`로 **사이클 HTML + 기획서 + 분석표 3개 파일을 함께** 전달한다.

## 2단계: 개발 — 상세 흐름 (★ 사용자 자원 확인)

### 2-1 시제품 개발 (레시피 설계)

```
1. 사용자에게 두 가지를 먼저 확인한다:
   ① "보유 식재료" 현황
   ② "매장 기물류" 현황 (예: 살라만더, 튀김기, 오븐, 스팀기 등)
   → ask_user_input_v0 또는 평문 질문으로 수집
2. 보유 식재료 + 기물류를 이용해 만들 수 있는 【레시피 설계서 3~5안】 생성
   - 각 안은 카드형, 재료·사용량·조리법 포함
   - 각 안에 원가(식재료 원가·예상 판매가·원가율) 포함
   - 사용자가 카드 안에서 재료를 추가/삭제하거나 사용량을 조절하면 원가가 자동 재계산되도록 인터랙티브 구현
3. 페르소나 토의 요약 (참고용)
4. ✓점검표 확인
   【게이트】 "레시피 설계안 중 어느 것으로 진행할까요? 재료·양을 조정하셔도 됩니다."
     - 사용자 선택·조정 → 확정안 반영 → HTML 병합·전달 → 2-2 진행
     - 수정 의견 → 반영 후 재확인
```

**2-1 산출물**
- 레시피 설계서 3~5안 (카드형, 재료·사용량·조리법·원가 포함, 인터랙티브 조정 가능)
- 사용자 보유 식재료·기물류 현황 기록

## 이후 단계 (2-2 ~ 6-1)

같은 사용자 확인 게이트 패턴을 따른다. (상세 흐름은 추후 단계별로 보강)

- 사용자 의견은 항상 산출물의 "주요 안건"으로 기록

## HTML 산출물 원칙

- 모든 산출물은 `assets/fragments/`의 조각으로 채워 `merge.py`로 단일 HTML 병합.
- 표(기획서·경쟁사 분석표)는 **인쇄/엑셀 복사 가능**하도록 깔끔한 `<table>`로 작성.
- 레시피 설계서는 **카드형 + 사용자 조정(재료 추가·삭제·사용량 변경 시 원가 자동 재계산)** 인터랙티브로 구현.
- 사용자 의견 안건은 별도 박스(`.user-voice`)로 눈에 띄게 표시.

## 전 문서 공통 규칙 (v3.1 ★)

1. **저장·인쇄 기능 전 문서 동일 적용**: 모든 산출 문서(신메뉴 기획서·경쟁사 분석표·레시피 설계서/테스트 기록지 등)와 개발사이클 HTML 상단에는 `assets/doc_toolbar.js` 기반 툴바(⬇엑셀(.xls)·⬇워드(.doc)·🖨PDF·인쇄)를 넣는다. 문서는 `<div id="docToolbar"></div>` + 끝부분 `/*__DOC_TOOLBAR_JS__*/` 자리에 JS 인라인. 개발사이클 HTML은 merge.py가 자동 주입한다.
2. **모든 문서 3~5장(A4) 구성 통일**: 각 독립 문서는 A4 3~5페이지(`.doc-sheet`)로 구성하고, 인쇄 시 페이지 분할(`page-break-after`)이 적용된다.
3. **레시피 설계서 = 테스트 기록지 겸용** (`assets/doc_recipe.html`): recipe-lab 개념을 반영해 ① 설계 개요+보유 자원 → ② 레시피 설계안 3~5(인터랙티브 원가) → ③ 회차별 테스트 기록(관능 5점·평가자별·직전 대비 ▲▼·개선점→다음 회차 연결) → ④ 표준화 확정(BOM·조리순서·원가율) 으로 A4 4장 구성. 회차는 합격까지 블록을 복제해 추가한다.

엑셀 export는 표 안의 입력값(인터랙티브 셀)도 현재 편집 상태로 반영한다.
