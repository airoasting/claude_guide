Git과 GitHub, 무엇인가요?

이름은 비슷한데 사실 다른 둘, 처음부터 제대로 이해하기

0 한눈에 개요 1 Git 버전 관리 2 GitHub 협업 저장소 3 무엇이 다른가 비교 4 시작하기 설치·가입

💡 한 문장으로

Git은 내 컴퓨터에서 파일 변화를 기록하고 되돌려 주는 프로그램(도구)이고, GitHub는 그 기록을 인터넷에 올려 보관하고 함께 일하는 웹 서비스(장소)입니다. 사진으로 치면 Git은 카메라, GitHub는 찍은 사진을 올려 공유하는 인스타그램입니다.

둘이 무엇이고 안전한지 짧게 정리했습니다. 실제로 코드를 올리고 웹 주소로 띄우는 방법은 GitHub 배포 가이드에서 다룹니다.

Git이란?

Git은 저장할 때마다 파일 상태를 사진처럼 찍어 이력으로 쌓아 두는 버전 관리 프로그램입니다. 그래서 언제든 원하는 시점으로 되돌리고, 무엇이 바뀌었는지 비교할 수 있습니다. 문서 이름 뒤에 '최종', '진짜최종'을 붙여 본 일을 자동으로 대신해 준다고 보면 됩니다. 내 컴퓨터 안에서 돌기 때문에 인터넷이 없어도 동작합니다.

누가 만들었나 · 짧은 역사 · 리눅스를 만든 핀란드 출신 리누스 토르발스가 만든 무료 오픈소스입니다(GNU GPL v2). 지금은 일본 출신 후니오 하마노를 비롯한 전 세계 자원봉사 개발자들이 관리하고, 공식 배포처는 git-scm.com입니다.
  • 2005년 4월 · 리눅스 개발에 쓰던 상용 도구 BitKeeper의 무료 사용이 막히자, 토르발스가 마음에 드는 대안이 없다며 직접 만들기 시작해 며칠 만에 초기 Git을 완성했습니다.
  • 2005년 7월 · 관리 권한이 하마노에게 넘어갔고, 그해 12월 첫 정식 버전(1.0)이 나왔습니다.
  • 지금 · 20년이 지나 전 세계 소프트웨어 개발의 사실상 표준이 되었습니다.
이름의 뜻 · 영어 단어 git은 영국 속어로 "고집불통" 또는 "얼간이"를 뜻합니다. 토르발스가 자기 이름처럼 고집 센 프로그램이라며 농담조로 붙인 이름이라고 알려져 있습니다.
안전한가 · 소스가 공개된 오픈소스라 전 세계가 20년간 검증해 왔고, 올리기(push) 전에는 내 코드가 컴퓨터 밖으로 나가지 않습니다. 공식 사이트 git-scm.com에서만 받으면 안전합니다.

GitHub란?

GitHub는 Git으로 관리한 프로젝트를 인터넷 서버에 올려 두는 웹 서비스입니다. 컴퓨터가 고장 나도 안전하게 백업되고, 여러 사람이 한 프로젝트를 함께 고치고, 원하면 코드를 세상에 공개할 수 있습니다. 1억 명이 넘는 개발자가 쓰는 세계 최대 코드 호스팅 서비스입니다.

누가 만들었나 · 짧은 역사 · 무료 플랜으로 공개·비공개 저장소를 무제한 만들 수 있고, 고급 협업 기능은 유료입니다. 공식 주소는 github.com입니다.
  • 2008년 · 미국 출신 톰 프레스턴 워너, 크리스 완스트라스, PJ 하이엇이 창업했습니다.
  • 2012년 · 약 1억 달러 투자를 유치하며 빠르게 성장했습니다.
  • 2018년 · 마이크로소프트가 약 75억 달러에 인수했습니다.
  • 지금 · 마이크로소프트가 운영하는 세계 최대 코드 호스팅 서비스입니다.
안전한가 · 마이크로소프트가 운영하는 업계 표준이라 서비스 자체는 믿을 만합니다. 다만 인터넷에 올리는 서비스이므로 두 가지만 지키면 됩니다. 저장소를 만들 때 공개(Public)·비공개(Private)를 꼭 확인하고, 비밀번호·API 키·고객 정보 같은 민감한 내용은 올리지 않습니다.

무엇이 다른가요?

항목GitGitHub
정체내 컴퓨터에서 도는 프로그램인터넷에 있는 웹 서비스(회사)
하는 일파일의 변경을 기록하고 되돌림기록한 프로젝트를 인터넷에 저장·공유
인터넷없어도 동작인터넷 접속이 필요
주인·비용토르발스·자원봉사자, 완전 무료마이크로소프트, 기본 무료·일부 유료
비유사진을 찍는 카메라사진을 올려 공유하는 인스타그램

※ GitHub 말고도 GitLab, Bitbucket처럼 Git 프로젝트를 올려 두는 장소는 여러 곳입니다. 모두 바탕에서는 같은 Git을 씁니다.

어떻게 시작하나요?

🪟 Windows에서 Git 설치

공식 사이트 git-scm.com/download/win에 접속하면 설치 파일이 내려받아집니다. 실행한 뒤에는 대부분 기본값 그대로 '다음(Next)'을 눌러 진행하면 됩니다. 설치가 끝나면 명령을 입력하는 'Git Bash'라는 검은 창도 함께 깔립니다.

🍎 Mac에서 Git 설치

터미널 앱을 열고 아래 확인 명령을 입력하면, Git이 없을 때 애플이 설치 안내 창을 띄워 줍니다. 이 방법이 가장 간단합니다. 또는 Homebrew로 brew install git을 쓰거나, git-scm.com/download/mac에서 받아도 됩니다.

✅ 설치 확인

🪟 Windows · 'Git Bash'나 명령 프롬프트를 열고 아래 한 줄을 입력합니다.

git --version

🍎 Mac · '터미널'을 열고 같은 한 줄을 입력합니다.

git --version

두 경우 모두 git version 2.55.0처럼 숫자가 나오면 설치가 끝난 것입니다.

GitHub 가입과 연결

github.com에서 이메일로 무료 가입하고, 'New repository' 버튼으로 저장소를 만듭니다. 이때 공개(Public)·비공개(Private)를 고르는 화면이 나오니 필요에 맞게 선택합니다. 내 컴퓨터의 Git 저장소를 GitHub에 올리면(push) 둘이 연결됩니다. 명령어를 외울 필요는 없습니다. 클로드 코드 같은 도구가 한국어 지시를 받아 대신 실행해 줍니다.

실제 업로드와 웹 배포는 GitHub 배포 가이드, 명령어 없이 시작하려면 클로드 코드 101을 보세요. 공식 문서는 Pro Git(한국어)와 GitHub Docs에 있습니다.
