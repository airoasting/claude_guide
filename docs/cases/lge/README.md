# LG전자 최고경영자 Claude 완전 정복 (LGE)

LG전자 최고경영자가 Claude를 직접 손에 익히도록 설계한 **3단계 프로그램**이다. AI ROASTING · Claude 가이드의 디자인 시스템을 그대로 따르고, 헤더 최상단에 LG전자 공식 로고(lge.co.kr에서 받음)를 둔다.

## 3단계 구조

| 단계 | 내용 | 연결 |
|---|---|---|
| 1단계 | 진단 · 멀티 페르소나 토론 | 상위 가이드 페이지 3개 (`claude-orientation`, `ai-levels`, `multi-persona`) |
| 2단계 | 실습 3개 | 이 폴더의 `persona`, `research`, `simulation` |
| 3단계 | 하네스 엔지니어링 구조 | 상위 가이드 3개 (`harness-engineering`, `claude-tools`, `harness-workflows`) |

인덱스는 상위 인덱스와 같은 카드 형태다. 카드는 가로 3열 그리드로 배열하고, 모바일에서는 1열로 쌓인다. 헤더는 LG전자 로고를 좌측 상단에 두고 제목을 왼쪽으로 정렬한 히어로 레이아웃이며, 배경은 LG 레드 그라데이션이다.

## 자립형 패키지

이 폴더는 외부 의존 없이 단독으로 동작한다. 1·3단계에 연결되는 상위 가이드 6개 페이지와 필요한 에셋(폰트·헤더 영상·로고)을 폴더 안으로 복사해 두었다. 폴더째 옮기거나 배포해도 깨지지 않는다.

- 6개 모듈 페이지는 LG 레드로 다시 입혔다(헤더 그라데이션·액센트·번호·CTA).
- 각 모듈의 "목록으로"는 이 폴더의 `index.html`로 돌아온다.
- LGE에 복사하지 않은 다른 가이드 페이지로 가던 링크는 라이브 가이드(`https://airoasting.github.io/claude_guide/`) 절대 URL로 바꿔 404가 없다.

## 실습 3개는 왜 이 순서인가

세 실습은 **나 → 팀 → 전사**로 범위를 넓히고, 동시에 난이도와 위험도가 한 방향으로 올라간다.

| 순서 | 실습 | 범위 | 위험 | 첫 결과까지 | 역할 |
|---|---|---|---|---|---|
| 1 | CEO 영문 메일 페르소나 | 나 | 낮음 | 5~10분 | 개인적·즉시적 성공으로 신뢰를 만든다 |
| 2 | 리서치 브리프 자동화 | 팀 | 중간 | 한 사이클 | 수집·요약·인사이트 워크플로우와 자동화 사고 |
| 3 | 데이터 기반 경영 시뮬레이션 | 전사 | 높음 | 데이터 연동 후 | 의사결정과 AI·사람 권한 경계 |

전체를 관통하는 질문은 하나다. **"AI가 어디까지 하고, 사람이 어디서 승인하는가."** 단계마다 그 경계를 직접 긋게 한다.

실습 4개 페이지(index/persona/research/simulation)는 `lge.css`/`lge.js`를 공유한다. 6개 모듈 페이지는 각자 인라인 스타일을 가진 원본 구조를 유지하되 색만 LG 레드로 바꿨다.

본문은 두 층으로 짰다.

- **따라하기 본문**: 단계별 do-step, 그대로 쓰는 프롬프트 박스, 비교 카드, 체크리스트.
- **진행자 시연 노트**: 파란색 `진행자 노트` 패널의 시간 배분과 동선, 예상 질문 Q&A.

## 보는 법

정적 HTML이라 더블클릭으로 열면 된다. 폰트·영상·로고를 모두 폴더 안 `assets/`에서 참조하므로 **이 폴더만 떼어내도 그대로 동작한다.** 더 매끄럽게 보려면 로컬 서버로 띄운다.

```
# 프로젝트 루트에서
node server.js     # 또는 임의의 정적 서버
# 브라우저에서 /LGE/ 로 접속
```

## 진행 가이드 (현장)

1. **인덱스에서 3단계 전체 그림을 먼저 보여 준다.** 진단 → 실습 3개 → 하네스 구조.
2. **1단계 진단·토론으로 워밍업한다.** 상위 가이드 페이지로 이동해 진행한다.
3. **2단계 실습은 순서대로.** 사장님이 직접 입력하게 한다. 진행자는 각 페이지의 노트(시간 배분)를 따른다.
4. **각 실습 끝의 체크리스트로 마무리.** 손에 남는 결과물 세 가지를 확인한다.
5. **실습 3은 사전 준비가 필요하다.** 원자재·환율 표본 데이터와 제품 원가 구조, Claude Code + VS Code 설치를 미리 챙긴다.
6. **3단계 하네스 엔지니어링으로 닫는다.** 세 실습의 경험을 구조 언어로 정리한다.

## 고칠 때

- 색·간격·컴포넌트는 `lge.css` 한 곳에서 바꾼다. 4개 페이지에 동시 반영된다.
- 디자인 규약은 상위 [AGENTS.md](../AGENTS.md)를 따른다. 핵심 두 가지.
  - em dash(`—`) 금지. 본문은 경어로 통일.
  - 박스 왼쪽 컬러 테두리(border-left accent) 금지. 유형 색은 채워진 라벨 태그에 담는다.
  - 섹션 제목은 전 페이지 공통으로 데스크톱 20px / 모바일(≤768px) 18px로 통일한다. 선택자는 페이지마다 다르다(`.section-header-block h2`·`.nm-card h2`·`.section-hd h2`·`.sec-head h2`·`.section-box-title`). 새 섹션 제목을 넣을 때 이 두 값을 맞춘다.
- 헤더 영상은 사이트 표준(soft-light 합성)을 그대로 쓴다. 색 보정 불필요.

## 분류

- **유형**: Prototype (특정 고객 대상 실습 자료)
- **재사용 자산**: `lge.css` / `lge.js`는 다른 고객용 실습 미니 사이트로 복제할 수 있는 골격이다. 실습 페이지의 do-step·프롬프트 박스·진행자 노트·권한 매트릭스 구조는 다른 주제에도 그대로 옮겨 쓸 수 있다.
