1막: 두 번의 겨울 · ~2016 ▼
AI 이전, 우리는 어디서 출발했나
AI는 1950년대에 처음 등장했지만 두 번의 겨울을 지나야 했습니다. 그 사이 컴퓨터는 자기만의 인터페이스 표준을 다듬으며, 오늘날 AI 시대에 다시 쓰일 도구들을 묵묵히 준비했습니다.
토대 CLI→ API→ 강화학습 부활

AI 토대

01
신경망Neural Network★

사람 뇌의 뉴런 구조를 본떠 만든 AI의 기본 빌딩 블록입니다. 입력·가중치·출력이 층층이 연결된 구조로, 1940년대 처음 제안돼 오늘날 LLM의 토대가 되었습니다.

사람 뇌 구조를 흉내 낸 AI의 기본 회로

02
머신러닝Machine Learning★★★

사람이 규칙을 일일이 짜는 대신, 데이터를 보고 AI가 스스로 패턴을 학습하게 하는 방법론입니다. 1950년대부터 연구돼 AI의 가장 큰 줄기로 자리 잡았습니다.

AI가 데이터로 스스로 배우는 방식

03
딥러닝Deep Learning★★

여러 층의 신경망을 깊게 쌓아 복잡한 패턴까지 학습하는 머신러닝의 한 갈래입니다. 2010년대 GPU 보급과 이미지넷 충격이 겹치며 폭발적으로 확산되었습니다.

신경망을 깊게 쌓아 학습하는 머신러닝

04
강화학습Reinforcement Learning★

좋은 결과에는 보상을, 나쁜 결과에는 패널티를 줘서 모델을 개선하는 학습 방식입니다. 1950년대부터 연구돼 왔고, 사람의 피드백을 더한 RLHF가 ChatGPT의 핵심 비결이 되었습니다.

AI를 칭찬과 벌로 훈련시키는 방법

05
이미지넷ImageNet★

2009년 페이페이 리 교수팀이 공개한 1,400만 장 규모의 이미지 분류 데이터셋입니다. 2012년 AlexNet이 이 데이터셋에서 압도적 성능을 내며 딥러닝 르네상스의 방아쇠를 당겼습니다.

딥러닝 부흥을 일으킨 이미지 데이터셋

06
AlphaGoAlphaGo · 알파고★

2016년 구글 딥마인드가 공개한 바둑 AI입니다. 강화학습과 딥러닝을 결합해 이세돌 9단을 4-1로 이기며, AI가 인간을 넘을 수 있다는 사실을 전 세계에 각인시켰습니다.

이세돌을 이긴 구글 딥마인드의 바둑 AI

컴퓨팅 토대

07
APIApplication Programming Interface★★★

프로그램끼리 데이터를 주고받는 공식 창구입니다. 수십 년 된 기초 개념이지만, OpenAI API와 함께 비즈니스 리더에게도 다시 익숙한 단어가 되었습니다.

앱들이 서로 대화하는 공식 전화번호

08
CLICommand Line Interface★★

텍스트 명령어로 도구를 실행하는 인터페이스입니다. 1960년대부터 있던 가장 오래된 컴퓨터 사용 방식이지만, 에이전트 시대에 다시 무대 중앙으로 돌아왔습니다.

텍스트로 컴퓨터에 명령하는 검은 화면

09
gitGit★★

2005년 리누스 토르발스가 만든 분산 버전 관리 시스템입니다. 코드 변경을 시점별로 기록·복원·병합하는 표준 도구가 되어, AI 에이전트의 자율 코딩 운영에도 필수가 되었습니다.

코드 변경 이력을 관리하는 표준 도구

10
GitHubGitHub★★

git 기반의 코드 호스팅·협업 플랫폼입니다. 2008년 출시 후 2018년 마이크로소프트가 인수했고, 오픈소스 개발의 중심지이자 AI 코딩 에이전트들이 가장 자주 만나는 무대가 되었습니다.

전 세계 코드가 모이는 git 기반 플랫폼

2막: Transformer 등장 · 2017–2021 ▼

2017년 'Attention is All You Need' 논문 한 편이 판도를 통째로 바꿨습니다. 새 모델 구조 LLM이 태어났고, 토큰·컨텍스트 윈도우·임베딩이라는 단어가 비즈니스 회의에서도 등장하기 시작했습니다.
의미 처리 텍스트→ 토큰→ 임베딩·벡터→ LLM

모델 구조

11
트랜스포머Transformer★★★

2017년 구글이 발표한 'Attention is All You Need' 논문에서 제안한 신경망 구조입니다. 어텐션 메커니즘으로 문장 전체 맥락을 한 번에 보며, 이후 모든 LLM의 표준 구조가 되었습니다.

모든 현대 LLM의 표준 구조

12
어텐션Attention★★

입력 토큰 사이의 관계 중요도를 계산해, 어느 단어에 더 집중할지 가중치를 매기는 메커니즘입니다. 트랜스포머의 핵심이며, AI가 긴 맥락을 이해하게 만든 결정적 기술입니다.

AI가 '어디에 집중할지' 정하는 방식

13
사전학습Pre-training★

방대한 일반 텍스트로 LLM을 먼저 학습시켜 언어의 기본 패턴을 익히게 하는 단계입니다. 이 위에 파인튜닝을 얹어 전문 영역에 맞춥니다.

LLM이 언어 기본기를 익히는 단계

14
파인튜닝Fine-tuning★★

사전학습된 LLM을 특정 도메인이나 업무에 맞게 추가 학습시키는 과정입니다. 회사 자료·전문 분야에 맞춘 모델을 만들 때 쓰입니다.

일반 AI를 전문 분야에 맞춰 다듬기

데이터 표현

15
토큰Token★★

AI가 텍스트를 처리하는 최소 단위입니다. 한글 한 글자가 보통 2~3토큰이며, 컨텍스트 윈도우의 용량과 API 비용도 모두 토큰 수로 측정합니다.

AI가 세는 글자 수 단위

16
임베딩Embedding★

텍스트의 의미를 숫자 벡터로 변환하는 과정입니다. 비슷한 내용끼리 가까운 좌표에 놓이기 때문에, RAG의 유사도 검색이 가능해집니다.

글의 의미를 숫자 좌표로 바꾸는 것

17
벡터Vector★

데이터의 의미를 다차원 숫자 배열로 표현한 결과물입니다. 벡터끼리의 거리를 비교하면 어떤 정보가 어떤 정보와 가까운지 알 수 있습니다.

의미를 표현하는 숫자 묶음, 가까우면 비슷한 뜻

모델·인프라

18
LLMLarge Language Model★★★

대량의 텍스트 데이터로 학습한 대규모 언어 모델입니다. GPT·Claude·Gemini가 모두 여기에 해당하며, 에이전트의 두뇌 역할을 합니다.

AI의 두뇌, 말을 이해하고 생성하는 엔진

19
컨텍스트 윈도우Context Window★★★

AI가 한 번에 읽을 수 있는 정보의 총량입니다. 한도를 넘으면 오래된 정보부터 잘려나가기 때문에, 윈도우 크기는 모델 세대를 가르는 핵심 지표가 되었습니다.

AI의 작업 책상 크기, 넘치면 자료가 떨어진다

20
LSPLanguage Server Protocol★

에디터와 코드 분석 서버 사이의 표준 프로토콜입니다. 2016년 마이크로소프트가 공개했고, 훗날 AI 에이전트 표준인 ACP·MCP의 사고방식에 영향을 미쳤습니다.

에디터에게 코드 구조를 알려주는 번역기

3막: ChatGPT 충격 · 2022–2023 ▼
누구나 AI에게 말을 걸 수 있게 되면 무엇이 따라오나
2022년 11월 ChatGPT가 등장하면서 모두가 AI에게 말을 걸기 시작했습니다. 프롬프트는 새로운 글쓰기로 자리 잡았고, 할루시네이션이라는 한계도 같이 드러났으며, 개발자들은 Cursor 같은 AI 친화 IDE로 무장했습니다.
AI 입력 구성 시스템 프롬프트+ 프롬프트+ 컨텍스트

21
시스템 프롬프트System Prompt★★

AI의 역할·규칙·성격을 미리 설정하는 숨겨진 지시문입니다. 사용자 프롬프트보다 먼저 적용돼서 AI의 행동 기준을 정합니다.

AI에게 미리 귀띔하는 '넌 이런 역할이야'

22
프롬프트Prompt★★★

AI에게 보내는 지시문입니다. '이걸 해줘'라는 모든 요청의 출발점이며, ChatGPT 이후 가장 빠르게 일상어가 된 단어 중 하나입니다.

AI에게 보내는 카카오톡 메시지 같은 것

23
프롬프트 엔지니어링Prompt Engineering★★★

지시문을 구조화해서 원하는 결과를 이끌어내는 기술입니다. 예시·역할·형식 지정이 핵심 기법이며, 한때 새 직업으로 주목받기도 했습니다.

AI에게 잘 부탁하는 기술

24
컨텍스트Context★★

프롬프트만이 아니라 시스템 설정·대화 기록·파일·메모리까지 AI가 참고하는 모든 정보를 통칭합니다. ChatGPT 이후 본격적으로 중요해진 개념입니다.

AI의 기억에 올라와 있는 모든 참고 자료

25
할루시네이션Hallucination★★★

AI가 사실이 아닌 내용을 그럴듯하게 만들어내는 현상입니다. AI의 가장 큰 한계이며, 이후 등장하는 모든 검증 장치의 근본 이유가 되었습니다.

AI가 자신 있게 말하는 거짓말

26
IDEIntegrated Development Environment★★

코드 편집·실행·디버깅을 하나로 묶은 통합 개발 환경입니다. ChatGPT 이후 Cursor 같은 AI 친화 IDE가 등장하면서, 개발자의 기본 무대가 재편되기 시작했습니다.

코딩용 올인원 작업 도구

컨텍스트 윈도우가 1만에서 200만 토큰까지 커졌습니다. 이제는 '어떤 정보를 어떤 순서로 보여줄지'가 핵심 기술이 되었고, RAG와 메모리로 회사 자료를 끌어 쓰는 방식이 표준으로 자리 잡았습니다.
정보 공급 메모리+ RAG→ 컨텍스트 엔지니어링

27
메모리Memory★★★

AI가 이전 대화나 작업 상태를 기억하는 저장소입니다. 단발 대화에서 벗어나 며칠·몇 주에 걸친 협업을 가능하게 만드는 핵심 인프라입니다.

AI의 메모장

28
메모리 계층Memory Hierarchy★

즉시 접근하는 단기 기억과 검색이 필요한 장기 기억으로 나눈 구조입니다. CPU의 캐시·메모리·디스크 계층과 같은 원리로 작동합니다.

빠른 포스트잇 vs 서류 캐비닛

29
RAGRetrieval-Augmented Generation★★★

외부 문서를 검색해서 AI의 컨텍스트에 실시간으로 주입하는 기법입니다. 임베딩으로 회사 자료에서 관련 부분만 골라 와서, 할루시네이션을 줄이는 표준 방법이 되었습니다.

시험 때 교과서 찾아보며 답 쓰는 것

30
컨텍스트 엔지니어링Context Engineering★★★

어떤 정보를 어떤 순서로 AI에게 전달할지 짜는 방법론입니다. CLAUDE.md 같은 운영 문서 작성도 여기에 포함되며, 프롬프트 엔지니어링의 다음 세대로 자리 잡았습니다.

AI의 책상 위를 정리하는 법

31
인텐트Intent★

사용자가 진짜 달성하려는 목표입니다. 에이전트가 어떤 도구를 골라야 할지, 어떤 정보를 더 끌어와야 할지 판단하는 기준점이 됩니다.

'진짜로 원하는 게 뭔데?' AI가 읽는 속마음

32
프로그레시브 디스클로저Progressive Disclosure★

모든 지시를 한 번에 주지 않고 상황에 맞춰 필요한 것만 꺼내 쓰는 패턴입니다. 컨텍스트 윈도우를 효율적으로 활용하는 핵심 전략으로 자리 잡았습니다.

필요한 정보만 그때그때 꺼내주는 것

5막: 에이전트의 등장 · 2024–2025 ▼
AI가 도구를 잡고 혼자 일하면 무엇이 바뀌나
2024년부터 에이전트가 도구를 직접 잡고 일을 끝내기 시작했습니다. 혼자서 부족하면 여럿이 협업했고, 한 세션을 넘어 며칠씩 이어 일하는 구조가 등장했으며, 사람은 코드 한 줄 쓰지 않고 결과만 받게 되었습니다.
분업 실행 사용자→ 오케스트레이터→ 서브에이전트 + 도구

핵심 능력

33
에이전트Agent★★★

목표를 부여받으면 스스로 판단하고, 도구를 선택하고, 결과를 검증하는 자율적 AI입니다. 단순 챗봇과 달리 여러 단계를 스스로 수행합니다.

시키면 알아서 해오는 AI 비서

34
에이전틱 코딩Agentic Coding★★★

사람이 코드를 한 줄씩 짜는 대신, 에이전트가 자율적으로 코드를 작성·수정·테스트하는 개발 방식입니다. Claude Code·Cursor와 함께 폭발적으로 확산되었습니다.

AI에게 '이거 만들어' 하면 코드가 나오는 것

35
도구 사용Tool Use★★

AI가 텍스트 생성을 넘어 함수 호출·파일 조작·웹 검색 같은 외부 도구를 직접 실행하는 능력입니다. 에이전트의 핵심 역량입니다.

AI가 직접 도구를 집어 들고 쓰는 것

36
스킬Skill★★

반복 사용할 수 있는 전문 지시 묶음입니다. Claude Code에서는 슬래시 명령으로 호출하는 능력 단위가 됩니다.

AI에게 가르쳐둔 '특기' 하나

분업 구조

37
오케스트레이터Orchestrator★★

여러 에이전트의 작업 순서를 정하고 결과를 통합하는 상위 조율자입니다. 오케스트라 지휘자처럼 누가 언제 무엇을 할지 결정합니다.

여러 AI를 지휘하는 지휘자

38
서브에이전트Subagent★★

메인 에이전트가 복잡한 작업을 쪼개서 맡기는 하위 에이전트입니다. 팀장이 팀원에게 업무를 배분하는 구조와 같습니다.

팀장 AI가 부르는 팀원 AI

39
핸드오프Handoff★

에이전트 사이, 또는 에이전트와 사람 사이에서 작업을 넘기는 인계 지점입니다.

업무 인수인계, '여기서부턴 네가 해'

실행 패턴

40
워크플로우Workflow★★★

작업의 순서와 조건을 미리 정의한 실행 절차입니다. 에이전트가 반복 작업을 자동으로 처리할 때의 청사진이 됩니다.

자동화 레시피, 순서대로 따라가면 완성

41
TDDTest-Driven Development★★

테스트를 먼저 작성하고, 그 테스트를 통과시킬 만큼만 코드를 짜는 개발 방식입니다. 에이전트가 자기 결과를 검증하는 피드백 루프의 기본 패턴으로 자리 잡았습니다.

시험 문제를 먼저 만들고 답을 푸는 방식

42
SDDSpec-Driven Development★★

사양 문서를 먼저 명확히 한 뒤, 그에 맞춰 코드를 구현하는 개발 방식입니다. CLAUDE.md 같은 운영 문서를 출발점으로 두는 에이전틱 코딩의 표준 시작점이 되었습니다.

설계도 먼저 정리하고 짓기

43
그래프Graph / DAG★

작업 사이의 의존 관계를 노드와 엣지로 표현한 구조입니다. 어떤 작업이 먼저 끝나야 다음으로 넘어갈 수 있는지를 정의합니다.

작업 순서를 화살표로 그린 지도

44
스웜Swarm★

여러 에이전트가 각자의 독립된 컨텍스트에서 동시에 작업하는 병렬 실행 구조입니다.

여러 AI가 동시에 따로 일하는 것

45
플릿Fleet★

운영 중인 에이전트들의 집합입니다. 개별 에이전트가 아니라 전체를 묶어서 모니터링하고 관리할 때 쓰는 표현입니다.

운영 중인 AI 팀 전체 명단

운영 변형

46
백그라운드 에이전트Background Agent★★

사용자 개입 없이 백그라운드에서 비동기로 도는 에이전트입니다. 야간 코드 리뷰나 자동 테스트에서 진가가 드러납니다.

밤새 혼자 일하는 AI 야근러

47
장기 실행 에이전트Long-Running Agent★★

한 번의 컨텍스트 윈도우를 넘어, 여러 세션에 걸쳐 작업을 이어가는 에이전트입니다.

며칠째 이어서 일하는 AI

48
REPLRead-Eval-Print Loop★

코드를 입력하면 즉시 결과를 보여주는 대화형 실행 환경입니다. 에이전트가 코드를 시험하고 검증하는 단계에서 핵심 도구로 다시 부각되었습니다.

코드를 치면 바로 결과가 나오는 대화창

49
오픈클로OpenClaw★

2025년 11월 오스트리아의 페터 슈타인베르거(Peter Steinberger)가 공개한 오픈소스 AI 에이전트입니다. 파일·터미널·브라우저를 자율적으로 제어해 '손이 달린 클로드'로 불리며, 카카오 PlayMCP가 연동을 지원합니다.

'손이 달린 클로드', 파일·터미널·브라우저를 직접 다룬다

50
Hermes AgentHermes Agent · Nous Research★

2026년 2월 Nous Research가 공개한 오픈소스 자율 에이전트입니다. 200개 이상 모델과 40개 이상 도구를 지원하며, 사용자 환경에서 스킬을 만들고 기억을 이어가는 자기 개선형 구조가 특징입니다.

Nous Research가 만든 자기 학습형 자율 AI 에이전트

6막: 하네스와 운영 · 2025– ▼
자율 AI를 회사에서 굴리려면 무엇이 필요한가
자율 AI를 회사에 들이려면 신뢰가 먼저였습니다. 하네스로 에이전트의 행동 범위를 묶고, 가드레일과 샌드박스로 위험을 차단하며, MCP라는 표준이 도구와 데이터를 잇는 시대가 열렸습니다.
안전망 + 표준 하네스+ 가드레일·샌드박스→ MCP
꼭 알아야 할 3선
51
하네스Harness★★★

에이전트의 시작부터 종료까지 전 과정을 감싸는 운영 틀입니다. 테스트 하네스에서 빌려온 개념으로, AI 시대의 핵심 운영 인프라가 되었습니다.

AI를 감싸는 운영 울타리

52
하네스 엔지니어링Harness Engineering★

에이전트에 제약을 걸고 피드백 루프를 짜는 운영 기술입니다. 대규모 코드 생성 사례에서 핵심 역량으로 부상하고 있습니다.

AI에게 규칙과 제한을 거는 기술

53
가드레일Guardrail★★★

위험한 출력이 나오기 전에 차단하는 규칙과 필터입니다. 고속도로 가드레일처럼 에이전트의 이탈을 미리 막아 줍니다.

위험한 방향으로 빠지지 않게 막는 난간

54
샌드박스Sandbox★★★

외부에 영향을 줄 수 없는 격리된 실행 환경입니다. Codex가 Docker 컨테이너 안에서 코드를 돌리는 것이 대표 사례입니다.

실수해도 피해가 없는 연습 공간

55
후크Hook★

특정 동작이 실행되기 직전이나 직후에 자동으로 끼어드는 커스텀 코드입니다. Git의 pre-commit hook처럼 에이전트의 행동을 가로채 검증하거나 변환합니다.

실행 직전에 자동으로 끼어드는 검문소

56
트레이스Trace★★

에이전트가 어떤 판단을 거쳤는지 기록한 실행 로그입니다. 문제가 생겼을 때 원인을 추적하는 핵심 자료가 됩니다.

AI의 행동 녹화 기록

57
디프Diff★★

코드 변경 전후를 나란히 비교하는 출력입니다. 에이전트 결과물의 가장 빠른 1차 검증 수단입니다.

뭐가 바뀌었는지 한눈에 보는 비교표

58
MCPModel Context Protocol★★★

AI 모델과 외부 도구를 연결하는 개방형 프로토콜입니다. M×N개의 커스텀 연동을 M+N으로 줄여 주는 표준 어댑터입니다.

AI와 도구를 잇는 만능 어댑터

59
ACPAgent Communication Protocol★

코드 에디터와 AI 에이전트 사이의 통신 규격입니다. Zed와 JetBrains가 주도하고 있으며, LSP의 사고방식을 에이전트 시대로 옮겨 온 표준입니다.

에디터가 AI와 대화하는 전용 회선

60
랄프 루프Ralph Loop★

매 반복마다 새 세션을 열되, git 기록과 진행 파일로 맥락을 이어가는 자율 코딩 루프입니다. Geoffrey Huntley가 공개한 패턴으로 알려졌습니다.

매번 새로 시작하되 일지로 이어가는 패턴
