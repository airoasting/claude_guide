Claude Code 개발 환경 코드 품질 Git & 버전 관리

Claude Code

프롬프트 Prompt

Claude에게 보내는 요청이나 지시 메시지입니다. 프롬프트를 잘 쓸수록 원하는 결과를 얻기 쉽습니다.

컨텍스트 Context

Claude가 현재 대화에서 기억하고 있는 정보의 범위입니다. 대화가 길어지면 자동으로 압축됩니다.

대화의 "기억 용량"이라고 생각하면 됩니다.

토큰 Token

AI가 텍스트를 처리하는 최소 단위입니다. 한글 한 글자는 약 2~3 토큰에 해당합니다.

글자 수를 세는 AI만의 단위입니다. 토큰이 많을수록 비용이 올라갑니다.

에이전트 Agent

여러 단계의 작업을 스스로 판단하고 수행하는 AI입니다. Claude Code 자체가 에이전트 역할을 합니다.

지시 하나로 알아서 여러 단계를 처리하는 비서와 비슷합니다.

서브에이전트 Sub-Agent

메인 에이전트가 복잡한 작업을 나눠서 처리하기 위해 생성하는 독립적인 하위 작업자입니다.

팀장이 팀원에게 세부 업무를 분배하는 것과 같습니다.

CLAUDE.md

프로젝트의 규칙과 맥락을 Claude에게 미리 알려주는 설정 파일입니다. 프로젝트 폴더에 넣어두면 자동으로 읽습니다.

새 직원에게 건네는 "업무 매뉴얼"이라고 생각하면 됩니다.

후크 Hook

특정 이벤트가 발생할 때 자동으로 실행되는 명령어입니다. 파일 저장 시 자동 정리 같은 용도로 씁니다.

"문이 열리면 자동으로 불이 켜지는" 센서와 비슷합니다.

스킬 Skill / Custom Command

자주 쓰는 작업을 마크다운 파일로 저장해두고 슬래시 명령어로 빠르게 실행하는 기능입니다.

스마트폰의 "단축어" 앱처럼 복잡한 작업을 한 번에 실행하는 매크로입니다.

개발 환경

터미널 Terminal / CLI (Command Line Interface)

텍스트 명령어를 입력해서 컴퓨터를 조작하는 화면입니다. CLI는 Command Line Interface의 약자로, 마우스 클릭 대신 텍스트 명령어로 컴퓨터와 소통하는 방식을 뜻합니다. Claude Code가 실행되는 곳입니다.

마우스 대신 키보드로 컴퓨터에 지시를 내리는 창이라고 생각하면 됩니다.

Node.js 노드

JavaScript를 컴퓨터에서 실행할 수 있게 해주는 프로그램입니다. Claude Code 설치에 필요합니다.

한글 파일을 열려면 한컴오피스가 필요하듯, JS 코드를 실행하려면 Node.js가 필요합니다.

npm Node Package Manager

다른 개발자가 만든 도구(패키지)를 설치하고 관리하는 프로그램입니다. Node.js와 함께 설치됩니다.

앱스토어처럼 필요한 도구를 검색하고 설치하는 곳입니다.

API Application Programming Interface

프로그램끼리 데이터를 주고받기 위해 정해놓은 규칙입니다.

식당의 메뉴판과 같습니다. 정해진 메뉴(요청)를 주문하면 정해진 음식(응답)이 나옵니다.

프론트엔드 Frontend

사용자 눈에 보이는 화면을 만드는 영역입니다. 웹페이지 레이아웃, 버튼, 색상 등이 해당됩니다.

백엔드 Backend

눈에 보이지 않는 서버 쪽 로직을 담당하는 영역입니다. 데이터 저장, 로그인 처리 등이 해당됩니다.

식당에 비유하면 프론트엔드는 홀, 백엔드는 주방입니다.

코드 품질

린트 Lint

코드에서 잠재적 오류나 스타일 문제를 자동으로 찾아주는 도구입니다.

문서의 맞춤법 검사기와 같은 역할입니다.

포맷팅 Formatting

코드의 들여쓰기, 줄바꿈, 공백 등을 일관된 스타일로 자동 정리하는 것입니다.

글의 문단 정렬과 줄 간격을 깔끔하게 맞추는 것과 같습니다.

리팩토링 Refactoring

동작은 그대로 유지하면서 코드의 구조를 더 깔끔하게 개선하는 작업입니다.

방 안의 물건은 그대로인데 정리 정돈을 다시 하는 것과 같습니다.

디버깅 Debugging

코드에서 오류(버그)의 원인을 찾아 수정하는 과정입니다.

기계가 고장났을 때 어디가 문제인지 하나씩 점검하는 것과 같습니다.

Git & 버전 관리

Git 깃

코드의 변경 이력을 기록하고 관리하는 도구입니다. 누가 언제 무엇을 바꿨는지 추적할 수 있습니다.

문서의 "변경 내역 추적" 기능과 비슷합니다.

커밋 Commit

변경한 코드를 하나의 버전으로 저장하는 것입니다. "이 시점의 상태를 기록한다"는 의미입니다.

문서를 "다른 이름으로 저장"하는 것과 비슷합니다.

리포지토리 Repository

프로젝트의 모든 파일과 변경 이력이 저장되는 공간입니다. 줄여서 "레포"라고 부릅니다.

프로젝트 전체가 들어있는 폴더라고 생각하면 됩니다.

브랜치 Branch

원본 코드에 영향을 주지 않고 별도로 작업할 수 있는 독립 공간입니다.

문서 원본은 그대로 두고 "사본"을 만들어 수정하는 것과 비슷합니다.

PR Pull Request

내가 수정한 코드를 팀에 검토해달라고 요청하는 것입니다. 승인되면 원본 코드에 반영됩니다.

"이렇게 바꿨는데 괜찮은지 확인해주세요"라는 공식 요청서입니다.

머지 Merge

분리된 브랜치의 변경사항을 원본 코드에 합치는 작업입니다.

수정한 사본의 내용을 원본 문서에 반영하는 것과 같습니다.

Worktree 워크트리

하나의 리포지토리에서 여러 브랜치를 동시에 열어놓고 작업할 수 있는 기능입니다.

같은 문서의 여러 버전을 각각 다른 창에 띄워놓고 동시에 편집하는 것과 비슷합니다.
