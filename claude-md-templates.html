# 프로젝트: 주간 브리핑

## 이 프로젝트는
매주 월요일 아침, 대표님께 드리는 1페이지 주간 보고서를 만듭니다.
원자료: 영업·재무·운영 3개 팀의 주간 보고

## 답변·문서 규칙
- 모든 출력은 ~합니다 체로 통일할 것
- 보고서는 항상 1페이지 안에 끝낼 것
- 숫자는 전주 대비 증감(%, 절대값)을 항상 함께 표기할 것
- 전문 용어를 쓸 때는 괄호로 풀어쓸 것

## 출력 포맷 (대표 보고용 1페이지)
1. 한 줄 요약 (이번 주 핵심)
2. 핵심 3가지 (영업/재무/운영 각 1줄)
3. 빨간 신호 (지난주 대비 악화된 지표)
4. 다음 주 액션 아이템 3가지

## 절대 하지 마세요
- 미공개 매출·이익 수치를 추정해서 채워넣지 말 것 (없으면 '미집계'로 표기)
- 인사 평가·연봉 데이터는 절대 인용하지 말 것
- 검증되지 않은 시장 예측은 보고서에 넣지 말 것
- 대표님 결정이 필요한 사안에 임의로 결론을 내리지 말 것

## 자주 쓰는 자료 위치
- 영업 주간 보고: data/sales-weekly/
- 재무 주간 보고: data/finance-weekly/
- 운영 KPI: data/ops-kpi/

## 주의사항
- 사내 보고용 자료입니다. 외부(고객·언론·투자자) 공유 금지
- 매주 월요일 오전 8시 전까지 완성되어야 합니다
월 월요일: 오늘도 첫 출근입니다, 처음 뵙겠습니다
Day 1 · 월요일
"분명 어제 알려드렸는데요…"

오늘 새로 출근한 신입 인턴 클로드. 일은 빠르고 똑똑한데 딱 한 가지 문제가 있습니다. 매일 출근하면 어제 일을 전부 잊어버립니다.

오전 10시, 코드 작업을 시켰더니 한국어로 부탁한 주석이 영어로 돌아왔습니다. 어제도 한국어로 적어달라 말씀드렸는데 말이죠. 11시, 정리한답시고 .env 파일을 지웠습니다. 그저께도 건드리지 말라 했는데 말입니다.

점심을 먹고 돌아온 사수는 그제야 깨달았습니다. 이 친구한테 매일 같은 말을 반복할 게 아니라, 책상 위에 매뉴얼 한 장을 올려둬야겠다고요.

사수 메모: 클로드는 매 세션마다 컨텍스트가 리셋됩니다. 매번 말로 알려주는 대신 CLAUDE.md 한 장이면 충분합니다.
📌 .env란? 비밀번호, API 키, 데이터베이스 접속 정보 같은 민감한 설정값을 따로 모아두는 파일입니다. 이게 사라지면 외부 서비스 연결이 끊겨서 프로그램이 멈춥니다. 그래서 .env는 절대 건드리면 안 되는 1순위 파일로 꼽힙니다.

그래서 CLAUDE.md는 인턴 클로드의 책상 위 메모지입니다. 프로젝트 폴더에 넣어두면 클로드가 매 출근(=세션)마다 자동으로 읽고 "아, 이 프로젝트에선 이렇게 일해야 하는구나"를 기억합니다.

📁 어디에 두나요? (적용 범위 두 가지)
프로젝트용
이 프로젝트에만 적용
특정 프로젝트 폴더 안에 두는 규칙입니다. 해당 프로젝트에서 Claude를 실행할 때만 읽힙니다.
프로젝트폴더/CLAUDE.md
글로벌 (전체 적용)
모든 프로젝트에 적용
"항상 한국어로 답해줘" 같은 나만의 기본 설정입니다. 어느 프로젝트에서나 공통으로 적용됩니다.
~/.claude/CLAUDE.md
🌐 글로벌 파일은 OS별로 위치가 다릅니다
🍎 macOS
/Users/사용자이름/.claude/CLAUDE.md
🪟 Windows
C:\Users\사용자이름\.claude\CLAUDE.md

~는 내 홈 폴더를 가리키는 약어입니다. .claude는 점(.)으로 시작해서 평소엔 숨김 폴더로 보이지 않습니다. macOS Finder에서는 Cmd + Shift + . 키로 보이게 할 수 있습니다.

📊 CLAUDE.md가 있을 때 vs 없을 때

CLAUDE.md 없음
  • 답변에 영어 전문 용어가 그대로 섞여 나옴
  • 코드 수정 이유를 설명 안 해줌
  • 중요한 파일을 실수로 건드림
  • 매번 같은 규칙을 다시 말해야 함
CLAUDE.md 있음
  • 전문 용어를 쉬운 말로 풀어서 함께 설명
  • 변경 사항마다 이유를 설명해줌
  • 보호 파일은 자동으로 건드리지 않음
  • 규칙을 기억하고 일관되게 작업
화 화요일: 사수가 책상에 메모 한 장을 두고 갔습니다
Day 2 · 화요일
"이 메모대로만 일하면 됩니다"

화요일 아침. 어김없이 어제를 잊은 채 출근한 클로드. 그런데 책상 위에 처음 보는 종이 한 장이 놓여 있습니다.

"이 프로젝트는 이런 일을 합니다 / 답변은 한국어로 / .env는 절대 건드리지 마세요 / 코드를 바꿨다면 왜 바꿨는지 한 줄로 설명할 것…" 사수가 새벽에 정리해 둔 오늘의 업무 매뉴얼이었습니다.

클로드는 메모를 한 번 훑고 일을 시작했습니다. 한국어로 답하고, .env에 손대지 않고, 코드 바꿀 때마다 이유를 적었습니다. 매일 잊어도 괜찮습니다. 메모는 매일 그 자리에 있으니까요.

사수 메모: 메모는 두 종류로 준비해두면 충분합니다. 비즈니스 리더용 간단 버전과 개발자용 풀버전. 아래에서 골라 [대괄호]만 바꾸세요.

코드를 몰라도 바로 쓸 수 있는 간단 버전. [대괄호] 안만 바꾸세요

수 수요일: 메모도 잘 쓰는 법이 따로 있더군요
Day 3 · 수요일
"메모를 줬는데도 이상하게 일을 못 합니다"

이틀째 메모를 받고 일하던 클로드. 그런데 사수는 묘한 답답함을 느꼈습니다. 분명 메모를 줬는데도 결과물이 매번 살짝 다르게 돌아왔거든요.

"코드 정리해줘"라고 적힌 메모를 본 클로드는 변수명을 바꾸고, 함수를 쪼개고, 폴더 구조까지 갈아엎었습니다. 사수가 원한 건 단지 주석 한 줄 추가였는데 말입니다.

그제야 사수는 깨달았습니다. 메모는 종이 한 장으로 끝나는 게 아니라, 어떻게 적느냐가 절반이라는 것을요. 구체적으로, 짧게, 자주 하는 실수를 콕 짚어서. 메모 잘 쓰는 노하우 8가지를 정리했습니다.

사수 메모: 같은 메모도 표현 한 끗 차이로 결과물이 달라집니다. 아래 8가지 원칙만 지켜도 메모의 효과가 두 배가 됩니다.
  • 프로젝트 시작할 때 /init 먼저 실행. Claude가 프로젝트를 분석해 CLAUDE.md 초안을 자동으로 만들어줌
  • 규칙은 구체적으로. "코드 정리해줘"보다 "함수마다 설명 주석 추가해줘"
  • 자주 하는 실수 적어두기. Claude가 같은 실수 반복 안 함
  • 명령어 정리해두기. Claude가 빌드/테스트 알아서 실행
  • CLAUDE.md는 300줄 안팎으로 짧게 유지. 길어지면 클로드가 핵심을 놓치고 매 세션 토큰도 낭비됨
  • 300줄을 넘으면 파일을 쪼갤 것. 메인 CLAUDE.md에서 @docs/tone-guide.md 같은 식으로 다른 파일을 끌어와 한 장처럼 사용 가능
  • .claudeignore 파일로 Claude가 읽지 않을 파일/폴더 지정 가능
  • CLAUDE.md에서 파일 경로를 적으면 Claude가 자동으로 찾아봄 (예: docs/api-spec.md를 참고할 것)
  • [대괄호] 안의 내용은 본인 프로젝트에 맞게 바꿔 쓰세요
목 목요일: 매번 새로 쓰지 않습니다, 만드는 방법이 3가지
Day 4 · 목요일
"사수님, 매번 새벽에 메모 쓰시는 거 힘들지 않으세요?"

이 무렵 사수도 슬슬 지치고 있었습니다. 프로젝트마다 매번 책상 메모를 처음부터 적자니 반나절이 훌쩍 갔거든요. 그러던 중 옆자리 선배가 한마디 했습니다.

"메모, 매번 직접 쓰지 마세요. 방법이 세 가지나 있어요."

① 클로드한테 직접 시키면 알아서 초안을 만들어줍니다. ② 이미 검증된 템플릿을 복사해서 [대괄호]만 바꾸면 끝입니다. ③ 일하다가 새 규칙이 필요해지면 그때그때 클로드에게 추가만 부탁하면 됩니다.

사수 메모: 메모 작성에 30분 이상 쓰지 마세요. 아래 3가지 방법 중 상황에 맞는 걸 고르면 5분이면 충분합니다.

1 새 프로젝트: /init으로 자동 생성

프로젝트 폴더에서 Claude를 실행한 뒤 /init을 입력하면 Claude가 프로젝트 구조를 분석하고 CLAUDE.md 초안을 직접 만들어줍니다.

cd my-project claude > /init
✓ /init 실행 결과

Claude가 폴더 구조, 파일 유형, 기존 설정을 읽고 "이 프로젝트에 맞는 CLAUDE.md를 작성했습니다"라고 응답합니다. 생성된 파일을 열어 내용을 확인하고 필요한 부분만 수정하면 됩니다.

2 기존 프로젝트: 템플릿 붙여넣기

이미 진행 중인 프로젝트라면 위 템플릿을 복사해 프로젝트 폴더에 CLAUDE.md 파일로 저장하세요.

# 프로젝트 폴더에서 파일 생성 touch CLAUDE.md # 그 다음 위 템플릿을 붙여넣고 [대괄호] 내용을 수정

3 Claude에게 규칙 추가 요청

작업 중 새로운 규칙이 필요하면 Claude에게 직접 요청하면 됩니다.

> CLAUDE.md에 다음 규칙 추가해줘: > - 모든 API 응답에 타임스탬프 포함
금 금요일: 1주일을 돌아보니, 가장 자주 받은 메모 5장
Day 5 · 금요일
"이 다섯 줄만 있어도 일주일이 평화롭습니다"

금요일 오후. 사수가 이번 주 메모를 한 장씩 다시 꺼내봤습니다. 어떤 메모는 한 번 적고 잊혔고, 어떤 메모는 매일 클로드 책상에 다시 올려두었습니다.

가장 자주 다시 적은 메모 다섯 장. 이 다섯 줄을 적어두기 시작한 뒤로는 같은 사고가 거짓말처럼 사라졌습니다. 비즈니스 리더라면 코드를 한 줄도 몰라도 이 다섯 장만 책상 위에 두면 됩니다.

사수 메모: 처음 시작하는 분이라면 아래 텍스트만 복사해서 CLAUDE.md에 붙여넣으세요. 그것만으로도 7할은 해결됩니다.
# 1. 한국어로 답변
모든 답변과 주석은 한국어로 작성할 것
em dash 쓰지 말 것
자연스러운 한국어로, 주술 구조 맞추고 번역투 쓰지 말 것

# 2. 코드 설명 의무화
코드를 수정했다면 왜 바꿨는지 한 줄로 설명할 것

# 3. 확인 후 수정
데이터베이스 구조 변경 전 반드시 사용자 확인 받을 것

# 4. 파일 보호
.env 파일은 절대 수정하지 말 것

# 5. 스타일 유지
기존 코드 스타일과 동일한 패턴을 유지할 것
