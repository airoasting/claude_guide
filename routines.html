루틴이란

정해진 시각에, 사람 없이 알아서 도는 에이전트입니다

Routines는 일정에 맞춰 무인으로 실행되는 예약 에이전트입니다. "매일 아침 9시에 어젯밤 뉴스를 정리해 줘"처럼 시각과 할 일을 한 번 등록해 두면, 그 시각이 될 때마다 Claude가 알아서 작업을 수행합니다. 프롬프트, 저장소 접근, 연결한 도구를 한 번 묶어 두면 그다음부터는 스스로 돕니다.

⏰

정해진 일정

매시·매일·평일·매주 또는 특정 시각에 자동으로 시작합니다.

매주 월요일 08:00

무인 자동

사람이 매번 시작 버튼을 누르지 않아도 알아서 돌고 결과를 남깁니다.

한 번 예약하면 내가 자는 동안에도 일이 끝나 있습니다.
반복 작업을 사람의 손에서 완전히 떼어 내는 단계입니다.

언제 생겼나

Routines는 2026년 4월 14일 클로드 코드에 리서치 프리뷰로 처음 도입됐습니다. 유료 플랜(Pro·Max·Team·Enterprise)에서 쓸 수 있습니다.

시작하는 곳

클로드 데스크탑 앱에서 메뉴 하나로 시작합니다

클로드 데스크탑 앱을 열고 Code 탭으로 들어가면, 왼쪽 메뉴에 Routines가 있습니다. 이 메뉴를 클릭하면 예약을 만들고 관리하는 화면이 열립니다.

로컬과 원격

내 컴퓨터에서 돌릴지, 클라우드에서 돌릴지 고릅니다

루틴을 만들 때 실행 장소를 로컬과 원격 중에서 정합니다. 둘의 차이는 "내 기기가 켜져 있어야 하는가"입니다.

로컬 · 내 컴퓨터

내 기기에서 직접
  • 클로드 코드를 켜 둔 내 컴퓨터에서 실행
  • 기기가 켜져 있고 세션이 떠 있어야 함
  • 내 환경·파일을 그대로 쓰며 직접 지켜볼 때

원격 · 클라우드

Anthropic 클라우드에서
  • 내 노트북이 꺼져 있어도 실행
  • 트리거되면 클라우드가 컨테이너를 띄워 끝까지 수행
  • 결과는 산출물로 남기거나 웹훅으로 전달
고르는 기준

내 컴퓨터의 로컬 환경이나 켜 둔 세션이 꼭 필요하면 로컬, 항상 켜둘 서버나 직접 만든 크론 없이 알아서 돌게 하려면 원격입니다. 무인 자동화의 진가는 원격에서 나옵니다.

만드는 법

언제 돌릴지와 무엇을 시킬지, 두 가지만 정합니다

루틴은 일정과 작업 두 가지로 만듭니다. 일정은 "매일 오전 9시"처럼 말로 적어도 되고, 한 번만 돌리는 1회성 예약도 됩니다. 시작 방식은 세 가지입니다. 정해진 일정, 외부에서 부르는 API 호출, 그리고 깃허브 이벤트(PR·푸시 등)입니다.

루틴 예약 예시 
매일 오전 9시   # 일정 트리거
   어젯밤 들어온 주요 뉴스를 5건으로 정리해 메일로 보내 줘

매주 월요일 08:00   # 반복 일정
   지난주 매출 데이터를 요약해 한 장짜리 리포트로 만들어 줘

내일 오후 3시 한 번만   # 1회성 예약
   회의 전에 이 문서의 최신 버전을 점검해서 알려 줘

정할 것은 둘뿐입니다. 언제(일정) 그리고 무엇을(작업).
일정은 매시·매일·평일·매주 같은 주기로 걸 수 있습니다.

관리하기

예약을 보고, 고치고, 멈춥니다

한 번 만든 루틴은 목록으로 확인하고 언제든 수정하거나 멈출 수 있습니다. 무인으로 돌기 때문에 처음 설계할 때 안전장치를 함께 거는 것이 중요합니다.

이렇게 관리합니다

  • 예약 목록 보기 · 다음 실행 시각 확인
  • 일정·작업 내용 수정
  • 잠시 끄기 또는 완전히 삭제
  • 지금 한 번 바로 실행해 결과 점검

안전하게 거는 법

  • 되돌릴 수 없는 동작은 자동 실행 금지
  • 발송·결제·삭제는 초안까지만, 승인은 사람이
  • 결과를 받아 볼 곳을 분명히 지정
  • 처음엔 1회성으로 시험한 뒤 반복으로 전환
루프 엔지니어링(/loop)과 무엇이 다릅니까?
루프는 지금 열어 둔 세션 안에서 도는 반복이라 세션을 닫으면 멈춥니다. Routines는 일정에 맞춰 예약해 두는 것이라, 원격으로 걸면 내가 접속해 있지 않아도 정해진 시각에 무인으로 실행됩니다.
결과는 어디서 봅니까?
예약을 만들 때 받아 볼 곳을 지정합니다. 메일로 보내거나, 파일로 남기거나, 메신저로 알리도록 작업에 적어 두면 그곳으로 결과가 전달됩니다.
메일 발송이나 결제까지 자동으로 시켜도 됩니까?
권하지 않습니다. 무인으로 도는 만큼, 되돌리기 어려운 동작은 초안까지만 맡기고 발송·결제·삭제 같은 최종 실행은 사람 승인을 사이에 두는 것이 원칙입니다.
