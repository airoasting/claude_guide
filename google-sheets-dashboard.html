STEP 1 · Claude Cowork에 Google Workspace 연결

Cowork의 Google Workspace 커넥터 활성화

Claude Cowork는 Google Drive · Sheets · Calendar · Gmail을 별도 API 없이 한 번에 연결합니다

Claude Cowork에는 Google Workspace 커넥터가 기본으로 준비되어 있습니다. 사용자의 구글 계정을 한 번만 인증하면 Cowork 세션 안에서 시트의 셀 데이터를 바로 읽을 수 있습니다. 별도의 API 키나 서비스 계정은 필요하지 않습니다.

1
Claude Desktop 실행 후 Cowork 진입
Claude Desktop(macOS/Windows 데스크톱 앱)을 실행하고, 새 대화의 모델/도구 선택 영역에서 Cowork를 켭니다.
2
커넥터 설정 열기
Claude Desktop의 Customize > Connectors로 들어가면 사용 가능한 커넥터 목록이 보입니다. 거기에서 Google Workspace 항목을 찾습니다.
3
Google 계정 인증
Connect 버튼을 누르면 구글 OAuth 화면이 열립니다. 분석할 시트가 있는 계정으로 로그인하고, Drive · Sheets 읽기 권한을 승인합니다.
4
연결 확인
커넥터 옆에 Connected 표시가 뜨면 완료입니다. 이제 Cowork 대화창에서 내 드라이브의 시트 URL을 그대로 붙여넣고 분석을 요청할 수 있습니다.
TIP · 회사 계정과 개인 계정 분리

회사 워크스페이스 계정과 개인 구글 계정을 함께 쓰는 경우, Cowork 커넥터는 한 번에 한 계정만 인증할 수 있습니다. 분석 대상 시트가 있는 쪽 계정으로 연결하세요. 계정을 바꿀 때는 Disconnect를 누른 다음 다시 Connect하는 방식이 가장 안전합니다.

STEP 2 · 예제 시트를 내 드라이브로 복사

예제 시트를 내 드라이브에 복사하기

Cowork 커넥터는 내 드라이브 안의 파일을 읽습니다. 따라서 공용 예제 시트를 먼저 내 드라이브로 복사합니다

Cowork 커넥터가 읽을 수 있는 시트는 커넥터에 연결한 구글 계정의 드라이브에 들어 있는 파일입니다. 공용 예제 시트를 그대로 쓰면 권한 문제가 생기기 쉽기 때문에, 먼저 내 드라이브로 사본을 만들어 둡니다. 사본을 한 번 만들어 두면 다음부터는 그 시트만 갱신하면 됩니다.

1
예제 시트 열기
아래 링크로 예제 시트를 브라우저에서 엽니다. 이 시트는 누구나 뷰어로 접근할 수 있도록 열어 두었습니다.
2
사본 만들어 내 드라이브에 저장
상단 메뉴에서 파일 > 사본 만들기를 선택합니다. 저장 위치는 내 드라이브(혹은 원하는 폴더)로 지정하고, 파일 이름은 기본값을 그대로 둬도 좋습니다.
3
사본의 URL 복사
사본이 열리면 주소창의 URL을 복사합니다. gid=... 부분이 붙어 있으면 그대로 두세요. 특정 탭을 가리키는 값입니다.
4
계정 일치 확인
사본을 만든 계정과 STEP 1에서 Cowork에 연결한 구글 계정이 같아야 합니다. 다르면 Cowork가 시트를 찾지 못합니다.
실습용 예제 시트

아래 시트를 열고 파일 > 사본 만들기로 내 드라이브에 옮기세요. STEP 3부터는 사본의 URL을 씁니다.

왜 내 드라이브로 복사부터 하나

Cowork가 "시트에 접근할 수 없습니다"라고 답하는 가장 흔한 이유가 권한 누락입니다. 내 드라이브의 사본은 처음부터 내 권한으로 만들어지기 때문에 권한 문제가 생기지 않습니다. 그 다음부터는 자연어 한 줄로 작업이 끝납니다.

STEP 3 · 탐색적 분석 (EDA) 실행

한 줄 프롬프트로 EDA 시작하기

복잡한 지시는 필요하지 않습니다. 시트 주소와 한 문장만으로 충분합니다

EDA는 거창한 절차가 아닙니다. Cowork 대화창에 내 드라이브의 시트 주소를 붙여 넣고, "이 데이터를 살펴봐 줘" 정도로만 부탁해도 Claude가 알아서 컬럼을 정리하고 결측치와 이상치, 핵심 지표를 찾아 줍니다.

EDA 프롬프트

아래처럼 한 줄로 던지면 됩니다. 시트 URL은 STEP 2에서 복사한 내 드라이브 사본의 주소를 씁니다.

아래 구글 시트를 EDA로 분석해줘. https://docs.google.com/spreadsheets/d/내-시트-ID/edit 어떤 데이터가 들어 있는지, 흥미로운 패턴이 뭔지 편하게 알려줘.
결과로 받게 되는 것

한 줄 요청만으로도 다음이 한 번에 정리됩니다.
· 데이터 개요. 행 수와 컬럼 수, 각 컬럼의 의미와 단위.
· 데이터 품질. 결측치와 이상치 위치.
· 핵심 KPI. 총 매출, 영업이익률, 매장별 순위 같은 요약 지표.
· 인사이트. 사람이 놓치기 쉬운 패턴 3~5가지.

STEP 4 · 글래스모피즘 인터랙티브 대시보드 생성

대시보드 한 장으로 압축하기

EDA에서 잡아낸 인사이트를 글래스모피즘 디자인의 단일 HTML 파일로 패키징합니다

차트를 따로따로 만들어 두면 보는 사람이 흐름을 잃습니다. 모든 인사이트를 한 페이지 안에 모으고 필터로 좁혀 가며 보는 인터랙티브 대시보드로 만들 때 의사결정 속도가 크게 빨라집니다. 디자인은 글래스모피즘(반투명 유리 카드와 컬러풀 그라데이션 배경)을 씁니다. 정보 밀도가 높아도 시각적 피로가 덜합니다.

설계 원칙 · 한 화면에 들어가야 할 것

좋은 대시보드는 다음 4단으로 구성됩니다.
1단 (필터)에서 매장과 기간, 카테고리를 칩(chip)으로 토글합니다.
2단 (KPI 카드 5개)은 매출·영업이익·이익률·주문수·객단가를 보여 줍니다.
3단 (메인 차트)은 큰 화면에 월별 추세를, 작은 화면에 카테고리 믹스를 배치합니다.
4단 (보조 차트)은 매장 랭킹과 비용 구조, 채널·시간대, 품질·고객·리스크를 묶어 보여 줍니다.

대시보드 생성 프롬프트

EDA를 마친 직후 같은 세션에서 이 한 줄을 던집니다. 디자인 톤과 인터랙션 방식 정도만 정해 주면 나머지는 Claude가 알아서 채워 줍니다.

방금 분석한 데이터로 EDA로 탐색 가능한 인터랙티브 대시보드를 만들어줘. - 글래스모피즘 디자인으로 - 매장 · 기간 · 카테고리 필터로 좁혀 볼 수 있게 - 한 화면 안에 핵심 KPI와 차트가 모두 보이게
왜 데이터를 별도 JSON으로 분리하나
  • HTML 파일이 가벼워집니다. 그래서 편집과 배포가 빨라집니다.
  • 월간 자동 갱신을 할 때 JSON만 바꿔치기하면 됩니다.
  • 다른 사람과 공유할 때 데이터 노출 범위를 따로 통제할 수 있습니다.
  • 차트 코드와 데이터가 분리되어 있어 나중에 수정하기가 쉽습니다.
글래스모피즘이 데이터 시각화에 맞는 이유

차트가 많은 대시보드는 시각적 피로가 쉽게 쌓입니다. 글래스모피즘은 배경 그라데이션이 전체 분위기를 잡아 주고, 카드가 반투명으로 떠 있어 정보 위계가 자연스럽게 잡힙니다. 색상이 강한 차트도 카드의 흐릿한 경계 덕분에 서로 충돌하지 않습니다.

STEP 5 · 배포와 월간 자동 갱신

완성한 대시보드를 매달 꾸준히 쓰기

한 번 만든 대시보드가 한 달 안에 잊히지 않도록, 갱신과 공유 흐름까지 함께 설계해 둡니다

대시보드는 만든 순간보다 운영하는 동안 쌓이는 가치가 더 큽니다. 한 번 만들고 끝낼 일이 아니라, 매월 데이터가 갱신될 때 새 버전이 자동으로 올라오도록 같이 설계해 둡니다.

1
로컬에서 동작 확인
생성된 HTML을 브라우저로 열어 보면서 필터·KPI·차트가 모두 정상으로 움직이는지 확인합니다. JSON 경로(상대 경로)와 차트 라이브러리 CDN이 제대로 잡혀 있는지도 같이 점검합니다.
2
GitHub Pages로 무료 배포
생성된 폴더를 GitHub 저장소에 푸시하고 Pages를 켜면 24시간 접근 가능한 URL이 생깁니다. 이 URL을 노션이나 슬랙에 그대로 공유하면 됩니다.
3
월간 갱신 스킬 등록
Cowork에 refresh-dashboard 같은 스킬을 등록해 둡니다. 매월 1일 시트를 다시 읽어 JSON만 새로 써 주면 대시보드가 자동으로 갱신됩니다. 차트 코드는 손대지 않아도 됩니다.
4
의사결정자에게 링크 한 줄
대시보드는 만들었다고 끝이 아닙니다. "매월 첫째 주 월요일, 이 링크 한 번씩만 봐 주세요"까지 같이 약속해 두세요. 그래야 대시보드가 계속 살아 있는 도구로 남습니다.
의사결정자가 실제로 보게 만드는 한 줄

대시보드를 만들어 두고도 아무도 보지 않으면 만든 의미가 없습니다. 만든 다음 날, 의사결정자에게 짧은 한 문장으로 알리세요. "이번 달 가장 큰 변화는 X였습니다. 자세한 내용은 이 링크에서 보실 수 있습니다." 한 번 클릭해 본 페이지는 다음 달에도 다시 들여다보게 됩니다.

RESULT · 결과 미리보기

예제 시트로 만든 완성품

위 5단계를 그대로 따라가 만들어진 실제 글래스모피즘 대시보드입니다

아래 카드를 누르면 새 탭에서 인터랙티브 대시보드가 열립니다. 매장과 연도, 카테고리를 자유롭게 토글해 보세요. 모든 차트와 KPI가 즉시 다시 계산됩니다.

이 실습이 끝나면 다음 네 가지가 손에 남습니다.

  • 재사용 가능한 대시보드 틀. 글래스모피즘 HTML과 JSON을 분리한 구조라서 다른 종류의 데이터에도 그대로 갖다 쓸 수 있습니다.
  • 한 줄짜리 EDA 프롬프트. "이 시트를 EDA로 분석해줘" 한 줄이 앞으로 모든 시트 분석의 출발점이 됩니다.
  • 대시보드 4단 구성 감각. 필터 · KPI · 메인 차트 · 보조 차트로 정리하는 흐름이 앞으로 차트를 고를 때 기준이 됩니다.
  • 매달 가볍게 갱신하는 습관. JSON만 바꿔치기하면 되니까, 대시보드가 매달 살아 있는 도구로 이어집니다.
