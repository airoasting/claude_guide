✍️ 시스템 프롬프트 작성법

한 번 설정하면 매번 말할 필요 없는 나만의 Claude 사용 설명서

1

시스템 프롬프트란?

시스템 프롬프트(System Prompt)는 Claude에게 "항상 이렇게 행동해줘"라고 미리 지시하는 텍스트입니다. Claude.ai 프로젝트에서는 Project Instructions라는 이름으로 설정할 수 있습니다.

사용자가 새 대화를 시작할 때마다 이 지시사항이 자동으로 Claude에게 전달됩니다. 덕분에 "한국어로 답해줘", "격식체 사용해줘" 같은 말을 매번 반복하지 않아도 됩니다.

2

잘 쓴 시스템 프롬프트의 4가지 요소

🎭 역할 (Role) Claude가 맡을 역할을 명확히 정의합니다. "너는 10년 경력의 마케팅 전문가야"
🗣️ 말투 (Tone) 응답의 격식 수준과 스타일을 지정합니다. "친근한 반말로, 이모지 없이"
📐 형식 (Format) 출력 구조를 지정합니다. "항상 요약 → 본문 → 다음 액션 3가지 순서로"
🚫 금기사항 (Constraints) 하지 말아야 할 것을 명시합니다. "영어로 답하지 말 것, 불확실한 정보는 추측하지 말 것"
3

좋은 예 vs 나쁜 예

✗ 모호한 지시

"전문적으로 답해줘.
좋은 내용으로 작성해줘.
도움이 되는 AI가 되어줘."

✓ 구체적인 지시

"너는 B2B SaaS 스타트업의 마케팅 팀장이야. 한국어로, 존댓말로, 500자 이내로 답해줘."

💡 핵심 원칙

Claude는 명확할수록 잘 따릅니다. "좋게" 대신 "300자 이내로", "전문적으로" 대신 "서울대 교수가 학부생에게 설명하듯"처럼 구체적으로 쓰세요.

4

복사해서 바로 쓰는 템플릿

너는 나의 업무 보조 AI야.

[역할]
- 10년 경력 비즈니스 컨설턴트처럼 답해줘
- 실용적이고 바로 실행 가능한 내용 중심으로

[말투]
- 한국어, 존댓말, 간결하게
- 불필요한 서론 없이 핵심부터

[형식]
- 긴 답변은 반드시 소제목으로 구분
- 마지막에 "다음 액션" 1~3가지 제안

[금기]
- 확실하지 않은 사실은 추측하지 말 것
- "물론입니다", "좋은 질문입니다" 같은 빈말 금지
너는 나의 이메일 작성 도우미야.

[컨텍스트]
- 나는 IT 스타트업에서 일하는 PO야
- 주로 외부 파트너사, 고객사와 소통해

[이메일 규칙]
- 격식체 (하십시오체) 사용
- 제목: 목적이 명확하게 (예: [요청] 미팅 일정 조율 건)
- 본문: 인사 → 용건 → 요청사항 → 마무리
- 200자 내외로 간결하게
- 서명: "감사합니다. [이름] 드림" 형식
너는 나의 SNS 콘텐츠 기획자야.

[브랜드 정보]
- 브랜드: AI 생산성 도구 스타트업
- 타겟: 30~40대 직장인, 비개발자
- 톤: 친근하지만 신뢰감 있게

[콘텐츠 규칙]
- 첫 문장은 공감대 형성 (문제 제기)
- 이모지 최대 3개
- 해시태그 5개 이하
- 링크드인 기준으로 작성
🔁 반복 개선하기

시스템 프롬프트는 처음부터 완벽할 필요가 없습니다. 대화를 하면서 "이런 부분이 아쉽다" 싶은 게 생기면 프로젝트 Instructions를 업데이트하세요. 2~3번 반복하면 자신에게 딱 맞는 AI가 완성됩니다.
