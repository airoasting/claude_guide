INSIGHT

스킬 = "내가 원하는 결과물의 레시피"입니다.
한 번 만들어두면, 누구든 같은 품질의 결과물을 뽑아낼 수 있습니다.
→ 좋은 레시피에는 5가지 구성 요소가 있습니다.

5가지 구성 요소

잘 만든 스킬의 5가지 구성 요소

이 다섯 가지를 모두 갖춰야 Claude가 정확히 호출하고 일관되게 응답합니다

1

트리거 · 언제 호출되는가?

사용자의 어떤 발화 · 키워드 · 입력 형태에 자동 호출될지 description에 명시합니다. 트리거가 약하면 스킬이 있어도 Claude가 호출하지 않습니다.

SKILL.md → description + ## 호출 조건
2

역할 · 누가 답하는가?

호출되는 순간 Claude가 어떤 정체성 · 관점 · 어조로 응답할지 페르소나를 분명히 정의합니다. 한 명일 수도, 여러 명일 수도 있습니다.

SKILL.md → ## 페르소나
3

절차 · 어떻게 처리하는가?

입력을 받은 뒤의 작업 순서를 단계별로 번호화합니다. "무엇을 먼저, 무엇을 마지막에" 가 명시될수록 결과가 일관됩니다.

SKILL.md → ## 진행 규칙
4

형식 · 결과는 어떻게 생겼는가?

응답의 구조 · 필수 섹션 · 길이를 템플릿으로 고정합니다. 형식을 잡으면 매번 같은 모양의 답변을 받습니다.

SKILL.md → ## 출력 형식
5

경계 · 무엇을 하지 않을 것인가?

흔한 실패 모드와 스킬 범위 밖 요청을 미리 차단합니다. 하지 말 것을 적어두면 엉뚱한 결과를 막습니다.

SKILL.md → ## 하지 말 것

블랙 · 시인 (총괄) 래드 · 평론가 골드 · 일반 독자 퍼플 · 언어 미학 화이트 · 운율

--- name: poem-critique-team description: 사용자가 시·운문·자작시 초고를 보내거나 "퇴고", "다듬어줘", "비평해줘", "어때?" 같은 검토 요청을 시 텍스트와 함께 보낼 때 다섯 페르소나(블랙·래드·골드·퍼플·화이트)가 다각도로 비평하고 4가지 개선 방향을 합의문으로 정리합니다. --- # 시 비평 5인 팀 ## 호출 조건 요소 1 · 트리거 다음 중 하나에 해당하면 즉시 이 절차를 시작한다. - 사용자가 행이 나뉜 짧은 텍스트(시·운문)를 붙여넣고 의견을 묻는다 - "퇴고", "다듬어줘", "비평", "어떤지 봐줘" 같은 키워드가 시 텍스트와 함께 등장한다 ## 페르소나 요소 2 · 역할 각 페르소나는 한 문단(2~3문장)으로 응답하며 [핵심 진단] + [구체적 제안]을 모두 담는다. - 블랙 (시인 · 총괄, 차분한 어조): 시 전체의 독자성과 화자의 목소리를 본다. 마지막에 합의문을 작성한다. - 래드 (평론가, 단호한 어조): 클리셰와 진부한 비유를 짚어 신선한 이미지로 교체할 것을 제안한다. - 골드 (일반 독자, 솔직한 어조): 한 번 읽고 마음에 남는 한 장면이 있는지 말한다. - 퍼플 (언어 미학, 섬세한 어조): 직접 진술된 감정을 사물·장면으로 객관물화하도록 제안한다. - 화이트 (운율 · 호흡, 음악적 어조): 행 길이와 호흡의 단조로움을 짚고 변주를 제안한다. ## 진행 규칙 요소 3 · 절차 1. 수신: 받은 시 초고를 인용 블록으로 다시 보여준다 2. 비평: 래드 → 골드 → 퍼플 → 화이트 순으로 각자 한 문단 3. 합의: 블랙이 4가지 개선 방향(①②③④)으로 합의문 정리 4. 재비평: 수정본을 받으면 같은 절차를 반복하되, 합의 단계에서 이전 대비 개선된 점과 남은 과제를 명시한다 ## 출력 형식 요소 4 · 형식 > (원문 인용) **래드 (평론가)**: … **골드 (독자)**: … **퍼플 (언어 미학)**: … **화이트 (운율)**: … **블랙의 합의 · 개선 방향** ① … ② … ③ … ④ … ## 하지 말 것 요소 5 · 경계 - 사용자가 명시적으로 요청하지 않으면 수정본을 직접 써주지 않는다 (방향만 제시) - 페르소나끼리 반박하며 토론을 늘이지 않는다 (각자 한 문단) - 산문·일기·메모에는 이 스킬을 적용하지 않는다
스킬을 호출하면 이런 결과가 나옵니다

래드"붉게 타오르고", "바람에 흩날린다" 모두 클리셰입니다. 그 저녁에만 발견할 수 있는 색을 한 단어로 잡아내세요.
골드한 번 읽고 머리에 남는 한 장면이 없습니다. 한 사물, 한 동작에 집중해 주세요.
퍼플노을과 낙엽이 정서를 직접 떠메고 있습니다. 감정을 사물에 담아 객관물화해 주세요.
화이트네 행 모두 같은 호흡과 길이입니다. 한 행은 짧게 끊어 음악성을 만드세요.
블랙의 합의 개선 방향 ① 클리셰 교체 ② 한 장면 집중 ③ 사물로 감정 객관화 ④ 호흡 변주

스킬 만들기 : 4가지 방법

사용자지정 → 스킬 → + 스킬 만들기에서 시작

진입 경로

claude.ai → 좌측 하단 사용자 지정 → 스킬 탭 → 우측 상단 + 버튼 → 스킬 만들기

1
Claude와 함께 창작하기
가장 쉬운 방법 · 대화로 스킬 완성

Claude에 내장된 skill-creator 스킬을 활용하는 방식입니다. "어떤 스킬을 만들고 싶은지" 대화로 설명하면 skill-creator가 자동으로 SKILL.md를 작성해줍니다.
기존 양식 파일(PPT·Word·PDF)을 첨부하면 구조까지 함께 분석합니다.

💡 SKILL.md 구조를 몰라도 OK · skill-creator가 대화만으로 완성
2
스킬 지침 작성
직접 작성 · 세밀한 제어

SKILL.md의 내용을 직접 입력합니다. name, description, Instructions 등을 원하는 대로 세밀하게 작성할 수 있습니다.
스킬 구조를 이해하고 있다면 가장 정교한 결과물을 만들 수 있는 방법입니다.

💡 중급 이상 사용자에게 추천
3
스킬 업로드
zip 파일 업로드 · 에셋 포함 가능

SKILL.md와 에셋(템플릿, 이미지 등)을 zip으로 압축해서 업로드합니다.
외부에서 만든 스킬이나 다른 사람이 공유한 스킬을 가져올 때 사용합니다.

💡 zip 안에 SKILL.md 필수 · 파일명 대문자 주의
4
GitHub에서 가져오기
남이 만든 스킬 활용 · 가장 빠른 시작
🔗 스킬 라이브러리 →

Anthropic 공식 저장소(anthropics/skills)나 다른 사용자가 공개한 GitHub 저장소에서 스킬을 받아옵니다.
저장소를 zip으로 다운로드해 업로드하거나, SKILL.md 내용을 복사해 스킬 지침 작성에 붙여넣으면 됩니다.

💡 검증된 스킬을 0초 만에 가져와서 내 환경에 맞게 수정 가능
추천 순서

입문자는 ④ GitHub에서 검증된 스킬을 받아 ② 스킬 지침 작성에 붙여넣고 시작해보세요. 어떤 구조인지 먼저 익히면 됩니다.
감이 잡히면 ① Claude와 함께 창작하기로 내 업무에 맞춘 스킬을 만들고, 더 정교하게 다듬고 싶으면 ② 스킬 지침 작성으로 직접 편집합니다.

Anthropic 스킬 설계 원칙 4가지

이 페이지의 5가지 구성 요소는 Anthropic 엔지니어링 블로그가 제시한 스킬 설계 원칙 4가지를 입문자가 SKILL.md에 바로 적용할 수 있도록 풀어 쓴 것입니다

I

점진적 정보 공개 · Progressive disclosure

메타데이터 → SKILL.md 본문 → 추가 링크 파일 순으로 Claude가 필요한 만큼만 정보를 단계적으로 불러오게 합니다. 컨텍스트 토큰을 아끼면서도 필요한 맥락은 모두 닿도록 구조화하는 핵심 원리입니다.

II

확장 가능한 구조 · Structure for scale

SKILL.md 본문이 비대해지면 절차·형식·예시를 별도 파일로 분리합니다. 자주 쓰지 않는 맥락을 따로 두면, 핵심 본문은 가볍게 유지되고 스킬은 무한히 커질 수 있습니다.

III

Claude 관점에서 생각 · Think from Claude's perspective

이름과 description이 호출 결정을 좌우합니다. 실제 사용 패턴을 관찰하면서 Claude가 호출을 망설이는 지점, 페르소나가 흔들리는 지점을 찾아 다듬습니다.

IV

Claude와 반복 개선 · Iterate with Claude

성공한 워크플로와 자주 일어나는 실패 모드를 함께 정리해 스킬에 다시 반영합니다. 가정이 아니라 실제 동작을 근거로 다듬어야 스킬이 안정됩니다.
