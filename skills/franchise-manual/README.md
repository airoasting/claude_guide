# F&B 프랜차이즈 운영 매뉴얼

본사와 가맹점의 전 부서 운영 SOP를 다섯 에이전트가 분석·기획·작성·검증·배포 준비까지 한 번에 만든다.

## 무엇을 하나

부서별로 흩어진 운영 절차를 일관된 톤과 구조로 통합한다. 작성과 검증을 다른 에이전트가 맡아 객관성을 확보하고, 부서별 라이터를 병렬로 돌려 수 주 분량을 하루 안팎으로 압축한다. 사람이 직접 쓰면 약 12주 걸리는 작업이다.

## 다섯 에이전트

| 에이전트 | 역할 |
|---|---|
| 벤치마크 분석가 | reference 톤·구조 정량 추출 |
| 운영기획본부장 | 설계서와 작업 목록 작성 (사람 승인 게이트) |
| SOP 라이터 | 부서별 챕터 작성 (병렬) |
| QSC 검수관 | 4개 기준 자동 검증 |
| 현장 배포 슈퍼바이저 | 배포 계획서와 공지문 작성 |

## 언제 쓰나

- 5개 이상 부서의 운영 매뉴얼을 한꺼번에 만들거나 개정할 때
- "프랜차이즈 매뉴얼", "본사 SOP", "QSC 매뉴얼", "부서별 표준작업절차" 같은 요청
- 신규 가맹점 오픈이나 정기 개정으로 표준 세트가 필요할 때

부서가 2~4개면 라이터 수를 줄여 축소 운영한다. 챕터 한 개 수정이면 일반 편집이 낫다.

## 실행

작업 디렉터리를 이 폴더로 두고 `SKILL.md`의 STEP 1~7을 따른다. 흐름은 분석 → 기획(사람 승인) → 병렬 작성 → 검증 → 재작업 → 배포 준비다. 상태는 `shared/feature_list.json`과 git에 남아, 세션이 끊겨도 그대로 이어서 작업한다.

상태 파일 검증 게이트:

```bash
python3 scripts/validate_feature_list.py
```

## 산출물

- `chapters/{부서}/Ch*.md` 부서별 매뉴얼 챕터
- `shared/` 설계서·용어집·KPI 감사·검증 이력
- `output/` 배포 계획서, 가맹점 공지문, 변환 이력

## 구성

- `SKILL.md` 실행 절차와 검증 기준
- `.claude/agents/` 다섯 에이전트 정의
- `scripts/validate_feature_list.py` 상태 파일 검증 게이트
- `schema/`, `reference/`, `docs/USAGE_GUIDE.md`

## 이런 건 하지 않는다

- 챕터 한 개 윤문·번역·요약
- 완성된 매뉴얼의 단순 점수 평가
- 매뉴얼이 아닌 일반 문서 작성

가맹점 배포 전 노무·세무·계약 챕터는 사람의 검토가 필요하다.
