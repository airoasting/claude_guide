# output/ 폴더 — PPTX 변환 결과

이 폴더에는 `chapters/` 안의 44개 .md 챕터를 **매뉴얼 v4 빌더**로 변환한 PPTX 파일이 부서별로 저장됩니다.

---

## 📋 변환 절차

### 1) 매뉴얼 v4 빌더 준비
파일명: `매뉴얼_자동생성기_v4.html`

이 빌더는 본 하네스에 포함되지 않으며, 사용자가 별도로 보유한 HTML 파일입니다. 빌더가 제공하는 기능:
- 마크다운 → PPTX 자동 변환
- 좌측 빨간 밴드(`#B91C1C`, 폭 8%) 자동 삽입
- 노란 박스(`#FEF3C7`) 도식화 영역 변환
- 한글 폰트 `+mj-ea`, line-height 150% 적용
- 제목 20pt / 본문 13pt / 캡션 11pt
- STEP 흐름 표 → 가로 화살표 SmartArt 변환
- 체크리스트 → 체크박스 표 슬라이드

### 2) 변환 명령

빌더의 사용 방식에 따라 두 가지 옵션:

**A. 일괄 변환 (권장)**
```
빌더 HTML 열기 → '폴더 일괄 변환' 모드 선택
→ chapters/ 디렉터리 지정
→ output/ 디렉터리 지정
→ 변환 실행
```

**B. 부서별 순차 변환**
```
빌더 HTML 열기 → '단일 파일 변환' 모드 선택
→ 챕터 .md 파일 선택
→ output/{부서}/Ch{NN}_{제목}.pptx 저장
```

### 3) 결과 검증

각 부서 폴더에 다음 개수의 PPTX가 생성되어야 함:

| 부서 | PPTX 파일 수 |
|---|---|
| 해산물물류 | 7개 |
| 조리 | 8개 |
| 홀서비스 | 6개 |
| 가맹운영지원 | 7개 |
| 마케팅 | 5개 |
| 인사·교육 | 6개 |
| 재무·정산 | 5개 |
| **합계** | **44개** |

## 🎨 디자인 호환성 체크

빌더가 변환할 때 다음 마크다운 요소가 PPTX로 매핑됩니다 (`shared/design_spec.md` §5.2 참조):

| 마크다운 요소 | PPTX 변환 결과 |
|---|---|
| `## [매뉴얼 목적]` | 좌측 빨간 밴드 + 섹션 헤더 슬라이드 |
| `### 제 N 조 (...)` | 노란 박스 헤더 |
| STEP 흐름 표 (1행 N열) | 가로 화살표 SmartArt |
| STEP 요약 박스 표 | 카드형 박스 (3개 액션) |
| `※` 시작 라인 | 회색 작은 글씨 캡션 |
| CHECK LIST 표 (4컬럼) | 체크박스 표 슬라이드 |
| `# 챕터 제목` | 표지 슬라이드 |

### 변환 전 사전 체크 (이미 모든 챕터 통과)

- [x] L1은 평문(`# ` 사용 금지) — 빌더가 표지 인식 시 충돌 방지
- [x] h1은 챕터 제목 1개만 — 빌더가 단일 표지 생성
- [x] 표 문법 정상 (`|` 정렬, 헤더 구분선)
- [x] 헤딩 레벨 계층적 (h1 → h2 → h3)
- [x] 깨진 문자(`�`) 없음
- [x] 코드 블록 ``` 펜스 사용

## ⚠️ 변환 실패 시 대응

빌더가 특정 챕터에서 오류를 낼 경우:

1. **표 깨짐**: 챕터의 4컬럼 체크리스트 표 정렬 확인 (헤더 구분선 `|---|---|---|---|`)
2. **헤딩 레벨 충돌**: 해당 챕터 첫 5줄 점검 → L1 평문 / L3 `# 제목` 형식 일치 여부
3. **부록 표 변환 실패**: 부록 표의 컬럼 수 일관성 확인 (모든 행이 같은 `|` 개수)
4. **이미지 누락**: 본 매뉴얼은 이미지 미사용. 빌더가 이미지 자리표시자를 생성한다면 무시

문제 챕터는 chapters/{부서}/{파일} 직접 수정 후 재변환.
