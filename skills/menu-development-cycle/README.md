# 메뉴개발 사이클 (Menu Development Cycle)

외식 브랜드의 신메뉴를 기획부터 출시까지 단계별로 함께 개발하는 사용자 주도형 스킬입니다.

## 이 스킬이 하는 일

신메뉴 개발은 기획·시제품·원가·시연·표준화·매뉴얼·교육·출시까지 여러 단계를 거칩니다. 이 스킬은 그 과정을 자동으로 완주하지 않고, 단계마다 사용자에게 의견을 묻고 동의를 받아야 다음으로 넘어갑니다. 현장 사정(보유 식재료, 기물, 브랜드 정체성, 목표 마진)을 가장 잘 아는 사람은 사용자이기 때문입니다. 각 단계의 산출물은 HTML 조각으로 채워 단일 HTML로 병합·전달하며, 사용자의 의견과 제약은 산출물에 "사용자 의견" 안건으로 명시 기록해 끝까지 추적합니다.

## 언제 쓰나

다음과 같은 표현이나 요청에서 발동합니다.

- "신메뉴 개발", "메뉴 개발 시작", "메뉴 개발 사이클", "레시피 개발"
- "신메뉴 기획서 만들어줘", "경쟁사 분석표", "원가 레시피 설계", "시제품 개발"
- "사이클 진행", "처음부터 끝까지 개발", "/menu-development-cycle"
- 브랜드명(예: 인생아구찜, 어화락, 삼대미역)에 컨셉이나 메뉴 방향을 덧붙여 신메뉴 개발을 의뢰할 때

다음은 이 스킬의 영역이 아닙니다. 이미 완성된 레시피의 단순 원가 계산, 메뉴판 디자인이나 가격표 인쇄, 일반 가정용·1회성 요리 레시피 질문, 식당 창업·상권 분석 전반, 마케팅 카피나 SNS 콘텐츠 제작.

## 입력과 출력

**입력**: 시작에 필요한 최소 정보는 두 가지입니다.

- 브랜드명 (예: 어화락, 인생아구찜, 삼대미역, 인생식탁)
- 대략적인 컨셉

둘 다 있으면 1단계를 시작하고, 없으면 먼저 묻습니다. 이후 단계에서는 보유 식재료·매장 기물류 현황, 목표 판매가, 실제 매입 단가 같은 사용자만 아는 정보를 단계마다 확인합니다.

**출력**: 단계별 산출물을 HTML 조각으로 채워 단일 HTML로 병합해 전달합니다. 대표 산출물은 다음과 같습니다.

| 단계 | 산출물 | 형식 |
| --- | --- | --- |
| 1-1 기획 | 신메뉴 기획서 | A4 3~4장, 별도 HTML |
| 1-1 기획 | 경쟁사 분석표 (3~5곳) | A4 3~4장, 별도 HTML |
| 2-1 시제품 | 레시피 설계서·테스트 기록지 (3~5안, 원가 자동 재계산) | A4 4장, 별도 HTML |
| 2-2 ~ 6-1 | 원가·수율, 시연, 표준화, 매뉴얼, 교육, 출시 점검 | 단일 HTML 병합 |

모든 독립 문서와 개발사이클 HTML 상단에는 툴바가 있어 엑셀(.xls)·워드(.doc) 저장과 PDF·인쇄가 가능합니다. 원가·단가는 추정값임을 명시하고, 확인 못 한 항목은 빈칸 대신 "미확인"으로 둡니다.

## 산출물 예시

실제로 어떤 결과물이 나오는지는 `sample.html`을 열어 확인하세요.

## 사용법

Claude에 스킬로 등록한 뒤 자연어로 요청하면 됩니다.

```
어화락 브랜드로 매운 해산물 신메뉴 개발 시작해줘
```

각 단계 끝에서 사용자 확인 게이트가 제시됩니다. "이대로 진행" 또는 "수정 의견"을 답하면, 동의 시 다음 단계로 넘어가고 수정 의견은 산출물 안건에 반영해 다시 확인합니다.

HTML 병합은 내부적으로 다음 명령으로 수행됩니다.

```bash
python3 assets/merge.py --stages 1-1,2-1 --out {브랜드}_{메뉴}_개발사이클.html
# --stages 생략 시 fragments에 있는 모든 스테이지 병합
```

완료한 스테이지만 병합되어 단일 HTML이 되고, 매 단계 완료 시 같은 파일명을 덮어씁니다.

## 폴더 구성

```
menu-development-cycle/
  SKILL.md                  기능·트리거·출력 형식 정본
  README.md                 이 문서
  LICENSE
  sample.html               산출물 예시
  references/
    workflow.md             사용자 확인 게이트·1-1/2-1 상세 흐름
    stages.md               스테이지 스펙 + 완료 점검표
    personas.md             페르소나 프로파일 (참고용 토의)
  assets/
    _shell.html             골격(스타일·헤더·진행바·스크립트)
    merge.py                조각 → 단일 HTML 병합
    build_fragments.py      2-2~6-1 빈 조각 생성기
    doc_toolbar.js          엑셀·워드·PDF·인쇄 툴바
    doc_planning.html       신메뉴 기획서 템플릿
    doc_competitor.html     경쟁사 분석표 템플릿
    doc_recipe.html         레시피·테스트 겸용 템플릿
    doc_cost.html           원가 문서 템플릿
    doc_demo.html           시연 문서 템플릿
    fragments/
      phase_1~6.html        페이즈 헤더
      stage_1-1 ~ 6-1.html  스테이지별 조각
      _outputs.html         BOM·레시피·알레르기·체크리스트·출처
```
