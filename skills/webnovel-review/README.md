# 웹소설 회차 검토

웹소설 한 화를 5인 멀티 페르소나로 10점 만점 진단하고, 닿을 수 있으면 합격선 9.5까지 다듬되 못 닿으면 정직하게 보고하는 회차 검토 스킬입니다.

## 이 스킬이 하는 일

웹소설 연재분 또는 단편 한 화를 분석가 3인(블랙·레드·블루)과 독자 2인(그린·옐로우)이 각자의 시선으로 냉정하게 채점합니다. 미달이면 메인 페르소나 블랙이 문장·구성·정보 배치 안에서 수정본을 만들어 점수가 어떻게 움직이는지 보여줍니다. 합격선은 9.5점(그 장르 상위 회차와 나란히 둘 수 있는 화)이며, 다듬어서 닿으면 합격본을, 닿지 않으면 합격을 가장하지 않고 현재 최선본과 막힌 원인을 보고합니다. 묻힌 보석은 닦지만 없는 보석을 만들지는 않습니다.

## 언제 쓰나

- "내 웹소설 회차 봐줘", "이번 화 검토·피드백·합평·퇴고·교정해줘"
- "소설 한 화 점수 매겨줘", "연재분 다듬어줘"
- "클리프행어(절단) 약한지 봐줘", "회차 평가"
- 웹소설·소설 회차(연재분 또는 단편) 원고를 붙여넣고 봐달라고 할 때(검토라는 말을 직접 쓰지 않아도 발동)
- `/webnovel-review`

발동하지 않는 경우. 시·산문시 평가(다른 스킬), 줄거리·설정 브레인스토밍, 작품 전체 기획 컨설팅, 단순 사실 검색·계산, 이미 검토가 끝난 회차의 요약·번역. "그냥 오탈자만 빨리" 요청이면 블루 단독 교정 모드로 축소해 발동합니다.

## 입력과 출력

작업을 시작하기 전에 다음을 받습니다.

| 구분 | 항목 | 필수 여부 |
|---|---|---|
| (a) | 원고 본문(한 화 분량, 붙여넣기 또는 .txt·.docx 파일) | 필수 |
| (b) | 장르(정통·퓨전·어반 판타지, 다크판타지, 로맨스, 로판, 무협 등) | 필수 |
| (c) | 연재 위치·연속성 메모(몇 화인지, 직전 화 요약, 복선 메모) | 선택 |
| (d) | 이번 화 의도(작가가 노린 것) | 선택 |

(a)와 (b)가 비면 시작하지 않고 사용자에게 요청합니다. 연재 모드(기본)와 단편 모드를 구분하며, 단편이면 절단·견인력을 결말·여운으로 치환해 봅니다.

출력은 결과에 따라 둘로 나뉩니다.

- **합격 시(3종)**: 검토 리포트(화 성격·페르소나별 점수표·항목별 코멘트·연재처 심의·좋은 점·우선순위 수정 리스트), 인라인본(원문 → 수정 제안 + 색깔별 코멘트), 원문 ↔ 최종 수정 반영본 비교
- **미달 종료 시**: 현재 최선본, 막힌 지점, 정체 원인 진단, 라운드별 점수 추이, 다음 수

검토 항목(축)은 오탈자·반복/어색한 표현·도입부 훅·절단(견인력)·페이싱·정보 전달·캐릭터/전개 일관성·모바일 가독성·장르 체감/연속성이며, 여기에 진짜 강점만 최대 3가지를 더합니다. 노벨피아 19금 연재처 심의(등급 표기·수위 적합성)도 함께 점검합니다.

## 사용법

Claude에 스킬로 등록한 뒤 자연어로 요청하면 됩니다. 원고와 장르를 함께 전달하면 됩니다.

```
이번 화 검토해줘. 장르는 다크판타지, 23화야.
[원고 본문 붙여넣기]
```

별도의 설치·환경변수·외부 API 키·MCP 의존성은 필요하지 않습니다.
