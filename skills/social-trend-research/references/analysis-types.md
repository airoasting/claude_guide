# 분석 유형별 출력 템플릿 (6종)

각 유형은 아래 템플릿을 기본으로 한다. 모든 템플릿은 공통으로
**수집 기간 + 다룬 플랫폼 + 데이터 한계**를 포함한다.
인용은 SKILL.md의 공통 인용 형식([직접/간접] · 원문 · 번역 · 출처)을 따른다.
근거 없는 단정·환각 금지. 못 찾은 플랫폼은 비워두고 그 사실을 적는다.

---

## 0. 빠른 답변 (Quick Answer)
단순 질문에 대한 짧고 직접적인 답. 깊은 분석 없이 핵심만.

```
**빠른 답변: [질문]**

[2~4문장으로 결론 요약]

근거(실제 언급):
- [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크
- [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크

📅 수집: [기간] · 플랫폼: [다룬 곳]
⚠️ 한계: [공백/간접 비중 등, 있으면]
```

---

## 1. 랭킹 (Rank)
커뮤니티가 실제로 많이 추천·언급하는 순으로 정렬. SEO 리스트가 아니라 실제 언급 근거 기반.

```
**[주제] 랭킹 — 실제 언급 기반**
📅 수집: [기간] · 플랫폼: 더우인 · 샤오홍슈 · X

🥇 1위. [항목]
   왜: [추천/언급 이유 요약]
   - [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크
🥈 2위. [항목]
   왜: ...
   - 근거 인용...
🥉 3위. [항목] ...

플랫폼별 차이: [예: X는 A를 밀고, 샤오홍슈는 B 후기가 많음 등]
⚠️ 한계: [공백 플랫폼 / 간접 비중 / 표본 편향 등]
```

순위 근거가 빈약하면 순위를 억지로 매기지 말고 "언급량 상위 후보군"으로 제시한다.

---

## 2. 감성 (Sentiment)
긍정·부정·중립 분포와 대표 인용. 수치는 "확보한 표본 기준 대략적 인상"임을 분명히 한다.

```
**[주제] 여론 감성 분석**
📅 수집: [기간] · 플랫폼: ...

전체 분위기: [긍정 우세 / 부정 우세 / 팽팽 / 혼재]
대략적 분포(확보 표본 기준, 정밀 통계 아님): 긍정 ○ · 부정 ○ · 중립 ○

👍 긍정
- [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크
👎 부정
- [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크
😐 중립/혼재
- ...

플랫폼별 온도차: [예: X는 비판적, 샤오홍슈는 호의적 등]
한 줄 결론: [...]
⚠️ 한계: [표본 적음 / 더우인 간접 등]
```

퍼센트는 "엄밀한 통계"로 오해되지 않게 항상 "확보 표본 기준 인상"임을 적는다.

---

## 3. 트렌드 (Trend)
언제 떴는지, 지금 오르는지/식는지, 무엇이 스파이크를 만들었는지.

```
**[주제] 트렌드 흐름**
📅 수집: [기간] · 플랫폼: ...

현재 모멘텀: [상승 ↑ / 유지 → / 하락 ↓]

타임라인:
- [시점]: [무슨 일 — 계기/이벤트]  근거: 링크
- [시점]: [급등/급락 지점과 원인]  근거: 링크
- [현재]: [지금 상태]

스파이크 원인: [영상/이슈/인물/협업 등]
플랫폼별 차이: [어디서 먼저 떴고 어디로 번졌나]
⚠️ 한계: [시계열 근거가 간접이면 명시]
```

시점 정보가 추정이면 "추정"이라고 표기. 정확한 날짜를 지어내지 않는다.

---

## 4. 논쟁 (Controversy)
찬반/대립 구도를 양쪽 논거와 함께 지도화. 한쪽으로 기울지 않게 균형.

```
**[주제] 논쟁 구도**
📅 수집: [기간] · 플랫폼: ...

쟁점 한 줄: [무엇을 두고 갈리나]

🅰️ 입장 A: [요약]
   핵심 논거: [...]
   - [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크
🅱️ 입장 B: [요약]
   핵심 논거: [...]
   - [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크

대립의 핵심: [양쪽이 끝까지 안 맞는 지점]
플랫폼별 경향: [예: X에 A 다수, 샤오홍슈에 B 다수 등]
⚠️ 한계: [...]
```

어느 쪽이 "옳다"고 판정하지 않는다. 양쪽 논거를 충실히 보여주는 게 목적.

---

## 5. 발굴 (Discovery)
아직 메인스트림 전, 지금 막 떠오르는 키워드/주제를 포착. 주제 없이 "요즘 뭐 떠?"류 질문에 대응.

```
**지금 뜨는 신호 — [영역/플랫폼]**
📅 수집: [기간] · 플랫폼: ...

1. [떠오르는 키워드/주제]
   왜 뜨나: [계기]
   주도 플랫폼: [어디서 먼저]
   - [직접/간접] (플랫폼) "원문" → "번역"  출처: 링크
2. [키워드] ...
3. [키워드] ...

📌 한 줄 요약: [전체 흐름]
⚠️ 한계: [신생 키워드라 근거가 적을 수 있음 등]
```

"곧 터질 것"이라는 예측은 단정하지 말고 "초기 신호" 수준으로 표현한다.

## 공통 마무리 (모든 유형)
- 데이터 한계를 마지막에 한 번 더 솔직하게.
- 다음 단계 제안 한 줄(특정 플랫폼 심화 / 다른 분석 유형 전환 / 시각화 등).
