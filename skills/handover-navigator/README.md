# Handover Navigator (인수인계 내비게이터)

전임자가 인수인계 없이 떠나 자료 더미만 남았을 때, 그 더미를 맥락 0인 후임자가 첫 30일을 버틸 수 있는 인수인계 문서로 바꿔주는 Claude 스킬입니다.

## 이 스킬이 하는 일

퇴사·휴직·발령으로 떠난 전임자가 남긴 문서·이메일·메모·스프레드시트·회의록·폴더를 받아, 지금 무엇이 진행 중이고 후임자가 무엇을 이어받아야 하는지 재구성합니다. 모든 정보를 6블록 캔버스(현황·미결·자산·관계·맥락·리스크)로 정리하고, 인수인계 요약서·미결 과제 트래커·온보딩 체크리스트·물어볼 질문 리스트를 만듭니다. 단순 요약기와 다른 점은 모르는 것을 지어내지 않는다는 원칙입니다. 모든 사실에 근거 파일을 표기하고, 자료로 확인되지 않는 것은 추측 대신 `UNKNOWN`으로 남깁니다.

## 언제 쓰나

다음과 같은 요청이나 상황에서 발동합니다.

- "인수인계서 만들어줘", "인계서 만들어줘"
- "전임자(퇴사자)가 남긴 자료 파악해줘", "이 폴더 정리해서 인계서 만들어줘"
- "담당자가 바뀌었는데 뭘 이어받아야 하는지 모르겠어"
- "갑자기 이 업무를 맡게 됐어", "앞사람이 정리 안 하고 나갔어"
- 업무 인계·넘겨받기(takeover), 지식 이전(knowledge transfer), 신규 담당자 온보딩

키워드를 명시하지 않아도, 떠난 사람이 남긴 자료에서 현황과 인계 대상을 재구성해야 하는 상황이면 사용합니다. 단일 문서 요약·번역, 빈 양식 신규 작성, 단순 파일 편집·이름변경, 자료 더미 없이 양식만 달라는 요청에는 발동하지 않습니다.

## 입력과 출력

입력은 전임자가 남긴 자료 더미입니다. 폴더 경로, 파일 묶음, 또는 붙여 넣은 메모·이메일 텍스트 모두 가능합니다.

출력은 아래 문서입니다. 원본은 읽기 전용으로 두고, 산출물은 `handover-output/`(또는 사용자가 지정한 폴더)에만 씁니다. 실행 항목은 `- [ ]` 체크박스로 작성해, 인계서가 곧 후임자의 작동하는 체크리스트가 되게 합니다.

| 산출물 | 내용 |
| --- | --- |
| 인수인계 요약서 | 1페이지 핵심 + 6블록 상세 (상사 보고용) |
| 미결 과제 트래커 | 과제·상태·담당자·기한·신뢰도·출처 표 |
| 첫 30일 온보딩 플랜 | 1주·2주·한 달 단계별 체크리스트 |
| 물어볼 질문 리스트 | 자료로 확인 불가한 빈칸을 "누구에게 무엇을 물을지"로 정리 |
| 신규 기획서 초안 (옵션) | 파악한 현황·맥락·미결을 토대로 한 다음 문서의 골격 초안 |

모든 사실에는 근거 파일을 표기하고, 신뢰도를 `문서확인` · `추론`(근거 동반) · `UNKNOWN` 세 단계로 구분합니다. 진행 상태는 제각각인 표현("홀딩", "거의 끝", "?")을 미착수·진행중·막힘(Blocked)·완료·불명(UNKNOWN) 다섯 개 표준 어휘로 정규화합니다.

## 산출물 예시

실제로 어떤 결과가 나오는지는 `sample.html`을 열어 확인할 수 있습니다. 예시 산출물 5종은 `examples/` 폴더에도 들어 있습니다.

## 사용법

Claude의 스킬 지원 환경(Cowork, Claude Code 등)에 스킬로 등록한 뒤, 자연어로 요청하면 됩니다. 별도의 설치 의존성이나 API 키, 환경변수는 필요하지 않습니다. 카카오톡 등 메신저로 1페이지 요약을 전송하는 옵션을 쓰려면, 연결된 메신저 도구(MCP 등)가 있을 때 사용합니다. 없으면 복사해 붙여 넣을 수 있는 요약 텍스트 블록을 제공합니다.

데모용 호출 예시입니다. `sample/` 폴더는 일부러 지저분하게 구성해, 스킬이 혼란을 어떻게 정리하고 무엇을 `UNKNOWN`으로 남기는지 보여줍니다.

```
이 sample 폴더가 퇴사한 전임자가 남긴 자료야. 인수인계서 만들어줘.
```
