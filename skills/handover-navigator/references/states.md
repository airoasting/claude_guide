# 진행 상태 표준 어휘

전임자가 제멋대로 적어둔 상태 표현을 아래 5개 표준값 중 하나로 정규화한다. 표준값과 별칭을 한 곳에서 관리해, 인계서를 쓰는 쪽과 읽는 쪽이 같은 어휘를 쓰게 한다.

## 표준값

- **미착수** — 아직 시작 안 함
- **진행중** — 현재 작업 중
- **막힘(Blocked)** — 외부 요인으로 멈춤 (회신·결재·자원 대기 등)
- **완료** — 끝남 (완료 근거가 문서에 있을 때만)
- **불명(UNKNOWN)** — 자료로 상태를 판단할 수 없음

## 별칭 매핑

| 자료에서 자주 보이는 표현 | 표준값 |
| --- | --- |
| TODO, 예정, 대기, 시작 전, "아직", "착수 예정" | 미착수 |
| 진행, 작업중, ing, WIP, "하는 중", "처리중" | 진행중 |
| 홀딩, hold, pending, 보류, "기다리는 중", "회신 대기", "결재 대기" | 막힘(Blocked) |
| done, 완료, 종료, "끝", "마무리", "발송함", "제출 완료" | 완료 |
| ?, 공란, "글쎄", "확인 안 됨", 날짜만 있고 상태 없음 | 불명(UNKNOWN) |

## 판정 주의

- "거의 끝", "막바지" 같은 표현은 **완료가 아니라 진행중**이다. 완료는 완료 근거가 있을 때만.
- "홀딩"은 종료가 아니라 **막힘**이다 — 후임자가 풀어야 할 일로 남긴다.
- 오래 활동이 없다는 이유만으로 "완료"나 "폐기"로 단정하지 않는다. 활동 없음 = 불명 + 리스크 후보.
