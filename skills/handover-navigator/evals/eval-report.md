# 평가 리포트

이 스킬의 두 가지 약속을 실측했다: ① **환각 방지가 실제로 작동하는가**(RED vs GREEN 대조), ② **필요할 때 켜지고 비슷한 요청엔 안 켜지는가**(발동 정확도).

---

## 1. 환각 방지 — RED vs GREEN 대조 실험 (핵심)

"규칙이 효과 있다"를 주장이 아니라 증명으로 만들기 위해, 동일한 `sample/` 더미를 **스킬 없이(RED)** 와 **스킬 적용(GREEN)** 으로 각각 처리해 비교했다.

- **RED**: 빠른 모델(Haiku)에 스킬 없이 "윗선 보고용 현황표 빠르게" 요청 — 무방비·속도 우선 요약의 현실적 재현.
- **GREEN**: 같은 더미를 Handover Navigator로 처리.
- **기준일**: 2026-06-23.
- **재현용 원본 출력**: RED 실제 산출 → `red-output-haiku.md`, `red-output-opus.md` / GREEN 실제 산출 → `../examples/`.

### RED가 지어낸/단정한 것 → GREEN의 처리

| # | RED(스킬 없음) 산출 | 원본 사실 | GREEN(스킬) 처리 |
| --- | --- | --- | --- |
| A | 월 매출 목표를 **"월평균 10억"** 으로 표기 | 자료엔 1·2월 목표 **9억**만 있음. 10억은 어디에도 없음(연 타겟 120억에서 임의 역산) (출처: 02) | 수치는 출처 있는 것만. 목표 9억/연 120억을 출처와 함께, **없는 월목표는 만들지 않음** |
| B | 약가 마감 5/22·5/29를 **"긴급·마감 임박"** 으로 표기 | 두 날짜 모두 기준일(6/23) 기준 **약 한 달 경과** (출처: 05, 06) | "오늘과 비교" 규칙 → **경과(미확인) + Blocker**로 표기 |
| C | 약가 대응 상태를 **"미진행"** 으로 단정 | 원본은 "진행중이었으나 전달 완료 여부 **불명**" (출처: 06) | 단정 금지 → **불명(UNKNOWN)**, "물어볼 질문"으로 이관 |
| D | 가정의학 KPI 담당을 **"정현우/영업팀"** 으로 배정 | 원본은 "누가 트래킹? 나? 영업팀? **애매**" (출처: 04) | **UNKNOWN** + 질문 리스트("KPI 담당 누구?")로 |
| E | 5월 가정의학 **"효과 관찰 시작"** 으로 단정 | 원본은 "효과 보기 시작**? 좀 더 봐야**" (출처: 02) | 불확실성 보존 → **추론(근거 명시)** |
| F | **출처 인용 0건** (전부 무출처 단정) | — | 모든 사실에 **(출처: 파일)** 부착 |

### 해석

흥미롭게도 **강한 모델(Opus)** 로 RED를 돌리면 환각이 훨씬 적었다(추정/불명을 스스로 표기, 날짜 경과도 일부 포착). 즉 환각 위험은 모델 강도에 의존한다. 그러나 강한 모델조차 RED에서는 **항목별 출처 인용이 없고, 빈칸을 "누구에게 물을지"로 구조화하지 못했다.**

이 스킬의 가치는 두 가지다: (1) 빠른/무방비 모델의 **노골적 환각(A~E)을 차단**하고, (2) 모델 강도와 무관하게 **출처·신뢰도·질문화·완료 게이트라는 구조적 보증(F 등)을 강제**한다. 즉 "운 좋게 신중한 출력"이 아니라 "항상 검증 가능한 출력"으로 만든다.

## 2. 발동 정확도 (트리거)

`available_skills`에 handover-navigator를 경쟁 스킬 10종과 함께 두고, 각 발화에서 발동되는지 독립 판정했다. 평가셋: `trigger-eval.json`(20문항) + 어려운 엣지 6문항 + 축소 description 회귀 12문항.

| 라운드 | 대상 | 결과 |
| --- | --- | --- |
| Baseline | 20문항 | should-trigger 10/10, 오발동 0/10 → **20/20** |
| Stress(엣지) | 6문항 | 암묵 발동 3/3, 근접 오답 차단 3/3 → **6/6** |
| 축소 description 재검증 | 12문항 | **12/12**, 회귀 없음 |

근접 오답(키워드만 겹치는 함정)도 전부 차단: '온보딩' 슬라이드 제작, '퇴사' 이메일 작성, '인수인계' 규정 양식, '전임' 대통령 정책, 전임자 엑셀 단순 편집, 회의록 1개 요약.

### description 개선

- **트리거 중심으로 축소**: "무엇을 만든다"(deliverables)와 가치 문장을 본문으로 내리고, description은 **언제 쓰는지(트리거 + negative scope)** 만 남김. → 발동 정확도 유지(12/12) + 토큰 절감.
- **Negative scope 명시**: "단일 문서 요약 / 빈 양식·정책 템플릿 / 단순 파일 편집엔 쓰지 말 것" → 과발동 차단.

## 3. 방법론·한계

- 자동 최적화 루프(skill-creator `run_loop`, `claude -p` 기반)는 본 환경에서 CLI 미로그인으로 실행 불가. 대신 동급 모델 서브에이전트를 판정자로 사용했다.
- 더 엄밀히 하려면 각 문항 3회 반복으로 분산을 보는 것이 좋다(`claude` 로그인 환경에서 권장).
- 완료 게이트는 현재 산출물의 self-check다. 더 높은 신뢰가 필요하면 독립 검증 단계(별도 claim-verify)를 연결하는 것이 다음 개선 과제다.
